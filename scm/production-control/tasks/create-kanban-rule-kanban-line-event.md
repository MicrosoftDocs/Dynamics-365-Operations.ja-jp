# <a name="create-a-kanban-rule-using-a-kanban-line-event"></a>かんばん明細行イベントを使用したかんばんルールの作成

[!include[task guide banner](../../includes/task-guide-banner.md)]

この手順では、プロセス活動からプルをトリガするかんばん明細行イベント設定を使用して、かんばんルールを作成します。 かんばんルールは、それぞれの数量が 25 かまたはそれより多くの数量のかんばんのプロセス活動によって発生します。 このタスクの作成に使用するデモ データの会社は USMF です。 このタスクは、リーン環境で新しい製品または変更された製品の生産を準備している、プロセス エンジニアまたはバリュー ストリームのマネージャーを対象としています。


## <a name="create-a-kanban-rule"></a>かんばんルールの作成
1. [製品情報管理] > [リーン生産] > [かんばんルール] の順に移動します。
2. [新規] をクリックします。
3. [補充方法] フィールドで、「イベント」を選択します。
    * これにより、かんばんは需要から直接生成されます。 受注生産シナリオを定義するルールを設定するのに使用されます。  
4. [最初の計画活動] フィールドで、値を入力または選択します。
    * SpeakerAssemblyAndPolish を入力または選択します。 製造かんばんルールの最初の活動は、生産フローのプロセス活動です。 活動を選択すると、その活動の有効期間がかんばんルールの有効期間にコピーされます。  
5. [詳細] セクションを展開します。
6. [製品] フィールドに「L0001」を入力します。
7. [イベント] セクションを展開します。
8. [かんばん明細行イベント] フィールドで、「自動」を選択します。
    * これにより、必要に応じたイベントのかんばんが生成されます。  ダウンストリームのプロセス活動に必要な材料を補充するかんばんルールをコンフィギュレーションするには、このフィールドが使用されます。 「自動」を選択すると、イベントのかんばんが需要と共に作成されます。 この設定は、同じ日に生産を実行する予定がある場合にお勧めします。  
9. [最小イベント数量] を「25」に設定します。
    * 需要数量がこのフィールド以上の場合、イベントのかんばんが生成されます。 ある機械ではこのフィールドより少ない注文数量を生産し、別の機械ではこのフィールドより多くの注文数量を生産する場合に便利です。  
10. [保存] をクリックします。

## <a name="create-sales-order-and-trigger-kanban-chain"></a>販売注文を作成し、かんばんチェーンをトリガーします。
1. [販売とマーケティング] > [販売注文] > [すべての販売注文] の順に移動します。
2. [新規] をクリックします。
3. [顧客口座] フィールドで値を入力または選択します。
    * 「顧客口座 US-003、Forest Wholesales」を選択します。  
4. [OK] をクリックします。
5. [品目番号] フィールドに、'L0001' を入力します。
    * L0001 はかんばんルールを作成した品目です。  
6. 数量を '27' に設定します。
    * 27 はかんばんルールにおける最小数量 25 より多いため、これによりイベントのかんばんが発生します。  
7. [サイト] フィールドで、「1」と入力します。
8. [倉庫] フィールドで、値を入力または選択します。
    * 倉庫 13 を選択します。  
9. [保存] をクリックします。

## <a name="view-the-kanban-generated-by-the-kanban-rule"></a>かんばんルールによって生成されたかんばんの表示
1. [製品情報管理] > [リーン生産] > [かんばんルール] の順に移動します。
2. 一覧で、目的のレコードを見つけ、選択します。
3. [かんばん] セクションを展開します。
    * 作成済みのかんばんルールに基づいて活動を処理するために 27 のかんばんが作成されたことを確認します。  
    * これは最後のステップです。
