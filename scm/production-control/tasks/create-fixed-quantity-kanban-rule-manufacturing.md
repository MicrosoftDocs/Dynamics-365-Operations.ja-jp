# <a name="create-a-fixed-quantity-kanban-rule-for-manufacturing"></a>製造のための固定数量かんばんルールの作成

[!include[task guide banner](../../includes/task-guide-banner.md)]

この手順は、リーン環境の作業セルで、変換する活動をトリガーする固定の製造かんばんルールを作成するのに必要な設定を対象としています。 この手順の作成に使用するデモ データの会社は USMF です。 この手順は、新しい製品または変更された製品の生産を準備している、プロセス エンジニアまたはバリュー ストリームのマネージャーを対象としています。


## <a name="create-new-kanban-rule"></a>新しいかんばんルールの作成
1. [かんばんルール] に移動します。
2. [新規] をクリックします。
    * 既定のタイプが製造、また補充方法が固定になっていることに注目してください。 これらのパラメーターを変更する必要はありません。  
3. [最初の計画活動] フィールドで、値を入力または選択します。
    * これはこのかんばんルールに基づいて作成されたかんばんに対して実行される活動です。  「SpeakerTestAndPackaging」を選択します。  
4. [詳細] セクションを展開します。
5. [製品] フィールドで、値を入力または選択します。
    * [L0050] を選択します。  
6. [計量単位] フィールドで、値を入力または選択します。
    * 分を選択します。  
7. [リード タイム] フィールドで、数値を入力します。
    * 5 を入力します。  

## <a name="set-quantities"></a>数量の設定
1. [数量] セクションを展開します。
2. [既定の数量] を「10」に設定します。
    * これは、かんばんごとに転送される数量です。  
3. [製品数量の差異] チェック ボックスを選択します。
    * これを使用すると、選択されたかんばんが既定の数量との差異で完了できます。  かんばんが 8 から 12 の数量で完了するようにするには、両方の差異を 2 に設定します。  
4. [差異の上限] を「2」に設定します。
    * これにより、数量が減って 10 - 2 = 8 完了することになります。  
5. [差異の下限] を「2」に設定します。
    * これにより、数量が増えて 10 + 2 = 12 完了することになります。  
6. [固定かんばん数量] フィールドで、数値を入力します。
    * これは有効にするかんばんの量です。 この場合、5 つのかんばんが 、それぞれ 10 処理します。  
7. [下限値に対する警告] フィールドで、「2」を入力します。
    * 出荷先にあるべきすべてのかんばんの最小数量を追跡するために使用されます。 たとえば、これはかんばんの数量の概要に使用されます。  
8. [上限値に対する警告] フィールドで、「4」を入力します。
    * 出荷先にあるべきすべてのかんばんの最大数量を追跡するために使用されます。 たとえば、これはかんばんの数量の概要に使用されます。  
9. [自動計画数量] フィールドで、「1」を入力します。
    * これを 1 に設定すると、すべてのかんばんが自動的に計画されることになります。   3 を入力してしまうと、3 つの空のかんばんに計画の準備が整うまで、そのかんばんは計画されないことになります。 これは、作業をグループ化し、余計な設定を回避するのに役立ちます。  

## <a name="create-kanbans"></a>かんばんの作成
1. [かんばん] セクションを展開します。
2. [保存] をクリックします。
    * かんばんルールは、かんばんを作成する前に保存する必要があります。  
3. [追加] をクリックします。
    * この提案されている [新しいかんばんの数] が 5 であるため、有効なかんばんが存在しないことに注意してください。。 これは、[固定かんばん数量] と等しくなります。  
4. [作成] をクリックします。
    * これは、5 つのかんばんを作成します。  
    * この製造かんばんルールのために、10 それぞれに対して 5 のかんばんが作成されたことに注目してください。 これは、この手順の最後のステップです。
