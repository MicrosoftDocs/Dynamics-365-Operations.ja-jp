## <a name="purchase-process-for-subcontracted-activities-services"></a>使用される活動 (サービス) の購買プロセス
外注された活動の購買プロセスはkanbanジョブ進捗たとえば、開始して登録または完了基づいていますが、物理的に材料のフローに。 資金の現在、外注された作業のたとえば、原価は、物理的な流れに従って二次フローです。 同時に、購買プロセスは、ステップの注文のドキュメントを手動調整を有効にする独立したプロセスです。 外注された活動の購買プロセスを挙げます。:

1.  購買契約を作成します。 購買契約は、サービス用に作成され、生産フローの活動に関連付けます。
2.  発注書を作成します。 リリース発注書はスケジュールされたkanbanジョブに基づいてサービスに作成できます。 同じサービスのジョブは、日、週、または月ごとの購買注文明細行にグループ化できます。 購買注文明細行がkanbanジョブの作成後にいつでも作成できます。 購買注文明細行がファクト後で作成できます。 このオプションは通常、外注業者が外注業者が受け取るkanbansまたはkanbanカードに基づいて追加通知なしでサービスを提供すると、 この場合、発注書と請求書の誤差は、最小限にすることができます。
3.  下請業者に処理を準備するに送信するようにkanbanカード、材料とピッキング リストを生成します。 生産フローの詳細なモデリングに基づいて、準備、プロセス活動のkanban板材にピッキング リストや準備の機能を使用して実行されます。 また、準備によって移動のkanban板材にピッキング リストと開始または完了使用して実行されます。 在庫されている材料に対して、プロセスも、WMSピッキングおよび出荷プロセスをサポートできます。 船荷証券はオンデマンド式で作成できます。
4.  kanban処理の単位とkanbanカードを生成します。 処理した後で、カードは外注業者から戻ります。 通常、カードは出荷済の物理的な材料を指定する出荷メモが含まれます。 指定されたサービスへの参照は必要ではありません。 顧客の材料または製品の着荷はkanbanカードによってkanban板材に、登録されます。  (プロセス活動のkanban板材またはジョブ転送のkanban委員会、模擬の活動に応じて、使用されます)。
5.  入庫の状況レポートを作成します。 入庫の状況レポートを受け取るサービスの梱包明細ドキュメントを交換で使用できます。 入庫の状況レポートは、選択した期間の使用の活動の完了kanbanジョブに基づいて生成できます。 各ジョブ入庫に対して、状況レポートは、関連する購買注文明細行に対して作成されます。 入庫の状況レポートは、入庫の確認として外注業者に印刷され、送信できます。
6.  請求書を生成します。

プロセスは外注業者が期間の請求時に終了します。 請求書の照合が作成されている入庫の状況レポートに対して実行されます。 入庫の状況レポートで材料の正確な現物入庫を表すため、三方照合は、簡単です。

## <a name="service-quantity-calculation"></a>サービス数量計算
全体の購買プロセスは、サービスの品目参照に基づきます。 サービスの測定単位でこの品目タイプを指定します。 サービスは通常、サービス (単位) の数での時間に使用されます。 kanbanジョブに登録された完了に基づいてサービス数量を計算するには、次の方法を使用できます:

-   **ジョブの数** – 1のkanbanジョブに基づいて計算は、指定された製品の数量に関係なく、サービスの*n*の単位に、同じです。 細い製造で、一つのジョブは1処理の単位に対応します。 この計算方法は単位の処理1合ごとに固定価格のすべてのサービスに適用されます。 したがって、この方法は通常除隊処理中に適用されます。 ただし、全体処理の単位を処理するプロセス活動に適用できます。
-   **製品の数量** –サービス数量に基づいて計算は、スケジュールされたり/指定された製品の数量に関係しています。 支給された製品の数量が計算されると、エラー数量が含まれるか、または除外できます。 この計算方法で処理された製品のサービス単価が承認されているすべての場合に適用されます。
-   **活動の期間** –に基づいて計算で処理された製品の活動、合計処理済、スループット特定の処理時間に基づいて計算上活動の場合、計算されます。 この計算方法であれば、時間支払われ、処理された製品ごとの時間に変動を含むサービスに適用されます。

## <a name="cost-accounting-of-subcontracted-services"></a>外注されたサービスの原価計算

原価計算期間の終わりに細い製造のには原価の原価計算期間中の生産フローの生産される製品の実際の差異が計算されます。

## <a name="modeling-transfers-as-subcontracted-activities"></a>使用される活動として転送のシミュレーション
ユーザーは、に配送を非生産的考慮され、値を追加すると考えます。 ただし、外注業務の原価が内部生産の原価と比較すると、追加配送活動の原価を考慮する必要があります。 複数の場所にと、配送サービスを必要とする生産の流れ、顧客に製品を供給する原価の一部として運送費をシミュレーションする必要があります。 

細い製造の活動ベースの使用は、配送業者を統合し、生産フローの場所の間で材料と製品を移動する仕入先を含むことができます。 除隊処理中のシミュレーションによって、配送業者または仕入先を割り当てることができます。 除隊処理中またはジョブは、サービスや購買契約に基づいて発注書を作成し、実際のジョブ転送に基づいて状況レポートは、の入力を発行することもできます。 この機能は、外注されたプロセス活動の機能と同じです。  

したがって、Dynamics 365 for Operationsでは、生産フローの原価計算に関連する発注書の配送サービス、作成、配送サービスの原価に統合される入庫登録および統合を含むBOM計算をサポートします。
