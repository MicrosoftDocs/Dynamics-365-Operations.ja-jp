# <a name="backflush-costing"></a>一括引き落とし原価計算

[!include[banner](../includes/banner.md)]


このトピックでは、リーン生産に使用される一括引き落とし原価計算の概念について説明します。 

リーン生産の原価設定では、一括引き落とし原価計算として知られる原価集計方式を生産フローで使用します。 一括引き落とし原価計算の方法では、消費される直接材料は、生産フローの仕掛品 (WIP) の原価勘定に蓄積されます。 標準原価在庫モデル グループが使用されます。 生産フローから受信する製品は、標準原価で仕掛品から差し引かれます。 一括引き落とし原価計算と標準原価の主な違いは、一括引き落とし原価計算の場合、かんばんまたは完成品ごとに差異を計算しないことです。 代わりに、差異は、一定期間にわたり生産フローあたりで計算されます。 この方法は、材料消費の報告のための真のリーン コンセプトを紹介します。 材料の専用のピッキング済数量は、かんばんまたは製造オーダーにレポートされません。 代わりに、完全なバッチまたは材料取り扱い単位が、生産フローにステージされます。 バッチまたは材料取り扱い単位が空として登録されると、これらは消費されたと宣言されます。 [生産フローの構成](http://ax.help.dynamics.com/en/wiki/lean-manufacturing-modeling-the-lean-organization/) によっては、詳細な消費を使用する場合があります。 詳細な消費を使用する前に、組織は、材料が生産フローの仕掛品で消えることを自分でできるようにする必要があります。 定期的一括引き落とし原価計算により、期末における仕掛品の有効値が決定できます。 この決定は、かんばん材料取り扱い単位とかんばん作業状態に基づいています。 有効値と原価グループおよび品目ごとの実際の仕掛品値の誤差は転記され、差異として示されます。

## <a name="configuring-backflush-costing"></a>一括引き落とし原価計算の設定
原価計算を有効にするには、次の手順を完了する必要があります。

-   **生産グループおよび生産フローの仕掛品勘定を設定します。** 生産フローの仕掛品勘定は、生産グループで指定します。 生産フローの一括引き落とし原価計算は、各生産フローの一括引き落とし原価計算を実行する前後の仕掛品値の差を、差異として計算します。 したがって、すべての生産フローで仕掛品勘定を作成することをお勧めします。
-   **リソース グループに原価カテゴリを割り当てます。** 作業セルの実行時間カテゴリには、原価カテゴリを割り当てる必要があります。 活動で差異を決定するために、すべての作業セルに原価カテゴリを作成する必要があります。 設定および数量の原価カテゴリは、リーン生産の原価計算とは見なされません。 一括引き落とし原価計算では、リソース グループごとの仕掛品勘定が無視されます。 外注活動には、原価カテゴリは必要ありません。 有効なサービスに割り当てられている原価グループが、代わりに使用されます。
-   **原価グループの割り当て。** 生産フローの原価貢献度の区分を有効にするには、原価グループ タイプごとに原価グループを割り当てる必要があります。
    -   **直接材料の原価グループ** - 直接材料には、原価計算のために材料のカテゴリを識別する原価グループが必要です。 この原価グループは、直接材料ごとの原価、仕掛品、差異の総合的なビューを有効にします。
    -   **直接製造原価グループ** - 直接製造原価グループは、生産フローに対する業務資源の直接原価貢献度を取得します。 この原価グループは、直接製造原価ごとの原価、仕掛品、差異の総合的なビューを有効にします。
    -   **間接原価グループ** - 間接原価グループは、生産フローに対する間接費の貢献度を計算するために必要です。 この原価グループは、間接原価ごとの原価、仕掛品、差異の総合的なビューを有効にします。
    -   **直接外部委託のコスト グループ** - サービスの原価グループは、割り当てられた原価および仕掛品の総合的なビューを有効にし、外注サービスの原価の差異を決定します。
    -   **完成品の原価グループ** - 完成品は、原価計算のための製品カテゴリを識別する原価グループを必要とします。 この原価グループは、製品カテゴリごとの原価、仕掛品、差異の総合的なビューを有効にします。 製品の標準原価は、BOM 品目 (BOM) と、生産フローおよびかんばんルールまたは工順、に基づいた原価計算を使用して計算されます。

### <a name="costing-sheet"></a>原価計算表

原価計算表は、会社の原価構造をモデル化し、原価を分類する原価グループ別に構築されます。 原価計算表には、さまざまな形式があります。 設計された構造に従って、原価情報が表示されます。 原価計算の表に、間接原価の計算に使用する式を定義することもできます。 計算式は、数量、重量、量、または値に基づくことができます。

-   **原価バージョンを定義する。** 原価バージョンで、会社は原価の管理方法を定義します。 原価バージョンには、割り当てられた原価タイプに基づいて、一連の標準原価レコードまたは予定原価レコードを格納することができます。 リーン生産の原価計算のために使用する原価バージョンは、標準原価に基づいている必要があります。
-   **リリース済製品に対して在庫モデル グループを割り当てます。** 生産フローに関連するすべての製品は、**標準原価**在庫モデル グループを使用する在庫モデル グループに割り当てる必要があります。 標準原価は、サイトおよび有効化日付ごとに管理されます。 バリアントまたは製品マスターごとに原価を管理している場合、製品マスターには在庫モデル グループを選択できます。
-   **定義では、外注サービスは非在庫のサービスです。** 外注活動には在庫モデル グループがありません。 外注活動を正しく原価計算するには、サービス活動が属している在庫モデル グループで在庫ポリシーが**在庫製品 = いいえ**と設定されていることを確認してください。

出荷製品の場合、生産フローに基づいている原価の計算では、外注活動に関連付けられているサービスに対して、標準原価を管理する必要があります。 サービスに割り当てられている原価グループは、外注活動の原価の差異を決定するために使用されます。

## <a name="cost-calculation-for-lean-manufacturing"></a>リーン生産の原価の計算
生産フローから提供される製品の場合、BOM 計算は工順バージョンまたは生産フローのいずれかに基づいている必要があります。 BOM 計算では、製品および関連するリソースと製品を構築するために必要な材料の内訳の原価を計算します。 生産フローの仕掛品勘定からの控除には、品目および原価グループ別の製品の内訳を使用します。

### <a name="calculation-that-is-based-on-the-production-flow"></a>生産フローに基づく計算

Microsoft Dynamics 365 for Finance and Operations のリーン生産は、工順に依存しません。 生産フローから提供される製品の原価計算は、生産フロー自体に基づいて行うことができます。 計算を行う前に、生産フローから製品を支給するかんばんルールを作成する必要があります。 製品が計算日に同じサイトの複数の生産フローから供給される場合、BOM 計算のための生産フローを選択できます。 **既定生産フロー**ページで、各品目の既定生産フローを構成することができます。 計算日に有効になる同じ生産フローに同一製品の複数のかんばんルールがある場合、計算に対して有効な最初のかんばんルールを使用して計算します。

### <a name="calculation-that-is-based-on-the-route"></a>工順に基づく計算

工順を基準とする計算は、生産フローに基づく計算と同様に有効です。 ただし、工順を基準とする計算では、リーン生産機能の原価設定を使用しません。 工順は、リソース グループのリソース要件を使用する必要があります。 体系的な差異を回避するために、同一の作業セル、または少なくとも同じ原価カテゴリを使用する必要があります。 設定および数量には原価カテゴリを避ける必要があります。 それらは、リーン生産一括引き落とし原価計算よりも詳細な内訳で原価計算を行う助けにはなりません。 どちらのオプション (生産フローまたは工順) を原価計算に使用する必要があるかを判断するには、原価内訳の結果を検討してください。 現実に近く、少ない差異を生成するバージョンが、全体としてより良いオプションです。 製品が 1 つの生産フローおよび 1 つのかんばんルールによって供給されるリーン生産環境では、生産フローに基づく計算がより正確な可能性があります。 同じサイトでリーン生産および製造オーダーによって製品が供給される場合、または複数の生産フローあるいは同じフローで複数のかんばんルールがある場合、生産のためではなく原価計算用に組み込まれている工順バージョンに基づく計算の方がより正確な可能性があります。 外注が関連する製品の計算には、生産フローの計算を使用する必要があります。 Microsoft Dynamics 365 for Finance and Operations では、製造オーダーによる外注およびリーン生産での外注の原価モデルでは、2 つの異なるアプローチを使用しています。 リーン生産方式では新しい原価グループ タイプ**直接外部委託**を導入して、外注サービスを計算します。

## <a name="material-consumption"></a>材料消費量
材料が在庫から仕掛品に使用されると、材料の原価は原価グループの実際の標準原価で仕掛品に追加されます。 この操作は、次の条件下で発生します。

-   かんばん払出は、在庫を更新するかんばんピッキング ラインに転記されます。
-   ピッキング時に在庫が更新されまだレシートのない転送ジョブ (材料の在庫から仕掛品への転送) が完了します。

## <a name="receiving-products-from-the-production-flow"></a>生産フローからの製品の入荷
次の条件下で、製品を生産フローから受け取ります。

-   **入庫時の在庫更新**が**はい**に設定されているプロセス ジョブが完了した場合。
-   **ピック セットに基づく更新在庫**が**いいえ** (仕掛品から在庫への転送) に設定されている、入庫時に在庫を更新する転送ジョブが完了した場合。 このオプションにより、BOM および工順コンフィギュレーションとは無関係に、生産フローから任意の製品を受け取ることができます。 プロセスは、現物フローに従います。 このオプションは、特に副産物、連産品、または生産フローからの仕損を受け取り、それに応じて仕掛品の生産フローの原価残高を修正するのに適しています。

生産フローから受け取る製品は、仕掛品から差し引かれます。

## <a name="products-in-wip"></a>仕掛品の製品
Microsoft Dynamics 365 for Finance and Operations のリーン生産の仕掛品モデルでは、かんばんの材料取り扱い単位状態を使用して、材料、半製品、および仕掛品の一部である完成品を管理することができます。

-   **割り当て済** - かんばんは、仕掛品に転記済の消費された材料を持つことができます。
-   **受取済** - かんばんが、**入庫時の在庫更新**が**いいえ**に設定された最終活動を参照している場合、在庫に登録されていない製品または半製品の完全な材料取り扱い単位を表しています。

仕掛品の材料は、手持在庫の概要に表示されないことに注意してください。 ただし、かんばん数量の概要には表示されます。

## <a name="consuming-products-in-wip"></a>仕掛品の製品の消費
仕掛品の製品は、対応するかんばん材料取り扱い単位が空になったときに消費されます。 かんばん空シグナルは、有効な原価計算トランザクションを生成しませんが、次の一括引き落とし原価計算の実行時に有効になります。 空のかんばん材料取り扱い単位は手持在庫とはみなされず、したがって期間内で消費されたと計算されます。

### <a name="automatic-empty-registration"></a>空の自動登録

スケジュールされたまたはイベントのかんばんを、かんばんルールに空の自動登録として設定することができます。

-   **材料取り扱い単位の受領時** - スケジュール済かんばんの既定では、かんばんの最後のジョブの完了時に材料の消費が宣言されます。 固定数量かんばんの場合、このオプションは単一箱かんばんに対してのみお勧めします。最終宛先がかんばんを受け取ると、カードが要求のソースに返却されるためです。
-   **ソース要件の登録時** - このオプションはイベントかんばんにのみ利用でき、それらの既定のオプションです。 仕掛品との関係では、このオプションは仕掛品を何もない状態に保つのに便利です。親かんばんを準備した場合、仕掛品のコンポーネントのかんばんが自動的に空にされ、仕掛品から消費されるためです。

まとめると、かんばんの材料取り扱い単位は、割り当て済み (= プロセス)、受け取り済 (= 完全)、または空にできます。 部分的な空はありません。 そのため、消費の正確な登録を有効にするには、かんばんの製品の数量を、期間ごとの消費量よりも少なく制限することが重要です。 数日や数週間の需要の大きなバッチで作業現場に移動された製品は、仕掛品で消費しないようにする必要があります。 代わりに、これらの製品は在庫に保管する必要があります。

## <a name="backflush-costing"></a>一括引き落とし原価計算
定期的に一括引き落とし原価計算を実行して仕掛品の値を算出し、製品材料、労務、および間接原価の差異を計算する期末ステータスを生成する必要があります。 計算された差異は、差額勘定に転記されます。 一括引き落とし原価計算では、法人のすべての生産フローは同じバッチの実行で使用されます。 一括引き落とし原価計算がバッチで実行されると、生産フローによって処理がマルチ スレッドになる可能性があります。 一括引き落とし期間は、終了日によって定義されます。 一括引き落とし原価計算が実行された日付に、新しいトランザクションを転記することはできません。 一日が実際に終了する前に、今日の日付で一括引き落とし原価計算を実行しないでください。 一括引き落とし原価計算は、次の手順を実行します。

1.  期間の終了日の時点で、生産フローで使用されていない数量を決定します。 一括引き落とし原価計算の実行後、**未使用数量**ダイアログ ボックスで、原価計算の実行日付で未使用の数量を表示することができます。
2.  生産フローの期間中の正味実現消費を計算します。
3.  実現リソース消費量および製品の仕掛品をクリアします。
4.  期間の標準原価に対する製造差異を計算します。 **期間の消費コンポーネント。**
    -   期間中に生産フローが消費する材料の正味実現数量を、財務上更新します。 システムは、期間の正味実現数量に達するまで、先入れ先出し (FIFO) の順序で個々の在庫トランザクションを、生産フローの現物更新済トランザクションとして財務上更新します。
    -   トランザクションは正確な消費量にマップするよう財務上分割されます。
    -   生産フロー仕掛品で使用されていない数量は、現物更新済のステータスのままです。

    **期間の生産完了数量のために。**
    -   期間の生産完了数量のために、在庫トランザクションを財務上更新します。

    **換算原価のために。**
    -   期間に対して記録され適用されている換算原価トランザクション (工順トランザクション) が財務上更新されます。
    -   すべての直接製造原価が財務的に更新されます。 期間中に完了したすべてのかんばんプロセス ジョブが蓄積されます。
    -   期間内で消費された材料に対して計算された間接原価が、仕掛品から差し引かれます。 残りの間接原価は、差異として転記されます。

5.  標準原価に対する製造差異を計算します。 差異は、原価グループごとに計算されます。
