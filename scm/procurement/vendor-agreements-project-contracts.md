# <a name="pay-when-paid-vendor-agreements"></a>受給時支払の仕入先契約

[!include[banner](../includes/banner.md)]


この記事は、仕入先の契約に使用する受給時支払 (PWP) の条件について説明します。 PWP 条件では、プロジェクトの顧客が支払うまで、仕入先への支払を一部または全部留保できます。 この記事は、プロジェクトで PWP 条件を使用する方法を説明するために実際の例を示します。

仕入先を協力会社としてプロジェクトの作業を承認するとき、プロジェクトの顧客が支払うまで仕入先や協力会社への支払を留保したい場合があります。 仕入先への支払を留保するには、仕入先への発注書を設定するときに受給時支払 (PWP) 条件を設定します。 仕入先の発注書を作成し、その発注書にプロジェクト ID を割り当てると、プロジェクトの PWP 条件が発注書と仕入先に関連付けられます。 **受給時支払のある仕入先請求書**ページで、発注書および関連する顧客請求書の未払い仕入先請求書の一覧を表示できます。 このフォームを使用して、仕入先に支払うするかどうかと支払額を決定します。 たとえば、顧客がプロジェクト請求書の金額を支払うと、支払に関連する仕入先請求書の一部またはすべてをリリースできます。 顧客から関連する支払の特定の割合を受け取ってから、仕入先への支払を指定する PWP 条件を設定できます。 顧客から一部支払を受け取ると、支払に関連する仕入先請求書の一部を手動でリリースできます。 仕入先または協力会社への支払を、顧客が支払うまで留保する PWP 条件の使用方法を、次の例で表示します。 顧客が要求するソフトウェアをインストールする 100 台のコンピューターを提供するための契約を顧客と結びます。 顧客との間で合意する価格は、コンピューターの 1 台あたり 150 ドルです。 コンピュータを用意するために、サード パーティの仕入先のサービスを手配します。 顧客は、コンピュータの品質を承認してから支払を行うことを要求しています。 組織のポリシーは、プロジェクトの顧客によって支払われるまでサード パーティの仕入先への支払を留保することです。 したがって、100% の PWP の割合でプロジェクトを設定し、顧客請求書に対して全額の支払を受け取るまで、仕入先へのすべての支払を留保します。 仕入先はコンピューター 1 台当り 100 ドルを請求します。 仕入先がコンピューターを送ってくるとき、100 台のコンピューターに対する 10,000.00 ドルの請求書も送られてきます。 PWP 条件でプロジェクトを設定しているので、仕入先には支払いません。 仕入先からコンピュータを受け取った後、コンピュータに要求されているソフトウェアをインストールします。 顧客にコンピューターを送るとき、15,000円の請求書も顧客に送ります。 顧客はコンピューターを検査して製品の品質を承認します。 その後、顧客はプロジェクト請求書の全額を支払います。 顧客から全額の支払を受け取った後、仕入先に仕入先請求書の全額の 10,000.00 ドルを支払います。
