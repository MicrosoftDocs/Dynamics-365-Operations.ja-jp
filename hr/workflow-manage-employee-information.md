# <a name="use-workflows-to-manage-employee-information"></a>従業員情報を管理する、現在の作業を使用します。

このトピックでは、従業員情報を管理する方法、人事管理のためにワークフロー機能を使用する方法を説明します。 たとえば、職位に作業の現在の関連付け、従業員が自分のレコードを変更するときに開始する承認の現在の作業を設定できます。

人事管理のワークフローの管理機能は、人事管理の活動は数多くの現在の作業を提供します。 また、多くのオプションは、特定の作業の流れを変更し、レポート階層に関連付けることができるようになります。 作業は、現在の従業員情報の複数の標準タイプの変更の管理を支援できます。 職位に現在の作業を関連付けることができます。 その従業員は、従業員レコードを変更した場合、承認を要求する新しい情報が格納される前に現在の作業が開始されます。 現在の作業は次の種類の情報が変更の管理、従業員のデータを正確に維持を効率的に行えるように定義されます:

-   ID 番号
-   コース
-   教育
-   画像
-   貸与品目
-   職務経験
-   プロジェクト経験
-   スキル
-   要職
-   人事管理のアクション
-   コース登録

従業員の雇用、転記された場合、または完了したら、現在の作業が検証プロセスを含めることができます。 これにより、ドキュメントは変更できますまたは工程の条件として、現在の一部として定義できます。 確認プロセスが完了すると、ドキュメントまたは工程が完了し、作業の現在は、最終承認ステップに移動します。

## <a name="associate-a-workflow-with-a-position-hierarchy"></a>職位階層と現在の作業を関連付けます。
[設定した階層と現在の作業を関連付けることができます。 たとえば、職位がマトリックス レポート階層に関連付けられれば、その職位に関連付けられる従業員のマネージャではなく、プロジェクトの潜在顧客に特定のプロジェクトの経費の工順を指定する作業の流れを設定するかもしがあります。 新しい作業の流れを作成するか、または既存のワークフローを、**人事管理の作業の現在**ページで変更することは、[**新しい**。 ワークフロー デザイナーを開始するリストの作業の流れを選択します。 新しい作業の流れを作成するか、または既存のワークフローのステップを変更するのにデザイナーを使用できます。 既存のワークフローを変更すると、変更が新しいバージョンとして保存されます。 この場合、以前のバージョン、常に戻ることができます。
