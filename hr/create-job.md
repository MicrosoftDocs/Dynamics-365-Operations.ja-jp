# <a name="setting-up-the-components-of-a-job"></a>ジョブのコンポーネントの設定

このトピックでは、組織内にある要素をどのように使用できる含まれるジョブができるついて、例を示します概念的な要素を。 

ジョブを作成する前に参照情報を設定する必要があります。 名前だけであるジョブを作成できます。 ただしは、職位など) を含めることによって、ジョブに割り当てられた職位の既定値になります。 また、入力する情報には、特定のジョブに対する報酬プランをフィルタ処理で使用できます。 特定のジョブに対する報酬プランをフィルタ処理で使用できる資格を設定する場合は、ジョブを設定する前に、ジョブ機能とジョブ タイプを設定する必要があります。 使用できるこれらの既定値を作成することによってジョブに位置を追加する時間を節約できます。 

ジョブの詳細、職位などのタイプと機能、有効日です。 ジョブを作成し、今日が、後で別のこれらの詳細を追加したり、および作成日に、職務を表示し、これらの詳細も表示されません。 したがって、必要とする前に、この参照情報の一部を作成する必要があります。 これらの作成時にその方法、新しい職務に情報を追加できます。

## <a name="job-titles"></a>職位
ジョブを作成する前に、それらの職位を設定する必要があります。 職位は、その職位が関連付けられているジョブの職位を継承します。 

[検索機能を使用して開くことができる**タイトル**ページを使用して職位を管理します。 **タイトル**ページに、ジョブに使用する計画する名前を入力します。

## <a name="job-types"></a>ジョブ タイプ
カテゴリに同様のジョブをグループ化する、ジョブ タイプを使用します。 ジョブ タイプは必要ではありません。 ただし、報酬管理の適格性ルールを設定する際にジョブ タイプを使用する予定の場合は、ジョブを設定する前にジョブ タイプを設定する必要があります。 ジョブ タイプの例は、フルタイム、パートタイムですが、時間あたりの支払給料の支払します。 **ジョブ タイプ**ページを使用してジョブ タイプを管理します。 **ジョブ タイプ**ページで、ジョブ タイプの名前と簡単な説明を入力します。 **では除外ステータス**次のオプションを一つのフィールドに、このジョブ タイプである職務の公正労働基準の操作(FLSA)の除外ステータスを指定します:

-   **免税** –ジョブはFLSAで残業から控除されます。
-   **、免税** –ジョブはFLSAで残業から控除されません。
-   **適用されません– ** FLSAの補充は適用されません。

## <a name="job-functions"></a>ジョブ機能
ジョブ参照合点は、高レベルの機能カテゴリについて、高レベルの職務に関連付けます。 ジョブ機能は必要ではありません。 特定のジョブに対する報酬プランをフィルタ処理して、ジョブ機能とジョブ タイプと組み合わせて使用できます。 報酬プランに**適格性ルール**ページの適格性ルールを設定して、ジョブ機能とジョブ タイプを関連付けます。 報酬プランにその後、適格性ルールで定義したジョブ機能とジョブ タイプの特定の組み合わせに適用される一連のレベルを関連付けることができます。  (これらの機能は、固定報酬プランと変動報酬プランの両方に適用されます。) ただし、報酬管理の適格性ルールを設定すると、ジョブ機能を使用する予定の場合、ジョブを設定する前に、ジョブ機能を設定する必要があります。 次の表に、ジョブ機能の例を示します。

**ジョブ機能**ページを使用してジョブ機能を管理します。 **ジョブ機能**ページで、ジョブ機能のIDコードと簡単な説明を入力します。
