# <a name="manage-the-electronic-reporting-configuration-lifecycle"></a>電子申告コンフィギュレーション ライフサイクルの管理

このトピックでは、ソリューションのMicrosoft Dynamics 365の(ER)電子申告のコンフィギュレーションのライフサイクルを管理する方法について説明します。

<a name="overview"></a>概要
--------

電子申告 (ER) は、Microsoft Dynamics 365 for Operations の法律の要件および国固有の電子ドキュメントをサポートするエンジンです。 一般に、ER は、各電子ドキュメントに関して次のタスクの実行を想定しています。 詳細については、" "を参照してください ([一般的な電子reporting.md) 電子レポートの概要]。

-   電子ドキュメントのテンプレートを設計します:
    -   ドキュメントに表示できるデータのソースを識別します:
        -   データ テーブルは、データ エンティティとクラスなど、工程データの基になる365 for Operations。
        -   プロセス固有のプロパティ、実行日時など) とタイム ゾーン。
        -   実行時間、エンドユーザーを指定するユーザー入力パラメータ。
    -   必要なドキュメントの要素、および最終ドキュメントの形式を指定するトポロジを定義します。
    -   データソース経由でドキュメントの形式コンポーネントにバインドされ、定義されたドキュメント要素に選択したデータ ソースから要求されるデータ フローのコンフィギュレーションを行い、プロセスの制御ロジックを指定します。
-   他の Dynamics 365 for Operations インスタンスでテンプレートが使用できるようにします:
    -   Dynamics 365 for Operations で作成したドキュメント テンプレートを ER のコンフィギュレーションに変換し、現在の Dynamics 365 for Operations インスタンスからコンフィギュレーションを XML パッケージとしてエクスポートし、ローカルまたは LCS に保存します。
    -   ER コンフィギュレーションを Dynamics 365 for Operations ドキュメント テンプレートに変換します。
    -   ローカルまたは LCS に格納されている XML パッケージを、現在の Dynamics 365 for Operations インスタンスにインポートします。
-   電子ドキュメント テンプレートをカスタマイズします:
    -   LCS からテンプレートを ER コンフィギュレーションとして現在の Dynamics 365 for Operations インスタンスに取り込みます。
    -   基本バージョンを参照しながら、カスタム バージョンの ER コンフィギュレーションを設計します。
-   Dynamics 365 for Operations で使用できるように、特定の業務プロセスを含むテンプレートを連結します:
    -   Dynamics 365 for Operations の設定をコンフィギュレーションし、プロセスに関連したパラメーターでそのコンフィギュレーションを参照して、ER を使用開始できるようにします。 たとえば、請求書を処理するための電子支払メッセージを生成する特定の買掛金勘定の支払方法のERコンフィギュレーションを参照してください。
-   特定の業務プロセスでテンプレートを使用します:
    -   特定の業務プロセス (ERコンフィギュレーションを実行します。 たとえば、ERコンフィギュレーションを参照する支払方法が選択されると、処理用の電子支払メッセージを生成するために請求書。

## <a name="concepts"></a>概念
次のロールおよび関連する活動は、ERコンフィギュレーション ライフサイクルに関連付けられます。

| 役割                                       | 活動                                                      | 説明                                                                                                                                                                                                                  |
|--------------------------------------------|-----------------------------------------------------------------|------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| 電子申告機能コンサルタント | ER コンポーネントを作成および管理します (モデル、形式)。           | ER domain–specificのデータ モデルを作成して、電子ドキュメントに必要なテンプレートを作成し、それに応じて結合される業務ユーザー。                                                                           |
| 電子申告開発者             | データ モデルのマップを作成および管理します。                          | アクションのデータ ソースは必須、Dynamics 365 for Operationsを選択し、ER domain–specificのデータ モデルに連結する工程の担当者365 for Operations。                                                                 |
| 会計監修者                      | ER のコンポーネントを参照するプロセス関連の設定を構成します。 | たとえば、生成する、ERコンフィギュレーションの設定が特定の勘定の支払を支払方法で使用されるようにする**会計**管理者ロールは、処理用の電子支払の請求書。 |
| 買掛金勘定支払係            | 特定の業務プロセスで ER コンポーネントを使用します。                | たとえば、電子支払メッセージが請求書を処理するために生成されるようにする特定の支払方法に対して設定されたERのフォームに基づいて**買掛金勘定の支払が記録をとります**ロール。           |

## <a name="er-configuration-development-lifecycle"></a>ER コンフィギュレーションの開発ライフサイクル
ER に関連する次の理由から、個別の Dynamics 365 for Operations インスタンスとしての開発環境で、ER コンフィギュレーションを作成することをお勧めします:

-   **電子申告開発者**ロールまたは**電子申告機能コンサルタント**ロールのユーザーは、テスト目的でコンフィギュレーションを編集し、実行できます。 このシナリオでは、Dynamics 365 for Operations インスタンスの業務データと業績について有害なおそれがあるクラスおよびテーブル メソッドが呼び出されることがあります。
-   ER コンフィギュレーションの ER データ ソースとしてのクラスおよびテーブル メソッドの使用は Dynamics 365 for Operations エントリ ポイントおよび記録された会社の内容によって制限されません。 **電子申告開発者**ロールまたは**電子申告機能コンサルタント**ロールのユーザーは、機密情報にアクセスできます。

開発環境で設計されたERコンフィギュレーションは、評価 (適切なプロセス連結、正確性とその結果のパフォーマンス) とロール主導型のアクセス権の正確性と職務の分離など、品質保証のテスト環境に、アップロードできます。 ERコンフィギュレーション交換を可能にする機能はこの目的に使用できます。 最後に、認定ERコンフィギュレーションは、サービス サブスクライバーと共有できるまたは次の図に表示されるなど、内部で使用するための稼働環境にアップロードされて、いずれかになりますLCSにする。 ![ライフサイクルERコンフィギュレーション](./media/ger-configuration-lifecycle.png)
