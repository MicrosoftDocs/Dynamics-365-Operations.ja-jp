# <a name="dynamics-365-for-operations-glossary"></a>工程の用語集365 for Operations

Microsoft Dynamics 365 for Operations の用語集は、Dynamics 365 for Operations の製品の重要な用語および概念を定義します。 

### <a name="a"></a>**A**

###### <a name="accounting-journal"></a>**計算仕訳帳**

会計システムにおける会計イベントの財務結果を記録する仕訳。

###### <a name="accounting-rule"></a>**会計ルール**

会計イベントの財務結果を分類、記録、報告するときの原則、方法、および手順を管理する会計システムのルール。

###### <a name="ach-automated-clearing-house"></a>**ACH (自動決済機関)**

米国で利用できる電子決済システム。受領者と送金者の銀行口座間で効率的に資金の振り替えができます。

###### <a name="actual-quantity"></a>**実数量**

活動に対する入出力の測定数量。

###### <a name="address-verification-service"></a>**住所検証サービス**

カード所有者が提出した請求先住所が発行銀行のファイルの情報と一致するかどうかを確認するクレジット カード プロセッサによるサービス。

###### <a name="adjusted-unit-price"></a>**調整済単価**

属性ベースの価格に基づいて計算された単位価格、実際の受領ポテンシー。

###### <a name="advanced-pricing"></a>**詳細な価格決定**

ロジックがポテンシー品目の購買価格の計算に使用されます。

###### <a name="aging"></a>**エイジング**

期日経過日数別の期間分類プロセス。

###### <a name="aging-period"></a>**エイジング期間**

顧客支払の延滞残高の報告に使用する期間日数。

### <a name="b"></a>**B**

###### <a name="balance-sheet"></a>**貸借対照表**

特定の日付の資産、負債、および株主資本の状態を報告して組織の財務状態を示す明細書。

###### <a name="balance-sheet-account"></a>**貸借対照表勘定**

貸借対照表で報告される値の変化を示す勘定

###### <a name="bank-reconciliation"></a>**口座調整**

勘定科目項目を銀行口座明細書項目と照合して、銀行口座を表す勘定科目を調整するプラクティス。

###### <a name="batch-attribute"></a>**バッチ属性 **

一括処理または転送バッチの製品属性。

###### <a name="best-before-date"></a>**出庫期限**

製品の最適な品質または最適な特性が得られる推奨日。

###### <a name="best-before-period"></a>**賞味期間**

製品の最適な品質または最適な特性が得られる期間。

###### <a name="bill-of-exchange"></a>**受取手形**

要求に応じてセカンド パーティに対して支払いをサードパーティに求める無条件の要求を記載した元伝票。

###### <a name="bill-of-materials"></a>**部品表**

一つの製品を生産するために必要な製品の一覧とその数量。

###### <a name="blocking"></a>**ブロック**

文書や製品を保留にするアクション。

###### <a name="budget-control"></a>**予算管理**

将来の支払確約に対応する予算財源を確保できる場合にのみ経費を承認するプラクティス。

###### <a name="budget-control-dimension"></a>**予算管理分析コード**

予定の活動の支払に予算財源を割り当てるための有効な財務分析コード値の組み合わせ。

###### <a name="budget-control-rule"></a>**予算管理ルール**

ビジネス意志決定のエンコード。有効な予算管理分析コード値の組み合わせで定義された詳細な活動または集約活動に割り当てられた利用可能な予算財源に対して、委託された経費や実際の経費を照合します。

###### <a name="budget-cycle-time-span"></a>**予算サイクル期間**

会計年度期間の数で指定された期間。 予算期間は会計期間と異なることがあります。

###### <a name="budget-group"></a>**予算グループ**

予算分析コード階層の財務分析コード値のセット。下位の財務分析コード値に割り当てられた予算財源を集計して、上位財務分析コード値に割り当てられた総予算財源を計算するときに使用します。

###### <a name="budget-model"></a>**予算モデル**

予算財源の配賦と経費の計画に使用する計画構造。

###### <a name="bulk-item"></a>**バルク品目**

製品の納品活動に対する式品目の入力。

###### <a name="bundle"></a>**バンドル**

1 単位として販売する多数の製品の組み合わせ。

###### <a name="business-unit"></a>**事業単位**

戦略的なビジネス目標達成のために作成された半自治の事業単位。

### <a name="c"></a>**貸方**

###### <a name="capacity-load"></a>**最大能力負荷**

ワーク センターが必要な能力レベルで実行できる予定作業の最大量。

###### <a name="capacity-planning"></a>**能力計画**

特定期間内に、将来の出力需要を満たすリソースの能力要件を決定する手順。

###### <a name="carry-forward-budget"></a>**繰越予算**

1 会計年度から次の会計年度に振り替えられ、新しい会計年度では保留の発注書予算として留保される予算。

###### <a name="cash-generating-unit"></a>**キャッシュ生成単位**

組織内の他の資産に無関係の現金を生成する分類可能な資産の最小グループ。 資産のこれらのグループは、発生した減損損失を測定するために使用されます。

###### <a name="catch-weight"></a>**CW**

販売品目または在庫品目の実際の重量または公称重量。

###### <a name="chart-of-accounts"></a>**勘定科目表**

主勘定の一覧。

###### <a name="cogs-cost-of-goods-sold"></a>**COGS (売却済商品の原価)**

製品製造と在庫運搬の財務結果の合計を計算するための勘定科目カテゴリ。

###### <a name="commodity-pricing"></a>**商品価格決定 **

頻繁に価格が変更される商品から生まれた、販売品目価格の更新価格戦略。

###### <a name="consolidated-order"></a>**連結注文**

複数の参照注文を記載した元伝票。

###### <a name="contractor"></a>**契約者**

法人と、契約社員と雇用主の関係を築いた個人の担うロール。

###### <a name="co-product"></a>**連産品 **

別のもう一つの品目と共同で生産される品目。

###### <a name="cost-center"></a>**コスト センター**

予算経費と実績経費の責任をその管理者が負う作業単位。

###### <a name="cost-of-goods-sold"></a>**売却済商品の原価**

製品製造と在庫運搬の財務結果の合計を計算するための勘定科目カテゴリ。

###### <a name="cost-variance"></a>**原価の差異**

予想原価と実際原価との差額。

###### <a name="costing"></a>**原価計算**

組織が取得、生産、または提供する経済資源のコストを計算、割り当て、配分するプロセス。

###### <a name="ctp-capable-to-promise"></a>**CTP (生産可能在庫)**

特定の顧客の要求を満たすために必要な利用可能な運営リソースと利用可能な入力製品からの製品出力の部分。

###### <a name="currency-code"></a>**通貨コード**

通貨単位を表す英数字の識別子。

###### <a name="cycle-time"></a>**サイクル時間**

活動を完了するための所用時間。

### <a name="d"></a>**借方**

###### <a name="dashboard"></a>**ダッシュボード**

Dynamics 365 for Operations の標準的な開始ページ。 ダッシュボードで、ユーザーはアクセスした各ワークスペースのセクションを参照します。 各セクションは、関連するワークスペースで集計セクションのタイルを表示します。 ダッシュボードはタイルの名前とセクションで構成されます。

###### <a name="data-entity"></a>**データ エンティティ**

エンティティのデータ特性を表す情報構造。

###### <a name="delegate"></a>**デリゲート**

委任は、特定のパラメータ リストと戻り値の型のメソッドへの参照を表すタイプです。

###### <a name="delivery-note"></a>**納品書**

2 つの当事者間の製品の納品を記載したビジネス ドキュメント。

###### <a name="demand-forecast"></a>**需要予測**

将来の製品需要の予測。

###### <a name="demand-forecasting"></a>**需要予測**

将来の需要を見積もり、トランザクション データの履歴に基づいて需要予測を作成する業務プロセス。

###### <a name="deployable-package"></a>**配置可能パッケージ**

ユーザー承認テスト (UAT) および実稼働環境の配置に使用する車両。

###### <a name="designer"></a>**デザイナ**

モデルの要素の作成、更新、検査に使用した Visual Studio のツール。

###### <a name="financial-dimensionsdimension-based-configuration"></a>**。** \financial-dimensions\dimension-basedコンフィギュレーション

製品分析コードの値を選択することで、製品バリアントの作成に使用するコンフィギュレーション技術。

###### <a name="distinct-product"></a>**特徴的製品**

一意に識別可能な製品。

###### <a name="document-blocking"></a>**ドキュメント ブロック**

発注した品目を数量確認するプロセスの間、業務プロセスを保留にする方法。

###### <a name="document-management-system"></a>**ドキュメント管理システム**

組織のドキュメントを保存、処理するアプリケーション サービス。

###### <a name="duty"></a>**職務**

セキュリティ モデルにおいて、ユーザーがその職責を果たすために必要な一連のアプリケーションのアクセス権限。

### <a name="e"></a>**E**

###### <a name="eft-electronic-funds-transfer"></a>**EFT (電子決済)**

一つの銀行口座から別のもう一つの銀行口座に資金を振り替えるネットワーク システム。

###### <a name="event"></a>**イベント**

イベントは、何かが発生した場合、クラスやオブジェクトが他のクラスやオブジェクトに通知するようにします。

###### <a name="epe-every-product-every"></a>**EPE (繰り返し生産)**

標準的な繰り返し生産サイクルを確立するためのリーン コンセプト。

###### <a name="extension"></a>**内線**

既存のコードに機能を追加して、アプリケーションをカスタマイズします。

### <a name="f"></a>**F**

###### <a name="financial-dimension"></a>**財務分析コード**

組織の当事者、場所、製品、および活動から作成し、管理報告に使用する財務データの分類子。

###### <a name="financial-dimension-value"></a>**財務分析コード値**

財務分析コードのドメイン内のデータ要素。

###### <a name="financial-statement"></a>**財務諸表**

組織の財務情報と財務状態を記載したレポート。

###### <a name="fixed-cost"></a>**固定費**

製品の納品処理量が変化しても変化しないコスト。

###### <a name="fixed-currency"></a>**固定通貨**

別の通貨に関連して固定為替レートが使用される通貨。

###### <a name="fixed-quantity-kanban"></a>**固定数量かんばん**

かんばんルールに割り当てたかんばん数が一定のときに使用するかんばんのタイプ。

###### <a name="flexible-authentication"></a>**変動認証**

さまざまな識別管理システムのユーザーを認証できる認証プロセス。

###### <a name="formula"></a>**フォーミュラ**

生産プロセスの入力と出力間の数値関係。

###### <a name="formula-item"></a>**式品目**

式で制御されるバッチ処理の出力。

### <a name="g"></a>**G**

###### <a name="general-budget-reservation"></a>**一般予算引当**

これらの資金が他の目的に使用できないようにするため、予算財源を取り分けておくか、指定する公的機関法人によってよく使用されるドキュメント。

###### <a name="grace-period"></a>**支払猶予期間**

ペナルティなしで責務を果たせる指定日以後の経過期間。

###### <a name="gst-goods-and-services-tax"></a>**GST (商品及びサービス税)**

一部の国/地域で徴収される付加価値税。

### <a name="i"></a>**I**

###### <a name="intercompany"></a>**会社間**

配下の全法人の勘定を連結する一つの組織を構成する複数の法人の間で、あるいはその複数の法人に関連して発生と。

###### <a name="invoice-matching"></a>**請求書照合**

仕入先請求書の価格と製品の数量を発注書と製品受領書と照合するプラクティス。

###### <a name="item-allocation-key"></a>**品目配賦キー**

予測と需要スケジューリングに使用する製品ファミリーのグループ。

###### <a name="item-relation"></a>**品目関係**

品目割り当てグループまたはかんばんルールの品目とその製品分析コードに対する参照。

### <a name="k"></a>**K**

###### <a name="kanban"></a>**かんばん**

ある数量の製品の要求を伝える信号。

###### <a name="kanban-flow"></a>**かんばんフロー**

かんばんルールに対して作成され、かんばんに対して実行される活動の順序を定義します。

###### <a name="kanban-job"></a>**かんばん作業**

かんばんがトリガーする生産フローのプロセスや運搬活動。

###### <a name="kanban-job-consumption"></a>**かんばん作業消費**

かんばん生産ジョブを完了するための在庫からの製品コンポーネントの引き出し。

###### <a name="kanban-product-quantity"></a>**かんばん製品数量**

かんばんルールに基づいてかんばんで伝達される品目要件の数。

###### <a name="kanban-rule"></a>**かんばんルール**

プロセス活動と転送活動を生産フローとどのように調和させるかを制御して、材料計画と補充ポリシーを実現する リーン生産 システムのルール。

### <a name="l"></a>**L**

###### <a name="lean-manufacturing"></a>**リーン生産**

製造工程がリーン生産フローおよび営業活動を促進する理念。

###### <a name="lean-schedule-group"></a>**リーン スケジュール グループ**

たとえば、設定グループ、出荷グループ、または輸送グループに基づいて、生産する品目を集計する方法。

###### <a name="ledger-account"></a>**勘定科目**

勘定科目表に掲載された主勘定値とその他財務分析コード値を組み合わせて作成する分類子。この分類子で、経済活動の財務結果を分類します。

###### <a name="license-code"></a>**ライセンス コード**

使用許諾契約に規定された条件に従って、ソフトウェアまたはソフトウェア コンポーネントの使用権限を当事者に与え、ソフトウェア モジュール、ソフトウェアの能力、ソフトウェアの機能を有効、無効にする英数字のキー。

###### <a name="liquidity"></a>**流動性**

流動資産で流動負債を清算する当事者の能力。

### <a name="m"></a>**M**

###### <a name="manufacturing-overhead-based-on-material"></a>**原材料に基づく製造間接費**

製造間接費は、割増率に対する間接原価計算式として表されます。 原材料コンポーネントに割り当てられた原価グループは、計算式の基準として機能します。 計算式は、個々の製品または製品に割り当てられた品目グループに対して定義できるため、原材料に基づく製造間接費の計算を非常に柔軟に実行できます。

###### <a name="master-scheduling"></a>**マスタ スケジューリング**

供給と需要を一致させるためのタイムテーブルを生成するプロセス。

###### <a name="model"></a>**モデル**

モデルには、コード要素および参照メタデータが含まれます。 これは Visual Studio で作成され、確認されます。

###### <a name="model-element"></a>**モデル要素**

モデルは、頒布可能なソフトウェア ソリューションを構成する要素のグループ (ソース ファイルとメタデータ) です。 モデルはデザイン時概念です。

### <a name="n"></a>**無**

###### <a name="nbv-net-book-value"></a>**NBV (正味帳簿価格)**

固定資産の取得原価からその減価償却累計額を差し引いた差額として計算した固定資産の価値。

### <a name="o"></a>**O**

###### <a name="object-permission"></a>**オブジェクトのアクセス許可**

セキュリティ設定が可能なオブジェクトに対して許可された、作成、読み取り、更新、削除、または交換操作。

###### <a name="operating-unit"></a>**作業単位**

職務を備えた従業員に経済資源と業務プロセスの管理を分割して、乏しいリソースを最大限に活用し、プロセスを強化し、その実績の責任を負う組織。

###### <a name="output-product"></a>**製品**

活動の結果得られる物理的製品。

###### <a name="overlayering"></a>**オーバーレイヤ**

既定のソース・コードを上書きする新しいソース・コードを提供してソース・コードをカスタマイズする方法。

### <a name="p"></a>**P**

###### <a name="package"></a>**梱包**

パッケージは、一つ以上のモデルのコンパイル可能および配置可能な単位です。

###### <a name="party"></a>**当事者**

契約上の関係に関わるとき、法的拘束力を持つロールを担うエンティティ。

###### <a name="party"></a>**当事者**

経済活動に関わる個人または組織。

###### <a name="pegging"></a>**ペギング**

必要な品目の数量をそのソースまで追跡するプロセス。

###### <a name="pegging-event"></a>**ペギング イベント**

製品の要求を通知するリソース フロー イベント。

###### <a name="phantom-item"></a>**ファントム品目**

生産しても在庫しない品目。

###### <a name="planned-intercompany-demand"></a>**計画企業内需要**

仕入先ロールを担う法人による製品の予想需要。 予想需要は、下流法人からの製品需要予測から生成します。

###### <a name="post"></a>**転記**

実利的イベントの金銭的価値を特定の勘定に記録すること、または一般仕訳勘定項目と補助仕訳勘定項目を総勘定科目項目と補助勘定科目項目に集計して再分類すること。

###### <a name="preparer"></a>**作成者**

経済資源の要求を開始する元伝票を作成する担当者。

###### <a name="privilege"></a>**権限**

タスクを実行するためにユーザーが必要とする一つ以上のアクセス許可。

###### <a name="procurement-catalog"></a>**調達カタログ**

調達カテゴリ別にグループ化された提供製品の一覧。 調達カタログは、組織による内部使用のために製品を要求するときに使用します。

###### <a name="product-component"></a>**製品コンポーネント**

製品の一部。

###### <a name="product-dimension"></a>**製品分析コード**

使用されるサイズ、色、または製品コンフィギュレーション特質。\financial-dimensions\dimension-basedコンフィギュレーション。

###### <a name="product-family"></a>**製品ファミリ**

品目、サービス、または権限の一意のグループであって、同じ生産活動か納品活動に関わりがあるか、同じ市場セグメントに提供されます。 Dynamics 365 for Operations では、このグループは、予測配賦キーで表します。

###### <a name="product-master"></a>**製品マスター**

製品バリアントのコンフィギュレーションの基礎となる製品の標準または機能表現。

###### <a name="product-receipt"></a>**製品受領書**

発注された製品の受け取り、返品された製品の受け取り、または委託販売で受け取った製品の受け取りを記載した元伝票。

###### <a name="product-variant"></a>**製品バリアント**

製品マスターのコンフィギュレーション。

###### <a name="production-flow"></a>**生産フロー**

リーン原則で設計された生産プロセス。

###### <a name="production-flow-model"></a>**生産フロー モデル**

生産フロー プロセスの作業セル グループが提供する生産能力の表現。

###### <a name="production-order"></a>**製造オーダー**

需要に応じて品目を生産するための要件を記載した元伝票。

###### <a name="production-schedule"></a>**生産スケジュール**

特定の時間と、特定の人的資源と業務資源によって特定の品目と品目数量を生産するスケジュール。

###### <a name="prospect"></a>**見込顧客**

法人にサービスを提供し、将来ある程度の確率で経済的な恩恵をもたらす可能性のある既存の能力または潜在的な能力を備えた参加者。

###### <a name="purchase-agreement-policy"></a>**購買契約ポリシー**

当事者が購買契約書の条項を変更することを認めたポリシー。

###### <a name="purchase-order-policy"></a>**購買注文ポリシー**

当事者が発注書の条項を変更して、発注処理を制御することを認めるポリシー。

###### <a name="purchase-quotation"></a>**購買見積**

調達プロセスの見積依頼に応じて、指定価格で、指定日までに、一定数量の製品を購入するという申し出を記載した元伝票。

###### <a name="purchase-requisition"></a>**購買要求**

検討用に提出され、購入組織による購買承認のために利用される製品要求を記載した元伝票。

### <a name="q"></a>**Q**

###### <a name="quantity-variance"></a>**数量の差異**

新たな受け取り予定がない時点における、発注製品数量と受領製品数量間の差。

### <a name="r"></a>**R**

###### <a name="receipt-advice"></a>**受領通知**

購入者が仕入先から受け取る製品の概要を記載したビジネス ドキュメント。

###### <a name="reconciliation"></a>**調整**

数値を合わせるために複数の勘定や明細書を調整するプラクティス。

###### <a name="reduction-key"></a>**下方修正キー**

マスター プランの予測要求を増やしたり減らしたりするために使用される方法。特定の期間に適用されるユーザー定義の割合に基づいています。

###### <a name="register"></a>**レジスター**

会計システムのリソース フロー イベントの運用結果、法的責任、財務結果を記録するためのレコード。

###### <a name="regulated-product"></a>**規制製品**

政府当局によって規制された製品。コンプライアンスと監査報告が必要である。

###### <a name="rfq-request-for-quotation"></a>**RFQ (見積依頼)**

指定価格の製品を一定数量、指定日までに供給するという入札の案内を記載した元伝票。

###### <a name="requester"></a>**依頼者**

経済資源を要求する個人。

###### <a name="restricted-product"></a>**制限製品**

販売が政府当局によって制限されている製品。

### <a name="s"></a>**S**

###### <a name="sales-agreement"></a>**販売契約書**

魅力的な価格設定と割引の引き換えに、一定期間、販売当事者が製品を特定数量または特定金額で販売を実行するという認識に基づいて、1 社以上の当事者間の合意を記載した元伝票。

###### <a name="sales-agreement-policy"></a>**販売契約ポリシー**

当事者が販売契約書の条項を変更することを認めたポリシー。

###### <a name="sales-order-policy"></a>**販売注文ポリシー**

当事者が販売注文の条項を変更し、発注処理と支払いプロセスを管理することを認めたポリシー。

###### <a name="sales-quotation"></a>**販売見積**

販売プロセスの見積依頼に応じて、指定価格で、指定日までに、一定数量の製品を供給するという申し出を記載した元伝票。

###### <a name="security-role"></a>**セキュリティ ロール**

定義された一連の申請のアクセス権限。 ユーザーがどのタスクを実行できるか、ユーザー インターフェイスのどの部分を表示できるかは、ユーザーに割り当てられたセキュリティ ロールで決まります。 どのユーザーも、システムにアクセスするには、少なくとも一つのセキュリティ ロールを割り当てられる必要があります。

###### <a name="segregation-of-duties"></a>**職務分掌**

不正、反則、エラーのリスクを減らすための設計原則。この原則で、資産の記録、検証、許可、保管を区分し、経済取引の財務結果の文書化、または記録にたずさわる個人の職務を定期的に見直します。

###### <a name="sequencing"></a>**順番付け **

製造施設における目標達成のためのジョブの処理順序または業務の実施順序。

###### <a name="shared-asset"></a>**共有資産**

複数のキャッシュ生成単位 (CGU) により使用される資産。 一例は同じ物流センターを共有するさまざまな市場に転送される前に、品目を保管するために使用される物流センター。

###### <a name="single-use-kanban"></a>**単一の使用かんばん**

特別に高い需要を満たすために、固定数量かんばんルールで使用するかんばんのタイプ。 単一使用かんばんは破棄時に新しいかんばんが発生しません。

###### <a name="smart-rounding"></a>**スマート丸め**

価格設定に最も近い概数よりわずかに小さい奇数を使用するマーケティング プラクティス。

###### <a name="source-requirement"></a>**ソース要件**

元伝票行に記載された製品数量。ペギング要件を作成します。

###### <a name="statistical-baseline-forecast"></a>**統計ベースライン予測**

歴代のトランザクション データに予測のアルゴリズムを適用して作成された将来の需要の見積。

###### <a name="supermarket"></a>**スーパーマーケット**

受け入れ能力が有限で、消費ポイント付近まで材料をまとめるためのジャストインタイム在庫場所。

###### <a name="system-defined-financial-dimension-type"></a>**システム定義の財務分析コード タイプ**

データベースのテーブルやビューにマップされている財務分析コード。エンティティ タイプを表し、その値はそのエンティティ タイプの一つの属性のドメインに所属します。

### <a name="t"></a>**T**

###### <a name="takt-time"></a>**タクト タイム**

製品の 1 単位を生産するのに必要な時間。

###### <a name="task-guide"></a>**タスク ガイド**

タスクの記録の手順を通じて、制御され、ガイドされるエクスペリエンス。 タスク ガイドでは、記録されたエクスペリエンスを通じてユーザーを案内します。 このタスク ガイドなしにタスク ガイドを完了すると、すべてのセキュリティ、データ、およびアプリケーションの動作は同じになります。 ユーザーに記録の次の手順を実行するよう促すために、ユーザーが意図されている手順をいつ完了したかを知るためのタスク レコーダーとして、タスク ガイドは同じインストルメンテーションを使用します。

###### <a name="task-recorder"></a>**タスク レコーダー**

Dynamics 365 for Operations にプリインストールされたツール。 記録するときに、ユーザーが Dynamics 365 for Operations ユーザー インターフェイスに入力した、サーバーに対して実行される、追加された値、変更された設定、削除されたデータなどを含むすべてのイベントを記録します。

###### <a name="task-recording"></a>**タスクの記録**

タスク レコーダが Microsoft Dynamics 365 for Operations で実行されるときに把握される注記および活動を含むファイル。

###### <a name="three-way-matching-policy"></a>**スリーウェイ マッチング ポリシー**

一つ以上の仕入先請求書価格が一つ以上の発注書の価格と一致することが求められ、一つ以上の仕入先請求書の数量が一つ以上の受領書製品の数量と一致することが求められる照合ポリシー。

###### <a name="transfer-batch"></a>**転送バッチ**

転送される一つ以上の品目または転送が可能な一つ以上の品目の数量。

###### <a name="two-way-matching-policy"></a>**2 状況照合ポリシー**

一つ以上の仕入先請求書の価格が一つ以上の発注書の価格と一致することを求める照合ポリシー。

### <a name="u"></a>**U**

###### <a name="user-defined-financial-dimension-type"></a>**ユーザー定義の財務分析コード タイプ**

ユーザー定義の値一覧にマップされている財務分析コード。

### <a name="v"></a>**V**

###### <a name="value-stream"></a>**バリュー ストリーム**

一つ以上の生産フローを制御する作業単位。

###### <a name="variant-configuration-technology"></a>**バリアント コンフィギュレーション テクノロジ**

製品マスターをモデル化し、製品バリアントのコンフィギュレーションを検索する方法。

###### <a name="vat-value-added-tax"></a>** VAT (付加価値税) **

生産の各段階の付加価値税に基づいて各段階で製品に課せられる税。

###### <a name="vendor-catalog"></a>**仕入先カタログ**

仕入先からの購買に利用できる提供製品の一覧。

###### <a name="vendor-invoice"></a>**仕入先請求書**

仕入先の支払要求を記載した元伝票。 仕入先請求書には、一つ以上の発注書を指定できます。 仕入先請求書が承認されると、仕入先に支払いを実施できます。

### <a name="w"></a>**W**

###### <a name="work-cell"></a>**作業セル**

生産フロー活動に関わるリソース グループ。

###### <a name="worker"></a>**作業者**

従業員や契約社員のロールを担い、サービスと引き換えに報酬を支払われる個人。

###### <a name="workspace"></a>**ワークスペース**

ユーザーが実行活動の一つの概要を表示するページ。 ページは、保留中の作業および複数ソースからの関連データを表示するためにタイル、リストおよびグラフを使用し、このデータに関連付けられる頻繁なタスクを表示します。 ワークスペースの集計セクションに表示されるタイルは、ダッシュボードの関連セクションに表示されます。
