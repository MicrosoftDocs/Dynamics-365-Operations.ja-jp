# <a name="action-search"></a>アクション検索

この記事では、Microsoft Dynamics 365 for Operationsの検索機能について説明します。 工程の検索はページのアクションを指定して実行できます。

<a name="introduction"></a>はじめに
------------

、Microsoft Dynamics 365のページは、主にアクション ウィンドウのコマンドを、ページのさまざまなセクションに表示されるツール バー、ページの上部に表示される標準アクション ウィンドウを公開します。 [以前のバージョンでは、キー ヒント機能は、Altキーおよび一連の文字を押して、アクション ウィンドウのボタンにアクセスするには、有効にします。 

[![] keyTipsAX6 (。/media/keytipsax6.png) ] (。ただし、Dynamics 365 for Operationsの現在のバージョンのは/media/keytipsax6.png)、キー ヒントは使用できなくなりますが、工程の検索機能を取り替えられました。 この新しい機能ですぐに検索でき、表示されているどのアクション ウィンドウのボタンからも実行できます。

## <a name="using-action-search"></a>アクション検索の使用
アクション検索機能を使用するには、次の手順に従います。

1.  アクション ウィンドウで、**アクション検索**フィールドをクリックします。 (**アクション検索**フィールドには、虫眼鏡アイコンが含まれます)。
2.  [実行するボタンの名前の全体または一部を入力します。 [ボタンの「パスからの語を使用して検索できます」。  (詳細については、この記事で次のセクションを参照してください)。通常、ボタンの結果一覧の先頭の近くに2から4文字を入力すると表示されます。
3.  検索して、結果リストのボタンを実行します (マウスまたはキーボードを使用して)。

ボタンの実行後、作業し続けることができるように、フォーカスはページの最後の位置に戻されます。 

[![工程検索]フィールド (。/media/action検索Field.png) ] (。/media/action検索field.png) 

Ctrl + /、または Alt + Q を押しても、アクション検索を開始できます。 ページの最後の位置にフォーカスを返すには、キーボード ショートカットを再度押します。

## <a name="understanding-the-results-list"></a>結果一覧の理解
多くの場合、Dynamics 365 for Operationsで、ボタンの場所と完全に、ボタンの目的を理解するためにコンテキストの両方を知る必要があります。 したがって、ボタンが一覧に表示されるを理解するために、追加の情報は結果一覧で品目ごとに、表示されます。 特に、ボタンの「パス」が表示されます。 このパスは、必要に応じて、次の UI 要素のラベルを含む場合があります:

-   アクション ウィンドウ タブ
-   ボタン グループ
-   メニュー ボタン (ボタンがメニュー ボタンの中にある場合)
-   メニュー区切り (ボタンがメニュー ボタンの中の名前を付けられたグループの中にある場合)
-   ページのグループあるいはタブ (たとえば、クイック タブの名前)

たとえば、**tot**と**アクション検索**フィールドに入力し、現在結果一覧を確認します。 指定されたボタンの最初のエントリは、**合計**、強調表示されます。 **販売注文** **] &gt; ボタンのパス**も表示されます。 **パスの販売注文**コンポーネントはアクション ウィンドウの**販売注文**タブに対応し、パスの**表示**コンポーネントは、このタブの**表示**グループに対応します。 同様に、**割引合計**ボタンのパス (**販売** &gt; ** **計算し、このボタンがあり) **計算する**のグループ**販売**アクション ウィンドウのタブ) が表示されます。 したがって、この情報 (結果一覧でその]を選択した場合) ボタンは、アクションの検索によってトリガされたを理解するのに役立ちます。
