# <a name="set-up-sepa-direct-debit-mandate"></a>SEPA の口座引落の委任状の設定



顧客が債権者に署名済み委任状を付与した場合、単一ユーロ支払地域 (SEPA) の口座引落により、債権者は顧客の銀行口座から資金を収集することができます。 債権者が支払を回収することを承認する委任状に顧客が署名し、顧客の銀行に支払の回収を指示します。 このトピックでは、SEPA口座引落順序を設定するためのプロセスを表示するにまとめられます。

## <a name="prerequisites"></a>必要条件
次の表に、開始する前に準備が整っている必要のある前提条件を示します。

| カテゴリ       | 前提条件                                                                                                                                              |
|----------------|-----------------------------------------------------------------------------------------------------------------------------------------------------------|
| 国/地域 | 法人の基本住所は、オーストリア、ベルギー、ドイツ、スペイン、フランス、イタリア、またはオランダである必要があります。 |

1. 個々の口座引落順序は一意の番号が必要な口座引落注文の番号順序を設定します。 **番号順序**ページを使用して、口座引落委任状の番号順序を作成します。 この識別子を使用して、**売掛金勘定パラメーター** ページの口座引落委任状システムに番号順序を割り当てます。

2. 口座引落注文の売掛金勘定パラメータを使用して**売掛金勘定パラメータ**口座引落注文のパラメータを設定するには、次のページを設定します。 パラメータは、必要になるように**口座引落**]タブで設定するには、変更既定のパラメータ。 その後、**番号順序**]タブで、**口座引落注文ID **フィールドには、番号順序で先に設定して更新します。

3. 口座引落注文の支払方法を設定する必要がある口座引落注文の支払方法を設定します。 この支払方法を使用して、口座引落支払を生成する請求書をクエリします。 **支払方法**ページを使用して、支払方法を設定します。 口座引落委任状の支払方法を設定するには、支払い方法に関する次の追加手順に従う必要があります。

-   [**支払タイプ**] フィールドで、[**電子支払**] を選択します。
-   オプション: 顧客ごとの複数の注文を行うと期待した期間で** **、[フィールドについて**請求**します。 個別の支払がごとに作成されます。各支払が請求書に対して指定された順序を使用します。
-   口座引落の委任状を使用して支払を作成するには、[**委任状の要求**] オプションを選択します。 [**委任状の要求**] オプションは、[**支払タイプ**] フィールドで [**電子支払**] を選択している場合にのみ使用できます。
