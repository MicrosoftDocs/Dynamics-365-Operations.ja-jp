# <a name="handling-cash-discounts-for-overpayments"></a>過剰支払の場合の現金割引の処理

[!include[banner](../includes/banner.md)]


この記事では、顧客が現金割引を受け、かつ過剰支払をしている場合の支払の処理方法を示すシナリオについて説明します。 

請求書は、支払額が現金割引を差し引いた請求金額を超えると、過剰支払いと見なされます。 請求書が過剰支払された場合の取得可能な現金割引差額の処理方法を指定するには、[**現金割引管理**] ページと [**売掛金勘定パラメーター**] ページの [**最大の過剰支払/過少支払**] フィールドを使用します。 次の例では、顧客の請求書が 0.50 過剰支払になっています。

| 請求合計 | 利用可能な現金割引 | 支払額 (現金割引を含む) | 顧客の実際の支払額 |
|---------------|-------------------------|-----------------------------------------------------|-----------------------------------|
| 105.00        | 1050                   | 94.50                                               | 95.00                             |

## <a name="cash-discount-administration--specific"></a>現金割引管理オプション = 限定
[**自動トランザクションの勘定**] ページの [**現金割引管理**] フィールドで [**特定**] が選択されている場合、完全な現金割引が適用されます。 過剰支払い額は、現金割引差分勘定科目へ転記されるか、または顧客勘定の残高になります。 動作は、過剰支払の金額が 0.00 と [**最大の過剰支払/過少支払**] フィールドに入力されている額の間にあるか、または過剰支払金額が [**最大の過剰支払/過少支払**] 額より大きいかによって異なります。

### <a name="scenario-1"></a>シナリオ 1

このシナリオでは、過剰支払い額が 0.00 と最大過剰支払または過少支払の間にあります。 105.00 の請求書が、請求書が 7 日以内に支払われた場合に現金割引されるという条件で入力されます。

| 請求合計 | 利用可能な現金割引 | 支払額 (現金割引を含む) |
|---------------|-------------------------|-----------------------------------------------------|
| 105.00        | 1050                   | 94.50                                               |

顧客が現金割引期間内に 95.00 の支払を送信します。 支払は、105.00 の請求書に対して決済されます。 請求書と支払が決済されると、次のトランザクションは顧客の売掛金勘定で作成されます。

| トランザクション   | 金額 | 残高 |
|---------------|--------|---------|
| 請求書       | 105.00 | 0.00    |
| 支払       | -95.00 | 0.00    |
| 現金割引 | -10.50 | 0.00    |

次の勘定項目が、支払および決済に生成されます。 [**支払**]

| 口座             | 借方金額 | クレジット金額 |
|---------------------|--------------|---------------|
| 現金                | 95.00        |               |
| 売掛金管理 |              | 95.00         |

**決済**

| 口座                                                                                                          | 借方金額 | クレジット金額 |
|------------------------------------------------------------------------------------------------------------------|--------------|---------------|
| 現金割引 ([**現金割引**] ページの [**顧客割引の主勘定**] フィールド)                 | 1050        |               |
| 売掛金管理                                                                                              |              | 1050         |
| 顧客現金割引 ([**自動トランザクションの勘定**] ページの [**顧客現金割引**] フィールド) |              | 0.50          |
| 売掛金管理                                                                                              | 0.50         |               |

### <a name="scenario-2"></a>シナリオ 2

このシナリオでは、過剰支払い額が最大過剰支払または過少支払を超えています。 105.00 の請求書が、請求書が 7 日以内に支払われた場合に現金割引されるという条件で入力されます。

| 請求合計 | 利用可能な現金割引 | 支払額 (現金割引を含む) |
|---------------|-------------------------|-----------------------------------------------------|
| 105.00        | 1050                   | 94.50                                               |

顧客が現金割引期間内に 95.00 の支払を送信します。 支払は、105.00 の請求書に対して決済されます。 請求書と支払が決済されると、次のトランザクションは顧客の売掛金勘定で作成されます。

| トランザクション   | 金額 | 残高 |
|---------------|--------|---------|
| 請求書       | 105.00 | 0.00    |
| 支払       | -95.00 | -0.50   |
| 現金割引 | -10.50 | 0.00    |

0.50 の支払超過分の金額は、支払の未処理残高として残り、異なる請求書に対して決済することができます。 次の勘定項目が、支払および決済に生成されます。 [**支払**]

| 口座             | 借方金額 | クレジット金額 |
|---------------------|--------------|---------------|
| 現金                | 95.00        |               |
| 売掛金管理 |              | 95.00         |

**決済**

| 口座                                                                                          | 借方金額 | クレジット金額 |
|--------------------------------------------------------------------------------------------------|--------------|---------------|
| 現金割引 ([**現金割引**] ページの [**顧客割引の主勘定**] フィールド) | 1050        |               |
| 売掛金管理                                                                              |              | 1050         |

## <a name="cash-discount-administration--unspecific"></a>現金割引管理 = 非限定
[**自動トランザクションの勘定**] ページの [**現金割引管理**] フィールドで [**非限定**] が選択されている場合、現金割引金額が過剰支払金額によって減額されます。 この動作は、過剰支払の金額がまたはフィールドに入れられる金額未満であるかどうかに関係なく、常に **最大の超過/不足支払** 適用されます。

### <a name="scenario-3"></a>シナリオ 3

このシナリオでは 105.00 の請求書が、請求書が 7 日以内に支払われた場合に現金割引されるという条件で入力されます。

| 請求合計 | 利用可能な現金割引 | 支払額 (現金割引を含む) |
|---------------|-------------------------|-----------------------------------------------------|
| 105.00        | 1050                   | 94.50                                               |

顧客が現金割引の日付内に 95.00 の支払を送信します。 支払は、105.00 の請求書に対して決済されます。 請求書と支払が決済されると、次のトランザクションは顧客の売掛金勘定で作成されます。

| トランザクション   | 金額 | 残高 |
|---------------|--------|---------|
| 請求書       | 105.00 | 0.00    |
| 支払       | -95.00 | -0.00   |
| 現金割引 | -10.00 | 0.00    |

現金割引金額は 10.50 から 10.00 に減少します。 支払と請求書が決済済みと見なされます。 [**支払**]

| 口座             | 借方金額 | クレジット金額 |
|---------------------|--------------|---------------|
| 現金                | 95.00        |               |
| 売掛金管理 |              | 95.00         |

**決済**

| 口座                                                                                          | 借方金額 | クレジット金額 |
|--------------------------------------------------------------------------------------------------|--------------|---------------|
| 現金割引 ([**現金割引**] ページの [**顧客割引の主勘定**] フィールド) | 1050        |               |
| 売掛金管理                                                                              |              | 1050         |
