# <a name="sepa-credit-transfer-overview"></a>SEPA 口座振替の概要

この記事では、仕入先ごとに一つの統一の支払の領域(SEPA)の銀行口座振替およびそのほかの電子支払を含むISO 20022の銀行口座振替に関する一般情報を提供します。 銀行のSEPA口座振替は1社のユーザーの他の会社またはユーザーにユーロの支払の特定のタイプです。 トピックでは、銀行口座振替の支払ファイルを設定して送信する方法を説明します。

## <a name="what-is-a-credit-transfer-message"></a>銀行口座振替メッセージはどのぐらいか。
銀行口座振替メッセージが開始、関係者 (会社) がそれぞれの勘定から債権者に資金移動に送信する要求です。 多くの銀行口座振替メッセージの国/地域固有の銀行と固有の実装できます。 その一部は、一つの国や地域で使用され、一部となる標準です。 一つの設定したグローバルのISO標準は20022、銀行口座振替などの起動メッセージです。 次の図は、選択した銀行口座振替メッセージの関係と補充を示します。

## <a name="what-are-iso-20022-and-sepa-payments"></a>ISO 20022と、SEPAの支払はどのぐらいか。
単一ユーロ支払地域 (SEPA) は欧州委員会によって設定され、個人、事業または組織および銀行がどの国または地域にあるかに関係なく、すべての電子支払は国内と見なされます。 各国の支払と国境を越えた支払の違いはありません。 SEPAは、欧州連合(EU)の28の加盟国、およびアイスランド、リヒテンシュタイン、ノルウェー、スイス、モナコとサンマリノが含まれます。 SEPA により、欧州経済領域 (EEA) 内の支払トランザクションの単一市場が形成されます。 最終的に、SEPA により、銀行、事業、および個人が扱う支払形式の数を減らすと予想されます。 指令への支払サービス ディレクティブ (PSD) を介した SEPA 支払の法的根拠は欧州委員会が設定しました。 欧州決済協議会 (EPC) は、次の活動で SEPA をサポートしています。

-   これは、ISO 20022 の統合的な金融業務に関する通信メッセージ規格の XML 形式を使用して、SEPAの電子支払の標準を設定します。
-   これはユーロ支払処理に関するルールとガイドラインを設定します。

ヨーロッパの銀行から構成される EPC は SEPA 支払手段の商業的、技術的フレームワークを開発します。 3 つのタイプの SEPA 支払が使用されます。

-   口座振替
-   口座引落
-   カード

## <a name="what-is-a-sepa-credit-transfer"></a>SEPA 口座振替とは何ですか。
SEPA 口座振替は、個人または 1 つの会社から別の会社または個人に対する支払です。 支払はユーロで、国際銀行番号 (IBAN) および両方の当事者の銀行識別コード (BIC) を含む必要があります。  (BICエイリアス国際銀行間の通信Interbank \[SWIFTコード\] です)。また、取引費用は両方の関係者間で共有する必要があります。 当事者間で発生する口座振替は、ISO 20022 支払処理標準に準拠した XML ファイル、EPC によって指定されている XML 形式を使用する必要があります。

## <a name="how-is-a-credit-transfer-implemented"></a>銀行口座振替がどのように実行されるか。
欧州諸国の銀行口座振替の支払形式は、Dynamics 365 for Operationsで電子申告 ((ER)と支払方法の機能を使用して実行されます。 他の領域でまだ使用される数の銀行口座振替の形式は、レガシの支払フレームワークを使用します。 他の複数のフォームの間で使用可能な12のISO 20022の銀行口座振替のファイル形式があります。 これらのエクスポート形式は、SEPA ISO 20022 XMLの標準に準拠します。 品目のEPCをリリースする銀行のSEPA口座振替の方法での会議集のバージョン8.2で指定された、欧州の支払使用する国/地域の非ヨーロッパの支払振替を生成するのに使用されます。 銀行口座振替を実行する前に電子銀行決済ファイルをアップロードするために必要なソフトウェアを取得するために銀行にお問い合わせください。 銀行の支払形式を含むXMLファイルを転送する場合でも、そのソフトウェアを使用します。
