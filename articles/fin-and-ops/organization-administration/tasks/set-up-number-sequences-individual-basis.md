# <a name="set-up-number-sequences-on-an-individual-basis"></a><span data-ttu-id="7482d-103">番号順序を個別に設定する</span><span class="sxs-lookup"><span data-stu-id="7482d-103">Set up number sequences on an individual basis</span></span>

[!INCLUDE [task guide banner](../../includes/task-guide-banner.md)]

<span data-ttu-id="7482d-104">番号順序は、マスタ データ レコードおよびトランザクション レコードに必要な読みやすい固有 ID の生成に使用されます。</span><span class="sxs-lookup"><span data-stu-id="7482d-104">Number sequences are used to generate readable, unique identifiers for master data records and transaction records that require them.</span></span> <span data-ttu-id="7482d-105">ID が必要なマスタ データまたはトランザクション レコードは、参照先と呼ばれます。</span><span class="sxs-lookup"><span data-stu-id="7482d-105">A master data or transaction record that requires an identifier is referred to as a reference.</span></span> <span data-ttu-id="7482d-106">参照先に新しいレコードを作成するには、事前に番号順序を設定して参照先に関連付ける必要があります。</span><span class="sxs-lookup"><span data-stu-id="7482d-106">Before you can create new records for a reference, you must set up a number sequence and associate it with the reference.</span></span> <span data-ttu-id="7482d-107">必要な番号順序は、番号順序の設定ウィザードを使用してすべて同時に設定できるほか、[番号順序] ページを使用して個々の番号順序の作成または変更を行うことができます。</span><span class="sxs-lookup"><span data-stu-id="7482d-107">You can set up all required number sequences at the same time by using the Set up number sequences wizard, or you can create or modify individual number sequences by using the Number sequences page.</span></span>

1. <span data-ttu-id="7482d-108">[組織管理] > [番号順序] > [番号順序] の順に移動します。</span><span class="sxs-lookup"><span data-stu-id="7482d-108">Go to Organization administration > Number sequences > Number sequences.</span></span>
2. <span data-ttu-id="7482d-109">[番号順序] をクリックします。</span><span class="sxs-lookup"><span data-stu-id="7482d-109">Click Number sequence.</span></span>
3. <span data-ttu-id="7482d-110">[番号順序コード] フィールドに値を入力します。</span><span class="sxs-lookup"><span data-stu-id="7482d-110">In the Number sequence code field, type a value.</span></span>
4. <span data-ttu-id="7482d-111">[名前] フィールドに値を入力します。</span><span class="sxs-lookup"><span data-stu-id="7482d-111">In the Name field, type a value.</span></span>
5. <span data-ttu-id="7482d-112">[スコープ パラメーター] セクションを展開します。</span><span class="sxs-lookup"><span data-stu-id="7482d-112">Expand the Scope parameters section.</span></span>
    * <span data-ttu-id="7482d-113">[スコープ パラメーター] クイック タブで、番号順序の範囲を選択してから範囲値を選択します。</span><span class="sxs-lookup"><span data-stu-id="7482d-113">On the Scope parameters FastTab, select a scope for the number sequence and select scope values.</span></span>     <span data-ttu-id="7482d-114">範囲によって、番号順序を使用する組織が決まります。</span><span class="sxs-lookup"><span data-stu-id="7482d-114">The scope defines which organizations use the number sequence.</span></span> <span data-ttu-id="7482d-115">また、[共有] 以外の範囲が指定されている番号順序に、その範囲に対応する区分を設定することもできます。</span><span class="sxs-lookup"><span data-stu-id="7482d-115">In addition, number sequences that have a scope other than Shared can have segments that correspond to their scope.</span></span> <span data-ttu-id="7482d-116">たとえば、法人の範囲の番号順序には、法人の区分を含めることができます。</span><span class="sxs-lookup"><span data-stu-id="7482d-116">For example, a number sequence with a scope of Legal entity can have a legal entity segment.</span></span> <span data-ttu-id="7482d-117">範囲の詳細については、「番号順序の概要」ヘルプ トピックを参照してください。</span><span class="sxs-lookup"><span data-stu-id="7482d-117">For more information about scopes, see the "Number sequence overview" help topic.</span></span>  
6. <span data-ttu-id="7482d-118">[区分] セクションを展開します。</span><span class="sxs-lookup"><span data-stu-id="7482d-118">Expand the Segments section.</span></span>
    * <span data-ttu-id="7482d-119">[区分] クイック タブで、区分の追加、削除、および並べ替えを行って、番号順序の形式を定義します。</span><span class="sxs-lookup"><span data-stu-id="7482d-119">On the Segments FastTab, define the format for the number sequence by adding, removing, and rearranging segments.</span></span>  
    * <span data-ttu-id="7482d-120">すべての範囲の番号順序には、定数および英数字区分を含めることができます。</span><span class="sxs-lookup"><span data-stu-id="7482d-120">Number sequences of all scopes can contain Constant segments and Alphanumeric segments.</span></span> <span data-ttu-id="7482d-121">定数区分には、一定の英数字が含まれています。</span><span class="sxs-lookup"><span data-stu-id="7482d-121">Constant segments contain a set of alphanumeric characters that do not change.</span></span> <span data-ttu-id="7482d-122">この区分タイプは、番号順序区分の間にハイフンまたはその他の区切り文字を追加する場合に使用します。</span><span class="sxs-lookup"><span data-stu-id="7482d-122">Use this segment type to add a hyphen or other separators between number sequence segments.</span></span> <span data-ttu-id="7482d-123">英数字区分には、番号記号 (#) とアンパサンド (&) の組み合わせが含まれています。</span><span class="sxs-lookup"><span data-stu-id="7482d-123">Alphanumeric segments contain a combination of number signs (#) and ampersands (&).</span></span> <span data-ttu-id="7482d-124">これらの文字は、順序の番号が使用されるたびに繰り上がる文字と数字を表します。</span><span class="sxs-lookup"><span data-stu-id="7482d-124">These characters represent letters and numbers that increment every time that a number from the sequence is used.</span></span> <span data-ttu-id="7482d-125">番号記号 (#) は数字の繰り上げに、アンパサンド (&) は文字の繰り上げに使用します。</span><span class="sxs-lookup"><span data-stu-id="7482d-125">Use a number sign (#) to indicate incrementing numbers and an ampersand (&) to indicate incrementing letters.</span></span> <span data-ttu-id="7482d-126">たとえば、#####_2014 という形式を指定した場合、00001_2014、00002_2014 という順序で番号が割り当てられます。</span><span class="sxs-lookup"><span data-stu-id="7482d-126">For example, the format #####_2014 creates the sequence 00001_2014, 00002_2014, and so on.</span></span>     <span data-ttu-id="7482d-127">英数字区分は少なくとも 1 つ必要です。</span><span class="sxs-lookup"><span data-stu-id="7482d-127">At least one alphanumeric segment must be present.</span></span> <span data-ttu-id="7482d-128">会社や法人などの範囲区分は必須ではありません。</span><span class="sxs-lookup"><span data-stu-id="7482d-128">Scope segments, such as company or legal entity, are not required.</span></span> <span data-ttu-id="7482d-129">ただし、形式に範囲区分を含めない場合、選択した参照先の番号は範囲ごとに生成されます。</span><span class="sxs-lookup"><span data-stu-id="7482d-129">However, if you do not include scope segments in the format, numbers for the selected reference are still generated per scope.</span></span>  
7. <span data-ttu-id="7482d-130">[参照] セクションを展開します。</span><span class="sxs-lookup"><span data-stu-id="7482d-130">Expand the References section.</span></span>
    * <span data-ttu-id="7482d-131">[参照] クイック タブで、この番号順序を割り当てるドキュメント タイプまたはレコードを選択します。</span><span class="sxs-lookup"><span data-stu-id="7482d-131">On the References FastTab, select the document type or record to assign this number sequence to.</span></span>     <span data-ttu-id="7482d-132">この手順は、特別なアプリケーションの使用パターンに定義された順序については必須ではありません。</span><span class="sxs-lookup"><span data-stu-id="7482d-132">This step is optional for sequences that are defined for special application usage patterns.</span></span> <span data-ttu-id="7482d-133">これらのシナリオでは、参照先ではなく、番号順序コードまたは ID の値を使用して、新しい番号が生成されます。</span><span class="sxs-lookup"><span data-stu-id="7482d-133">In these scenarios, a new number is generated by using the value of a number sequence code or ID, without using a reference.</span></span> <span data-ttu-id="7482d-134">特殊なアプリケーションの使用パターンには、たとえば特定の仕訳帳名に使用される伝票シリーズなどがあります。</span><span class="sxs-lookup"><span data-stu-id="7482d-134">An example of a special application usage pattern is a voucher series that is used for specific journal names.</span></span> <span data-ttu-id="7482d-135">ただし、このようなパターンの使用はお勧めしません。</span><span class="sxs-lookup"><span data-stu-id="7482d-135">However, we do not recommend that you use such patterns.</span></span>  
8. <span data-ttu-id="7482d-136">[一般] セクションを展開します。</span><span class="sxs-lookup"><span data-stu-id="7482d-136">Expand the General section.</span></span>
    * <span data-ttu-id="7482d-137">[全般] クイック タブで、番号順序を手動で生成するかどうか、および連続させるかどうかを選択します。</span><span class="sxs-lookup"><span data-stu-id="7482d-137">On the General FastTab, specify whether the number sequence is manual, and continuous or non-continuous.</span></span> <span data-ttu-id="7482d-138">また、番号順序で使用できる最小番号と最大番号を入力します。</span><span class="sxs-lookup"><span data-stu-id="7482d-138">In addition, enter the lowest and highest numbers that can be used in the number sequence.</span></span>     <span data-ttu-id="7482d-139">連続しない番号順序を連続する番号順序に変更することはお勧めしません。</span><span class="sxs-lookup"><span data-stu-id="7482d-139">We do not recommend changing a non-continuous number sequence to a continuous number sequence.</span></span> <span data-ttu-id="7482d-140">番号順序が、完全には連続しなくなります。</span><span class="sxs-lookup"><span data-stu-id="7482d-140">The number sequence will not be truly continuous.</span></span> <span data-ttu-id="7482d-141">この変更を行うと、データベースに重複キーエラーが発生してしまうこともあります。</span><span class="sxs-lookup"><span data-stu-id="7482d-141">This change may also cause duplicate key violations in the database.</span></span> <span data-ttu-id="7482d-142">また、連続する番号順序は、パフォーマンスに大きな影響を与えます。</span><span class="sxs-lookup"><span data-stu-id="7482d-142">In addition, continuous number sequences have a larger effect on performance.</span></span>   
9. <span data-ttu-id="7482d-143">[保存] をクリックします。</span><span class="sxs-lookup"><span data-stu-id="7482d-143">Click Save.</span></span>
