# <a name="feature-management-overview"></a>機能管理の概要

[!include[banner](../../includes/banner.md)]
[!include [banner](../../includes/preview-banner.md)]

機能は、Microsoft Dynamics 365 for Finance and Operations のすべてのリリースで追加および更新されます 機能管理エクスペリエンスは、各リリースで提供されている機能のリストを表示できるワークスペースを提供します。 既定では、新機能が無効になっています。 ワークスペースを使用してそれらを有効にし、それらのドキュメントを参照できます。

## <a name="the-feature-management-workspace"></a>機能管理ワークスペースの使用

**機能管理**ワークスペースを開くには、ダッシュボードで適切なタイルを選択します。 このページには、機能管理エクスペリエンスにサポートされているすべてのリリースの機能一覧が表示されます。 時間の経過と共に、Microsoft は機能管理エクスペリエンスを強化して、機能を管理できるようにします。

機能一覧には、次の情報が含まれます。

- **機能名** – 追加された機能の説明。
- **有効な状態** – シンボルは、機能が有効になっているか (チェック マーク)、有効になっていないか (空白)、有効 (clock) にスケジュール設定されているか、必須の有効化 (ロック) であるかを示します。 ここに表示されている設定は、すべての法人に使用されます。 機能が有効になっている場合でも、セキュリティによって制御されることに注意してください。 したがって、この機能はセキュリティ ロールに基づいてアクセス権を持つユーザーのみが使用できるようになります。 このオプションは、ユーザーがアクセス権を持つ法人に対しても使用できます。
- **有効な日付** – 機能が有効になった日付または日付が未来の場合はオンになります。
- **追加機能** – 機能が環境に追加された日付。 月単位のリリース サイクルで環境を更新すると、この日付が自動的に入力されます。
- **モジュール** – 新しい機能によって影響を受けるモジュール。

機能を選択すると、追加情報が機能一覧の右側にある詳細ウィンドウに表示されます。 ウィンドウの上部には、機能名、機能が追加された日付、機能によって影響を受けたモジュール、および**詳細情報**リンクが表示されます。 機能のドキュメントを表示するには、このリンクを選択します。 ドキュメントを使用できない場合は、一時的なページに移動します。 詳細ウィンドウには、機能に関する独自のコメントを追加できる**コメント**フィールドもあります。

**機能管理**ワークスペースには、その中の機能一覧を含むいくつかのタブも含まれます。 
- **新しい** - 毎月の更新後に追加されたすべての機能を表示します。 毎月の更新をスキップした場合は、前回更新された後のすべての新しい機能が含まれます。 最新の機能は、一覧の先頭に表示されます。 新しい機能の合計数も、ページ上部のタイルに表示されます。
- **無効** - 有効になっていないすべての機能を表示します。 最新の機能は、一覧の先頭に表示されます。 新しい機能の合計数も、ページ上部のタイルに表示されます。
- **スケジュール済** - 未来の日付で有効になるようスケジュールされているすべての機能を表示します。 最も早くスケジュールされた日付を持つ機能が一覧の先頭に表示されます。 新しい機能の合計数も、ページ上部のタイルに表示されます。
- **すべて** - すべての機能を表示します。 最新の機能は、一覧の先頭に表示されます。


## <a name="enable-a-feature"></a>機能の有効化

機能が有効になっていない場合、**有効化**ボタンがウィンドウに表示されます。 このボタンを使用して機能を有効にすることができます。

1. 有効にする機能を選択し、詳細ウィンドウで**有効化**を選択します。
2. スライダーが表示され、機能を有効にする日付を指定できます。 既定では、この日付が現在の日付に設定されます。
3. 機能を有効にするには、**有効化**を選択します。

一部の機能は、有効にした後で無効にすることはできません。 有効にしようとしている機能を無効にできない場合は、警告が表示されます。 この時点で、**キャンセル**を選択して操作をキャンセルし、機能を無効のままにしておくことができます。 ただし、**有効化**を選び機能を有効にすると、後で無効にすることはできなくなります。

この機能を有効にすると、詳細ウィンドウの**詳細情報**リンクの下にメッセージが表示されます。 このメッセージは、その機能が有効になっていることを示しているか、または今後いつ有効になるかを示します。 将来の日付を使用すると、機能は**スケジュール済**リストに表示されます。 このメッセージは、機能一覧から機能を選択するたびに表示されます。 システムの日付によって表されるタイム ゾーンに基づくバッチ処理によって、今後は深夜 0 時にスケジュールされた機能が有効になります。 

## <a name="reschedule-a-feature"></a>機能の再スケジュール

将来、機能が有効になった場合、**再スケジュール**ボタンが詳細ウィンドウに表示されます。 このボタンを使用して、**有効日**を別の日付に変更することができます。

1. 再スケジュールするスケジュール済機能を選択し、詳細ウィンドウで**再スケジュール**を選択します。
2. スライダーが表示され、機能を有効にする日付を指定できます。 
3. **有効化**を選択し機能を再スケジュールまたは**無効化**を選択しスケジュールをキャンセルします。

## <a name="disable-a-feature"></a>機能の無効化

機能が既に有効になっていた場合、**無効化**ボタンが詳細ウィンドウに表示されます。 このボタンを使用して機能を無効にすることができます。 **無効化**ボタンは、有効にした後に無効にすることができない場合、使用できません。

- 無効にする機能を選択し、詳細ウィンドウで**無効化**を選択します。

- この機能は無効になっており、日付がクリアされています。

この機能を無効にすると、詳細ウィンドウの**詳細情報**リンクの下にメッセージが表示されます。 このメッセージは、機能がまだ有効になっていないため、**有効ではない**の一覧に表示されることを示しています。 メッセージは、機能一覧から機能を選択するたびに表示されます。

## <a name="features-that-must-be-enabled"></a>有効にする必要がある機能

更新時に自動的に有効にする必要がある重要な機能を配信することができます。 **有効日**に対して自動的に有効になります。 詳細ウィンドウの**詳細情報**リンクの下にメッセージが表示されます。 このメッセージは、機能が有効になっている、または**有効日**に対して自動的に有効になることを指定します。 メッセージは、機能一覧から機能を選択するたびに表示されます。

## <a name="assigning-roles"></a>ロールの割り当て

**機能管理**ワークスペースは、システム管理者によって、または機能管理エクスペリエンスをサポートするために作成された機能マネージャーまたは機能ビューアー ロールに割り当てられているユーザーによって開かれます。 機能マネージャー ロールのユーザーは、すべての機能を有効または無効にすることができます。 また、機能のコメント セクションを更新することもできます。 機能ビューアー ロールのユーザーは、**機能管理**ワークスペースの表示のみを実行できます。 機能を有効または無効にすることはできません。

機能マネージャー ロールおよび機能ビューアー ロールは、ユーザーの既存のセキュリティを上書きするものではありません。 ロールは機能を有効にするためのみに、アクセスを制御します。 機能自体へのアクセスは提供されません。

## <a name="using-feature-management-to-enable-isv-features-or-custom-features"></a>機能管理を使用して ISV 機能またはカスタム機能を有効にする

現時点では、機能管理プロセスは ISV 機能またはカスタム機能には使用できません。 機能管理を強化するための機能を追加し、それらの機能強化が完了すると、すべての機能に対して機能管理を開始し、機能を更新してそれを使用するための具体的な手順を提供します。

## <a name="feature-management-and-flighting"></a>機能管理とフライティング

機能管理を使用すると、各リリースで出荷される機能を制御できます。 フライティングを使用すると、Microsoft Teams が限られた数の顧客に機能をリリースして、すべての顧客に影響を与えることなく機能をテストおよび検証できるようにすることができます。 機能管理は、すべての機能のフライティングを制御するものではありません。
