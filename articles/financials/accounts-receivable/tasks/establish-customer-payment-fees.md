# <a name="establish-customer-payment-fees"></a>顧客支払手数料の設定

[!include [task guide banner](../../includes/task-guide-banner.md)]

顧客支払の支払手数料を作成します。

このタスクでは、USMF というデモ会社を使用します。

1. [売掛金勘定] > [支払設定] > [支払手数料] の順に移動します。
2. [新規] をクリックします。
3. [手数料 ID] フィールドに、手数料 ID を入力します。
    * 支払仕訳帳には手数料 ID が表示されるため、どの手数料が算定されるかを分かるようにそれを記述します。  
4. [名前] フィールドに手数料名を入力します。
5. [手数料の説明] フィールドに、手数料の説明を入力します。
6. 手数料が「顧客」または「勘定科目」に請求されるかどうかを選択します。
    * 顧客が手数料を算定する場合、「顧客」を選択します。 手数料が経費として組織に算定される場合、「元帳」を選択します。 このタスクに関しては、「顧客」を選択します。  
7. この支払手数料を使用する仕訳帳のタイプを選択します。
    * これらの手数料は、顧客支払に使用されると、仕訳帳タイプは顧客支払となる可能性が高くなります。  
8. [保存] をクリックします。
9. [支払手数料の設定] をクリックします。
    * 支払手数料の設定は、支払手数料が評価されるときに、基準を定義するために使用されます。  たとえば、銀行口座が USMF OPER であり、支払方法が小切手である場合、費用が計算されるように指定できます。  
10. どの銀行口座がこの手数料を評価するかを定義するためには、「テーブル」、「グループ」または「すべて」を選択します。
    * 「すべて」を選択した場合、すべての銀行口座がこの手数料を評価する場合があります。  「テーブル」を選択し、選択した銀行口座のみがこの手数料を評価できる可能性があります。 「グループ」を選択した場合、選択された銀行グループの銀行口座のみがこの手数料を評価できる可能性があります。  
11. 銀行グループまたは銀行口座を選択します。
    * 「テーブル」を選択した場合、ルックアップによって銀行口座を表示します。 「グループ」を選択した場合、ルックアップによって銀行グループを表示します。  
12. 一覧で、選択された行のリンクをクリックします。
13. 支払手数料を評価するために使用する支払方法を選択します。
    * たとえば、支払を電子支払ではなく、小切手で行った場合、顧客の手数料を評価できます。  
14. 一覧で、目的のレコードを見つけ、選択します。
15. 該当する場合、支払通貨を入力します。
    * 手数料が評価されるかどうかの追加の基準として、支払通貨が使用されます。  たとえば、ユーザーの銀行は通常 EUR 通貨で処理するため、USD 通貨で受け取った支払に対して手数料を請求することがあります。  
16. 手数料が割合、金額、または間隔であるかどうかを選択します。
17. 手数料の割合または金額を入力します。
    * [割合/金額] フィールドで割合を選択した場合、ここに入力する値は割合となります。 [割合/金額] フィールドで金額を選択した場合、ここに入力する値は金額となります。 [割合/金額] フィールドで間隔を選択した場合、階層を定義している [間隔] タブを使用します。  
18. [手数料の通貨] フィールドで、手数料の通貨を選択します。
    * これは、作成される手数料の通貨です。  
19. [保存] をクリックします。
