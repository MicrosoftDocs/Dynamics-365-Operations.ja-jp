# <a name="work-breakdown-structures-overview"></a><span data-ttu-id="50778-104">作業分解構造の概要</span><span class="sxs-lookup"><span data-stu-id="50778-104">Work breakdown structures overview</span></span>

[!include [banner](../includes/banner.md)]

<span data-ttu-id="50778-105">作業分解構造 (WBS) は、プロジェクトに対して実行される作業の説明です。</span><span class="sxs-lookup"><span data-stu-id="50778-105">A work breakdown structure (WBS) is a description of the work that will be done for a project.</span></span> <span data-ttu-id="50778-106">作業の構成や各コンポーネントまたはタスクのサイズ、原価、期間に対するプロジェクト チームの理解を表すタスクの階層です。</span><span class="sxs-lookup"><span data-stu-id="50778-106">It’s a hierarchy of tasks that represents the project team’s understanding of the composition of work, and of the size, cost, and duration of each component or task.</span></span> <span data-ttu-id="50778-107">WBS には 3 つの主な目的があります:</span><span class="sxs-lookup"><span data-stu-id="50778-107">A WBS has three major purposes:</span></span>

-   <span data-ttu-id="50778-108">タスクでの作業の内訳または構成を説明します。</span><span class="sxs-lookup"><span data-stu-id="50778-108">Describe the breakdown or composition of work in tasks.</span></span>

<span data-ttu-id="50778-111">WBS の詳細の程度は、見積に必要な正確性のレベルとこれらの見積に対して必要な追跡のレベルに依存します。</span><span class="sxs-lookup"><span data-stu-id="50778-111">The degree of detail in a WBS depends on the level of accuracy that is required in estimates and the level of tracking that is required against those estimates.</span></span> <span data-ttu-id="50778-112">スケジュールまたは原価のずれに対する許容範囲が非常に低いプロジェクトは、通常、より詳細な WBS と WBS に対する作業の進捗と原価の入念な追跡が必要です。</span><span class="sxs-lookup"><span data-stu-id="50778-112">Projects that have very low tolerance for slippages in schedule or cost usually require a more detailed WBS, and diligent tracking of work progress and cost against the WBS.</span></span> <span data-ttu-id="50778-113">この種のプロジェクトは、建設およびエンジニアリング業界では一般的です。</span><span class="sxs-lookup"><span data-stu-id="50778-113">This kind of project is common in the construction and engineering industries.</span></span>

<span data-ttu-id="50778-116">WBS の作成は、通常、長期にわたって実施される、さまざま人からのコラボレーションや情報を必要とする、集中的なプロセスです。</span><span class="sxs-lookup"><span data-stu-id="50778-116">Building a WBS is an intensive process that is usually done over a long period, and that requires collaboration and information from a wide variety of people.</span></span> <span data-ttu-id="50778-117">このトピックでは、見積もりと追跡の要件を満たす Microsoft Dynamics 365 for Finance and Operations の WBS 拡張機能の使用方法について説明します。</span><span class="sxs-lookup"><span data-stu-id="50778-117">This topic describes how you can use WBS enhancements in Microsoft Dynamics 365 for Finance and Operations to meet your requirements for estimates and tracking.</span></span>

## <a name="creating-a-wbs"></a><span data-ttu-id="50778-137">WBS の作成</span><span class="sxs-lookup"><span data-stu-id="50778-137">Creating a WBS</span></span>
<span data-ttu-id="50778-138">WBS の作成では、3 つの活動を実行します。</span><span class="sxs-lookup"><span data-stu-id="50778-138">Creating a WBS involves three activities:</span></span>

<span data-ttu-id="50778-142">次のセクションでは、WBS 機能がこれらの各活動でどのように役立つかについて説明します。</span><span class="sxs-lookup"><span data-stu-id="50778-142">The following sections discuss how the WBS capabilities can help with each of these activities.</span></span>

### <a name="work-decomposition"></a><span data-ttu-id="50778-143">作業の分解</span><span class="sxs-lookup"><span data-stu-id="50778-143">Work decomposition</span></span>

<span data-ttu-id="50778-144">通常、WBS 作成プロセスでは、まず最初に作業の分割または分解を作成します。</span><span class="sxs-lookup"><span data-stu-id="50778-144">Creating a breakdown or decomposition of work is usually the first step in the process of creating a WBS.</span></span> <span data-ttu-id="50778-145">WBS 機能は、作業の分割または分解のために次の基本的な構造をサポートします。</span><span class="sxs-lookup"><span data-stu-id="50778-145">The WBS functionality supports the following basic constructs for work breakdown or decomposition.</span></span> 

<span data-ttu-id="50778-146">**プロジェクト ルート タスク** プロジェクト ルート タスクは、プロジェクトのトップレベルの集計タスクです。</span><span class="sxs-lookup"><span data-stu-id="50778-146">**Project root task** The project root task is the top-level summary task for a project.</span></span> <span data-ttu-id="50778-147">他のプロジェクト タスクは、すべてその下に作成されます。</span><span class="sxs-lookup"><span data-stu-id="50778-147">All other project tasks are created beneath it.</span></span> <span data-ttu-id="50778-148">ルート タスクの名前が、常にプロジェクトの名前に設定されます。</span><span class="sxs-lookup"><span data-stu-id="50778-148">The name of the root task is always set to the project name.</span></span> <span data-ttu-id="50778-149">ルート ノードの工数、日付、期間は、ルート タスクの下のタスクの値を集計します。</span><span class="sxs-lookup"><span data-stu-id="50778-149">The effort, dates, and duration of the root node summarize the values for the tasks beneath the root task.</span></span> <span data-ttu-id="50778-150">ルート ノードのプロパティは、変更または削除できません。</span><span class="sxs-lookup"><span data-stu-id="50778-150">You can’t modify the properties of the root node or delete it.</span></span>

<span data-ttu-id="50778-151">**集計またはコンテナ タスク** 集計タスクは、下位にサブ タスクまたは構成タスクをもつタスクです。</span><span class="sxs-lookup"><span data-stu-id="50778-151">**Summary or container tasks** A summary task is a task that has sub-tasks or constituent tasks beneath it.</span></span> <span data-ttu-id="50778-152">集計タスクには、それ自体の作業工数または原価はありません。</span><span class="sxs-lookup"><span data-stu-id="50778-152">A summary task doesn’t have any work effort or cost of its own.</span></span> <span data-ttu-id="50778-153">その代わりに、集計タスクの作業工数と原価は、構成タスクの作業工数と原価の合計になります。</span><span class="sxs-lookup"><span data-stu-id="50778-153">Instead, the work effort and cost of a summary task are the sum of the work effort and cost of its constituent tasks.</span></span> <span data-ttu-id="50778-154">構成タスクの最も早い開始日は集計タスクの開始日として使用され、構成タスクの最も遅い終了日は集計タスクの終了日として使用されます。</span><span class="sxs-lookup"><span data-stu-id="50778-154">The earliest start date of the constituent tasks is used as the start date of the summary task, and the latest end date of the constituent tasks is used as the end date.</span></span> <span data-ttu-id="50778-155">集計タスクの名前は変更できますが、工数、日付、期間のスケジューリング プロパティは変更できません。</span><span class="sxs-lookup"><span data-stu-id="50778-155">You can modify the name of a summary task, but you can’t modify the scheduling properties for effort, dates, and duration.</span></span> <span data-ttu-id="50778-156">集計タスクを削除する場合は、構成タスクもすべて削除します。</span><span class="sxs-lookup"><span data-stu-id="50778-156">If you delete a summary task, you also delete all its constituent tasks.</span></span> 

<span data-ttu-id="50778-157">**リーフ ノード タスク** リーフ ノード タスクは、プロジェクトの最も細かい作業パッケージを表します。</span><span class="sxs-lookup"><span data-stu-id="50778-157">**Leaf node tasks** A leaf node task represents the most granular work package on the project.</span></span> <span data-ttu-id="50778-158">リーフ ノードには、見積工数、リソースの計画番号、予定開始日と終了日、および期間があります。</span><span class="sxs-lookup"><span data-stu-id="50778-158">A leaf node has an estimated effort, a planned number of resources, a planned start date and end date, and duration.</span></span> 

<span data-ttu-id="50778-159">プロジェクトの作業階層または分解の作成を有効にするには、次の階層の操作を実行します。</span><span class="sxs-lookup"><span data-stu-id="50778-159">You can complete the following hierarchy operations to enable the creation of a work hierarchy or decomposition for a project.</span></span> 

<span data-ttu-id="50778-160">**新しいタスク** 作成したすべての新しいタスクは、ルート ノードの下に自動的に追加されます。また、WBS 番号は自動的にタスクに割り当てられます。</span><span class="sxs-lookup"><span data-stu-id="50778-160">**New task** Any new task that you create is automatically added under the root node, and a WBS number is automatically assigned to the task.</span></span> <span data-ttu-id="50778-161">WBS 番号は、階層上のタスクのレベルを表します。</span><span class="sxs-lookup"><span data-stu-id="50778-161">The WBS number represents the task’s level in the hierarchy.</span></span> <span data-ttu-id="50778-162">プロジェクト ルート タスクの下の最初のレベルのタスクには、1、2、3 などの番号付けスキームが使用されます。</span><span class="sxs-lookup"><span data-stu-id="50778-162">For tasks in the first level under the project root task, a numbering scheme of 1, 2, 3, and so on is used.</span></span> <span data-ttu-id="50778-163">最初のレベルの下にあるタスクには、1.1、1.2、1.3 などの番号付けスキームが使用されます。</span><span class="sxs-lookup"><span data-stu-id="50778-163">For tasks that are under the first level, a numbering scheme of 1.1, 1.2, 1.3, and so on is used.</span></span> <span data-ttu-id="50778-164">前のレベルの下位に追加された各レベルには、新たに点を打たれた一連の番号が追加されます。</span><span class="sxs-lookup"><span data-stu-id="50778-164">For each level that is added under a previous level, a new dotted series of numbers is added.</span></span> 

<span data-ttu-id="50778-165">現在、WBS の番号付けをカスタマイズすることはできません。</span><span class="sxs-lookup"><span data-stu-id="50778-165">Currently, you can’t customize the WBS numbering.</span></span> 

<span data-ttu-id="50778-166">**インデント タスク** タスクをインデントすると、それより前にあるタスクの子になります。</span><span class="sxs-lookup"><span data-stu-id="50778-166">**Indent task** When you indent a task, it becomes a child of the task that precedes it.</span></span> <span data-ttu-id="50778-167">新しい子のタスクの WBS 番号は、自動的に新しい親の WBS 番号に基づいて再計算されます。</span><span class="sxs-lookup"><span data-stu-id="50778-167">The WBS number of the new child task is automatically recalculated based on the WBS number of its new parent.</span></span> <span data-ttu-id="50778-168">親タスクは、これで集計またはコンテナ タスクになり、それにより、それ自体の構造タスクのロールアップになります。</span><span class="sxs-lookup"><span data-stu-id="50778-168">The parent task is now a summary or container task, and therefore becomes a roll-up of its constituent tasks.</span></span> 

> [!NOTE]

### <a name="how-to-use-the-concepts-of-planned-value-earned-value-and-actual-cost"></a><span data-ttu-id="50778-325">計画値、達成額、実績原価の概念の使用方法</span><span class="sxs-lookup"><span data-stu-id="50778-325">How to use the concepts of planned value, earned value, and actual cost</span></span>

<span data-ttu-id="50778-326">**スケジュール差異** 計画時に、タイムラインの作業の予測を作成します。</span><span class="sxs-lookup"><span data-stu-id="50778-326">**Schedule variance** During planning, you create a forecast for work on a timeline.</span></span> <span data-ttu-id="50778-327">したがって、計画値は、プロジェクトのプランナーが作業がプロジェクトで完了すると考えた率です。</span><span class="sxs-lookup"><span data-stu-id="50778-327">Therefore, planned value is the rate at which project planners thought work would be completed on the project.</span></span> <span data-ttu-id="50778-328">プロジェクトの進行後、作業は完了し、プロジェクトは値を取得します。</span><span class="sxs-lookup"><span data-stu-id="50778-328">After a project is in progress, work is completed, and the project earns value.</span></span> <span data-ttu-id="50778-329">達成額に対する計画値を比較することにより、プロジェクトの作業がどのように進行しているかを確認できます。</span><span class="sxs-lookup"><span data-stu-id="50778-329">By comparing planned value to earned value, you can view how work on a project is progressing.</span></span> <span data-ttu-id="50778-330">この比較の結果は、スケジュール差異と呼ばれます。</span><span class="sxs-lookup"><span data-stu-id="50778-330">The result of this comparison is called schedule variance.</span></span> 

<span data-ttu-id="50778-331">ある期間の計画値が達成額を超える場合、プロジェクトで実行される作業量は、予定された作業量より少なくなります。</span><span class="sxs-lookup"><span data-stu-id="50778-331">If the planned value for a period is more than the earned value, the amount of work that has been done on a project is less than what was planned.</span></span> <span data-ttu-id="50778-332">したがって、プロジェクトのスケジュールは遅れています。</span><span class="sxs-lookup"><span data-stu-id="50778-332">Therefore, the project is behind schedule.</span></span> <span data-ttu-id="50778-333">計画値と達成額が金銭条件で示されるため、プロジェクトのラグ タイムも、金銭価値を与えられます。</span><span class="sxs-lookup"><span data-stu-id="50778-333">Because planned value and earned value are expressed in monetary terms, the lag time on the project is also given a monetary value.</span></span>
