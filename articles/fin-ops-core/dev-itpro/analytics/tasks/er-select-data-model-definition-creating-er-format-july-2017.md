# <a name="select-data-model-definitions-when-you-create-formats"></a>形式作成時のデータ モデル定義の選択

[!include [banner](../../includes/banner.md)]

この手順にあるステップを完了するには、まず「ER コンフィギュレーション プロバイダーの作成および有効なプロバイダーとしてのマーク付け」にある手順を完了する必要があります。 

この手順では、電子ドキュメントの生成に設計された電子レポート（ER）形式の構成を挿入するためのデータ モデル定義として、モデルのルート項目を選択する方法を示します。 この手順では、サンプル会社 Litware、Inc. 用の新しい ER 形式コンフィギュレーションを追加します。 

この手順は、「システム管理者」または「電子レポート開発者」ロールが割り当てられているユーザーを対象としています。 ステップは、どのデータ セットを使用しても完了することができます。

1. [組織管理] > [ワークスペース] > [電子申告] の順に移動します。
    * サンプル会社 Litware, Inc. のコンフィギュレーション プロバイダーが使用可能であり、アクティブとしてマークされていることを確認します。 この構成プロバイダーが表示されない場合は、「構成プロバイダーを作成し、有効としてマークする」 に記載の手順を完了しておく必要があります。  
2. [コンフィギュレーションをレポートする] をクリックします。

## <a name="add-a-new-er-data-model-configuration"></a>新しい ER データ モデル コンフィギュレーションを追加する
1. [コンフィギュレーションの作成] をクリックすると、ドロップ ダイアログが開きます。
    * 生成 ER レポートのデータ ソースとして使用するように設計されたデータ モデルを含む新しい ER モデル コンフィギュレーションを追加します。  
2. [名前] フィールドに、「支払モデル (架空)」と入力します。
    * 支払モデル (架空)  
3. [コンフィギュレーションの作成] をクリックします。
4. [デザイナー] をクリックします。
    * ER デザイナーを開いて、このコンフィギュレーションのデータ モデルの構造を指定します。  
    * 口座振替と口座引落という 2 つの支払方法をサポートするために、支払業務ドメインのデータ モデルを設計すると仮定します。  
5. [新規] をクリックすると、ドロップ ダイアログが開きます。
6. [名前] フィールドに、「支払 – 口座振替」と入力します。
    * 支払 – 口座振替  
7. [追加] をクリックします。
8. [新規] をクリックすると、ドロップ ダイアログが開きます。
9. [新しいノード] フィールドに、「モデル ルート」を入力します。
10. [名前] フィールドに、「支払 – 口座引落」と入力します。
    * 支払 – 口座引落  
11. [追加] をクリックします。
12. [保存] をクリックします。
13. ページを閉じます。
14. [状態の変更] をクリックします。
    * モデルのドラフト バージョンを完了して、新しいモデル マッピングと形式で使用できるようにします。  
15. [完了] をクリックします。
16. [OK] をクリックします。

## <a name="start-to-enter-a-new-er-format-configuration"></a>新しい ER 形式コンフィギュレーションの入力を開始する
1. [コンフィギュレーションの作成] をクリックすると、ドロップ ダイアログが開きます。
2. [新規] フィールドで、「支払モデル (架空) のデータ モデルに基づいた書式」と入力します。
3. [データモデル定義] フィールドで、値を入力または選択します。
    * 選択したデータ モデルのすべてのルート項目が、現在データ モデル定義として選択可能であることに注意してください。 データ モデルの必要なルート項目のいずれかを使用して、形式の設計を続行することができます。 選択したルート項目のモデル マッピングが存在しなくても、続行は妨げられません。  
4. ページを閉じます。

## <a name="add-a-new-er-model-mapping-configuration"></a>新しい ER モデル マッピング コンフィギュレーションを追加する
1. [コンフィギュレーションの作成] をクリックすると、ドロップ ダイアログが開きます。
2. [新規] フィールドで、「支払モデル (架空) のデータ モデルに基づいたモデル マッピング」と入力します。
3. [名前] フィールドに、「支払モデル マッピング (架空)」と入力します。
    * 支払モデル マッピング (架空)  
4. [データモデル定義] フィールドで、値を入力または選択します。
5. [コンフィギュレーションの作成] をクリックします。

## <a name="design-er-model-mappings"></a>ER モデル マッピングを設計する
1. [デザイナー] をクリックします。
    * ER デザイナーを使用して、必要なルート項目のモデル マッピングを指定します。  
2. [デザイナー] をクリックします。
    * 選択されたモデルのルート項目の選択されたモデル マッピングの設定をシミュレートします。  
3. ツリーで、'Dynamics 365 for Operations\Table records'を選択します。
4. [ルートの追加] をクリックします。
5. [名前] フィールドに、「元帳」と入力します。
6. [テーブル] フィールドで、「LedgerJournalTrans」と入力します。
    * LedgerJournalTrans  
7. [OK] をクリックします。
8. [保存] をクリックします。
9. ページを閉じます。
10. ページを閉じます。

## <a name="start-to-enter-another-new-er-format-configuration"></a>別の新しい ER 形式コンフィギュレーションの入力を開始する
1. ツリーで、「支払モデル (架空)」を選択します。
2. [コンフィギュレーションの作成] をクリックすると、ドロップ ダイアログが開きます。
3. [新規] フィールドで、「支払モデル (架空) のデータ モデルに基づいた書式」と入力します。
4. [データモデル定義] フィールドで、値を入力または選択します。
    * アプリケーション データ ソースにマップするのに、現在 1 つのみのルート項目が使用可能であることに注意してください。 少なくとも 1 つのモデル マッピングが導入された場合、ER 形式が追加されている間は、アプリケーションのデータソースにマッピングされているモデルのルート項目のみをモデル定義として選択することができます。   
5. ページを閉じます。
