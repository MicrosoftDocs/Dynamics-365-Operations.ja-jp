<span data-ttu-id="3a674-105">ロールベース セキュリティでは、個々のユーザーではなく、セキュリティ ロールに対してのみアクセスを許可します。</span><span class="sxs-lookup"><span data-stu-id="3a674-105">In role-based security, access is not granted to individual users, only to security roles.</span></span> <span data-ttu-id="3a674-106">ユーザーはロールに割り当てられます。</span><span class="sxs-lookup"><span data-stu-id="3a674-106">Users are assigned to roles.</span></span> <span data-ttu-id="3a674-107">セキュリティ ロールに割り当てられているユーザーは、そのロールに関連付けられている一連の権限にアクセスできます。</span><span class="sxs-lookup"><span data-stu-id="3a674-107">A user who is assigned to a security role has access to the set of privileges that is associated with that role.</span></span> <span data-ttu-id="3a674-108">任意のロールに割り当てられていないユーザーには権限がありません。</span><span class="sxs-lookup"><span data-stu-id="3a674-108">A user who is not assigned to any role has no privileges.</span></span>

<span data-ttu-id="3a674-112">自動ロール割り当てのルールを設定できるため、ユーザーの責任が変更されるたびに管理者が関与する必要はありません。</span><span class="sxs-lookup"><span data-stu-id="3a674-112">Because rules can be set up for automatic role assignment, the administrator does not have to be involved every time that a user's responsibilities change.</span></span> <span data-ttu-id="3a674-113">セキュリティ ロールとルールが設定された後、業務管理者は、業務データに基づく日常的なユーザーのアクセスを制御できます。</span><span class="sxs-lookup"><span data-stu-id="3a674-113">After security roles and rules have been set up, business managers can control day-to-day user access based on business data.</span></span>

<a name="security-roles"></a><span data-ttu-id="3a674-123">セキュリティ ロール</span><span class="sxs-lookup"><span data-stu-id="3a674-123">Security roles</span></span>
--------------

<span data-ttu-id="3a674-124">すべてのユーザーは Finance and Operations にアクセスするため少なくとも 1 つのセキュリティ ロールに割り当てられている必要があります。</span><span class="sxs-lookup"><span data-stu-id="3a674-124">All users must be assigned to at least one security role in order to have access to Finance and Operations.</span></span> <span data-ttu-id="3a674-125">ユーザーに割り当てられたセキュリティ ロールにより、ユーザーが実行できる職務と、ユーザーが表示できるユーザー インターフェイスの部分が決まります。</span><span class="sxs-lookup"><span data-stu-id="3a674-125">The security roles that are assigned to a user determine the duties that the user can perform and the parts of the user interface that the user can view.</span></span> 

<span data-ttu-id="3a674-126">管理者は、データ セキュリティ ポリシーを適用して、ロールのユーザーがアクセス権を持つデータを制限します。</span><span class="sxs-lookup"><span data-stu-id="3a674-126">Administrators can apply data security policies to limit the data that the users in a role have access to.</span></span> <span data-ttu-id="3a674-127">たとえば、ロールのユーザーは、単一の組織からのみ、データへアクセスできる場合があります。</span><span class="sxs-lookup"><span data-stu-id="3a674-127">For example, a user in a role may have access to data only from a single organization.</span></span> <span data-ttu-id="3a674-128">管理者は、ロールのユーザーが持つ、過去、現在、および将来のレコードに対するアクセスのレベルも指定できます。</span><span class="sxs-lookup"><span data-stu-id="3a674-128">The administrator can also specify the level of access that the users in a role have to current, past, and future records.</span></span> <span data-ttu-id="3a674-129">たとえば、ロールでのユーザーは、すべての期間のレコードを表示できるよう特権を割り当てられますが、現在の期間のみのレコードを変更できます。</span><span class="sxs-lookup"><span data-stu-id="3a674-129">For example, users in a role can be assigned privileges that allow them to view records for all periods, but that allow them to modify records only for the current period.</span></span> 

<span data-ttu-id="3a674-130">セキュリティ ロールによるアクセスを管理することで、ユーザーごとに個別にアクセスを管理する必要がないため管理者は時間を節約できます。</span><span class="sxs-lookup"><span data-stu-id="3a674-130">By managing access through security roles, administrators save time because they do not have to manage access separately for each user.</span></span> <span data-ttu-id="3a674-131">セキュリティ ロールは、すべての組織で 1 回定義されます。</span><span class="sxs-lookup"><span data-stu-id="3a674-131">Security roles are defined one time for all organizations.</span></span> <span data-ttu-id="3a674-132">さらに、ユーザーを業務データに基づくロールに自動的に割り当てることができます。</span><span class="sxs-lookup"><span data-stu-id="3a674-132">In addition, users can be automatically assigned to roles based on business data.</span></span> <span data-ttu-id="3a674-133">たとえば、管理者は、セキュリティ ロールで人事管理の職位に関連付けるルールを設定できます。</span><span class="sxs-lookup"><span data-stu-id="3a674-133">For example, the administrator can set up a rule that associates a Human resources position with a security role.</span></span> <span data-ttu-id="3a674-134">ユーザーがその位置に割り当てられるたびに、それらのユーザーは適切なセキュリティ ロールに自動的に追加されます。</span><span class="sxs-lookup"><span data-stu-id="3a674-134">Any time that users are assigned to that position, those users are automatically added to the appropriate security roles.</span></span> 

<span data-ttu-id="3a674-135">セキュリティ ロールは、階層構造に編成することができます。</span><span class="sxs-lookup"><span data-stu-id="3a674-135">Security roles can be organized into a hierarchy.</span></span> <span data-ttu-id="3a674-136">ロール階層を使用すると、管理者は別のロールに基づいてロールを定義できます。</span><span class="sxs-lookup"><span data-stu-id="3a674-136">The role hierarchy allows the administrator to define a role based on another role.</span></span> <span data-ttu-id="3a674-137">たとえば、販売マネージャー ロールは、マネージャー ロールおよび営業担当者ロールの親ロールとして定義されます。</span><span class="sxs-lookup"><span data-stu-id="3a674-137">For example, the sales manager role could be defined as a parent role of the manager role and the salesperson role.</span></span> <span data-ttu-id="3a674-138">親ロールは、職務、権限、およびその子ロールに割り当てられている条件を自動的に継承します。</span><span class="sxs-lookup"><span data-stu-id="3a674-138">A parent role automatically inherits the duties, privileges, and conditions that are assigned to its child roles.</span></span> <span data-ttu-id="3a674-139">したがって、親ロールに割り当てられているユーザーは、子ロールのユーザーが実行できるすべてのタスクを実行できます。</span><span class="sxs-lookup"><span data-stu-id="3a674-139">Therefore, a user who is assigned to the parent role can perform all of the tasks that users in the child roles can perform.</span></span> <span data-ttu-id="3a674-140">ロールは、1 つ以上の子ロールまたは 1 つ以上の親ロールを持つことができます。</span><span class="sxs-lookup"><span data-stu-id="3a674-140">A role can have one or more child roles or one or more parent roles.</span></span> 

<span data-ttu-id="3a674-141">既定では、サンプルのセキュリティ ロールが提供されています。</span><span class="sxs-lookup"><span data-stu-id="3a674-141">By default, sample security roles are provided.</span></span> <span data-ttu-id="3a674-142">すべての機能は少なくとも 1 つのサンプル セキュリティ ロールに関連付けられています。</span><span class="sxs-lookup"><span data-stu-id="3a674-142">All functionality is associated with at least one of the sample security roles.</span></span> <span data-ttu-id="3a674-143">管理者は、サンプル セキュリティ ロールにユーザーを割り当てるか、企業のニーズに合わせてサンプル セキュリティ ロールを変更するか、または新しいセキュリティ ロールを作成することができます。</span><span class="sxs-lookup"><span data-stu-id="3a674-143">The administrator can assign users to the sample security roles, modify the sample security roles to fit the needs of the business, or create new security roles.</span></span> <span data-ttu-id="3a674-144">既定では、サンプル ロールは階層内に配置されません。</span><span class="sxs-lookup"><span data-stu-id="3a674-144">By default, the sample roles are not arranged in a hierarchy.</span></span>

<span data-ttu-id="3a674-167">フォームまたはサービスなどの各機能は、エントリ ポイントを使ってアクセスします。</span><span class="sxs-lookup"><span data-stu-id="3a674-167">Each function, such as a form or a service, is accessed through an entry point.</span></span> <span data-ttu-id="3a674-168">メニュー項目、Web コンテンツ項目、およびサービス工程は、総称してエントリ ポイントと呼ばれます。</span><span class="sxs-lookup"><span data-stu-id="3a674-168">Menu items, web content items, and service operations are referred to collectively as entry points.</span></span> 

<span data-ttu-id="3a674-169">セキュリティ モデルでは、セキュリティ保護可能なオブジェクトと機能を実行するために必要なアクセス レベルは権限によってグループ化されます。</span><span class="sxs-lookup"><span data-stu-id="3a674-169">In the security model, permissions group the securable objects and access levels that are required to run a function.</span></span> <span data-ttu-id="3a674-170">これには、エントリ ポイントからアクセスされるテーブル、フィールド、フォーム、またはサーバー側のメソッドが含まれます。</span><span class="sxs-lookup"><span data-stu-id="3a674-170">This includes any tables, fields, forms, or server side methods that are accessed through the entry point.</span></span>
