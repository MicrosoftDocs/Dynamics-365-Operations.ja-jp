# <a name="identify-and-resolve-conflicts-in-segregation-of-duties"></a>職務分掌の競合の識別と解決

[!include [banner](../../includes/banner.md)]

この記事では、職務分掌の競合の識別と解決について説明します。 異なるユーザーが実行する必要がある職務を分割するルールを設定できます。 この概念は職務分掌と呼ばれます。 セキュリティ ロールの定義またはユーザーのロール割り当てがルールに違反する場合、競合がログに記録されます。 すべての競合は、管理者が解決する必要があります。 競合を識別および解決するには、次の手順を実行します。

ルールを追加した後、既存のすべてのロールが準拠している必要があります。 

## <a name="verify-that-existing-roles-and-duties-comply-with-new-rules-for-segregation-of-duties"></a>既存のロールと職務の新しい職務分掌のルールに準拠しているかどうかの確認
1. **システム管理** > **セキュリティ** > **職務分掌**  > **職務分掌ルール** の順に移動します。
3. **職務とロールの検証** を選択します。 ロールがルールに違反している場合、ルールの名前、ロール、競合する責務の名前を示すメッセージが表示されます。 競合するロールは **セキュリティ構成** を使用して変更する必要があります。また、競合する職務を含めすることはできません。 選択したルールに違反するロールがない場合、すべてのロールが遵守していることを示すメッセージが表示されます。   

> [!NOTE]
> この検証は、選択したルールに対するみ実行されます。 各ルールのコンプライアンスを検証することが重要です。   

役割を作成または変更すると、職務分掌のルールが自動的に適用されます。 競合する職務をロールに割り当てすることはできません。

次に、既存のすべてのロールの割り当てが準拠している必要があります。

## <a name="verify-that-user-role-assignments-comply-with-new-rules-for-segregation-of-duties"></a>ユーザー ロールの割り当てが職務分掌の新しいルールに準拠しているかどうかの確認
1. **システム管理 > セキュリティ > 職務分掌 > ユーザー ロール割り当ての遵守を確認** の順に移動します。
2. **OK** を選択します。 通知には検証結果が表示されます。 競合は、**職務分掌の未解決の競合** ページにも記録されます。   

ユーザーをロールに割り当てると、職務分掌のルールが自動的に適用されます。 競合する職務を含むロールにユーザーを割り当てると、エラー メッセージが表示されます。 その後、競合を解決するには、追加のロールの割り当てを拒否または許可する必要があります。 追加のロールは、割り当てが許可された後に割り当てられます。 

> [!NOTE]
> Active Directory ドメイン グループに基づいてロールが割り当てられているユーザーの競合は検証されません。

## <a name="view-and-resolve-conflicting-user-role-assignments"></a>競合しているユーザー ロールの割り当ての表示と解決
1. **システム管理** > **セキュリティ** > **職務分掌** > **職務分掌の未解決の競合** へ移動します。 
2. 競合を選択して、次のアクションのいずれかを選択します。 

  - **割り当てを拒否**: これにより、追加のセキュリティ ロールに対するユーザーの割り当てが拒否されます。 自動ロール割り当てを拒否すると、ユーザーはロールから除外としてマークされます。 除外されたユーザーは、ロールと関連付けられているアクセス権を付与されません。また、管理者が除外を削除するまでそのユーザーをロールに割り当てることはできません。 
-  **割り当ての許可**: これは競合を上書きし、ユーザーが追加のセキュリティ ロールに割り当てられることを許可します。 競合を上書きする場合は、**上書きの理由** フィールドに理由を入力する必要があります。 上書きされた役割の割り当てはすべて、**職務分掌の競合** ページで表示できます。  

> [!NOTE]
> 同じユーザーについて複数の競合が表示される場合は、ユーザー レコードを選択し、**ユーザー** ページで割り当てられたロール を評価します。 この競合を回避するには、各ルールを追加または変更した後で検証します。
