## <a name="determine-which-invoice-matching-validation-to-use"></a><span data-ttu-id="24ab3-110">使用する請求書照合の検証の決定</span><span class="sxs-lookup"><span data-stu-id="24ab3-110">Determine which invoice matching validation to use</span></span>
<span data-ttu-id="24ab3-111">4 つの異なるタイプの照合の検証が使用可能です。</span><span class="sxs-lookup"><span data-stu-id="24ab3-111">Four different types of matching validation are available.</span></span> 

- <span data-ttu-id="24ab3-112">**明細行レベルの照合** – 最もよく使用する照合のタイプは明細行の照合です。</span><span class="sxs-lookup"><span data-stu-id="24ab3-112">**Line level matching** – The most common type of matching is line matching.</span></span> <span data-ttu-id="24ab3-113">明細行レベルの照合は、ツーウェイまたはスリーウェイ マッチングになります。</span><span class="sxs-lookup"><span data-stu-id="24ab3-113">Line level matching can be two-way or three-way matching.</span></span> <span data-ttu-id="24ab3-114">既定の明細行レベルの照合は、**買掛金管理パラメーター** ページで法人に対して指定できます。</span><span class="sxs-lookup"><span data-stu-id="24ab3-114">The default line level matching can be specified for a legal entity on the **Accounts payable parameter** page.</span></span> <span data-ttu-id="24ab3-115">ツーウェイ マッチングは、請求書の単価を発注書の単価と照合します。</span><span class="sxs-lookup"><span data-stu-id="24ab3-115">Two-way matching compares the unit price of the invoice to the unit price of the purchase order.</span></span> <span data-ttu-id="24ab3-116">スリーウェイ マッチングは、さらに請求書数量を一致する製品受領書の数量と比較します。</span><span class="sxs-lookup"><span data-stu-id="24ab3-116">Three-way matching additionally compares the invoice quantity to the matched product receipt quantity.</span></span>
- <span data-ttu-id="24ab3-117">**請求合計照合** – 請求書の合計額を発注書の合計金額と照合します。</span><span class="sxs-lookup"><span data-stu-id="24ab3-117">**Invoice totals matching** – Match the total amounts on the invoice to the total amounts on the purchase order.</span></span> <span data-ttu-id="24ab3-118">このタイプの請求書照合では、最小限の詳細情報で処理します。したがって、請求書照合情報を確認するためのスタッフの所用時間が最小限で済むようにコントロールを設定するときは、このオプションを使用します。</span><span class="sxs-lookup"><span data-stu-id="24ab3-118">This type of invoice matching includes the least amount of detail, so you can use this option to set up controls that minimize the staff time that is required to review invoice matching information.</span></span> <span data-ttu-id="24ab3-119">小計、割引合計、請求金額、売上税、丸め、および請求金額を含む、6 つの合計が比較されます。</span><span class="sxs-lookup"><span data-stu-id="24ab3-119">Six totals are compared, including Subtotal, Total discount, Charges, Sales taxes, Round off, and Invoice amount.</span></span> <span data-ttu-id="24ab3-120">請求書のこれらの値のいずれかが許容差を超えて予測金額から逸脱している場合は、システムによって検証されます。</span><span class="sxs-lookup"><span data-stu-id="24ab3-120">The system will validate if any of these values on the invoice deviate from expected amounts by more than an acceptable variance.</span></span>
- <span data-ttu-id="24ab3-121">**料金照合** – 請求書の請求金額情報 (数量) を発注書の請求金額情報 (数量) と照合します。</span><span class="sxs-lookup"><span data-stu-id="24ab3-121">**Charges matching** – Match the charges information (amounts) on the invoice to the charges information (amounts) on the purchase order</span></span>
- <span data-ttu-id="24ab3-122">**明細行品目の価格合計照合** – このタイプの照合は通常、1 つの購買注文明細行に対して複数の請求書を受け取る会社の役に立ちます。</span><span class="sxs-lookup"><span data-stu-id="24ab3-122">**Price totals for line item matching** – This type of matching is useful for companies that typically receive multiple invoices for a single purchase order line.</span></span> <span data-ttu-id="24ab3-123">通常、購買注文明細行に対して請求書を 1 つだけ受け取る場合、このタイプの照合は必要ありません。</span><span class="sxs-lookup"><span data-stu-id="24ab3-123">If you typically receive only one invoice per purchase order line, this type of matching is not necessary.</span></span> <span data-ttu-id="24ab3-124">この照合には、ツーウェイまたはスリーウェイ マッチングが有効になっており、許容範囲の割合と金額に基づいて、正味金額の上限の検証を行わないように機能している必要があります。</span><span class="sxs-lookup"><span data-stu-id="24ab3-124">This matching requires that two-way or three-way matching is enabled and serves as a not-to-exceed net amount validation, based on a tolerance percentages and amounts.</span></span>  <span data-ttu-id="24ab3-125">このタイプの照合は、請求書の各明細行の正味金額の価格情報、保留中および転記済みの請求書明細行すべてを、対応する発注書明細行の正味金額と比較します。</span><span class="sxs-lookup"><span data-stu-id="24ab3-125">This type of matching compares price information for the net amount of each line on the invoice, and all pending and previously posted invoice lines, with the net amount of the corresponding purchase order line.</span></span> <span data-ttu-id="24ab3-126">この正味金額は、(単価 \* 行数) + 行請求金額 - 行割引という式で計算します。</span><span class="sxs-lookup"><span data-stu-id="24ab3-126">The net amount is determined by the following formula: (Unit price \* Line quantity) + Line charges - Line discounts.</span></span> <span data-ttu-id="24ab3-127">価格合計をパーセントで照合する場合、取引通貨を使用して値が比較されます。</span><span class="sxs-lookup"><span data-stu-id="24ab3-127">When matching price totals by percentage, the system compares values using the transaction currency.</span></span> <span data-ttu-id="24ab3-128">価格合計を量で照合する場合、会計通貨を使用して値が比較されます。</span><span class="sxs-lookup"><span data-stu-id="24ab3-128">When matching price totals by amount, the system compares the values using the accounting currency.</span></span>
