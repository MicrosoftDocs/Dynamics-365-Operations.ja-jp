# <a name="invoice-capture-solution-workspace"></a>Invoice Capture ソリューションのワークスペース

[!include [banner](../includes/banner.md)]
[!include [preview banner](../includes/preview-banner.md)]

## <a name="side-by-side-viewer-for-the-invoice-capture-solution"></a>Invoice capture ソリューションを並べて表示するビューアー

事務員は正しい情報を収集するために複数の添付ファイルとウィンドウを移動する必要があるため、請求書をシステムに入力することは通常ミスしやすい手間のかかるプロセスでした。 並べて表示するドキュメント ビューアーを使うと、プロセスをより簡単、効率的、およびより正確にすることで、請求書を処理する際の経験を向上できます。

並べて表示するドキュメント ビューアーでは、ユーザーが元のドキュメントを使用し、請求書を同じウィンドウに並べて開くことができます。 請求書ページには、元の請求書ドキュメントから取得された情報が入力されます。 ビューアーは、請求書ページのフィールドと元の請求書ドキュメントの間をつなぎます。 ビューアーでは、ユーザーが請求書ページに存在するエラーを見つけやすくなります。

### <a name="open-the-side-by-side-viewer"></a>並べて表示するビューアーを開きます

Microsoft Dynamics 365 Finance では、集計ページの一覧または請求書リスト ページから並べて表示するビューアーを開くことができます。 レコードをダブルタップ (またはダブルクリック) するか、請求書番号を選択して開くことができます。

### <a name="using-the-side-by-side-viewer"></a>並べて表示するビューアーの使用

#### <a name="manually-review-an-invoice"></a>請求書を手動でレビューする

インポートされた請求ドキュメントで、エラーや警告があるために手動での確認する必要がある場合があります。 並べて表示するビューアーで、ドキュメント ヘッダーには **インポート済み** というステータスが表示され、現在のバージョンは **元のバージョン** になります。

請求書のレビューを開始するには、**レビューの開始** を選択します。 すべてのフィールドは編集可能になります。 **ステータス** フィールドは **レビュー中** に更新され、**現在のバージョン** フィールドは **変更済バージョン** に更新されます。

#### <a name="view-and-work-with-messages"></a>メッセージの表示および処理

ユーザーは、メッセージ ウィンドウからレビュー プロセスを開始する必要があります。 エラー メッセージは赤い X で示され、警告メッセージは三角形によって、情報メッセージは円で示されます。 信頼性スコア関連のメッセージは、構成グループによって設定されたしきい値に応じて、警告またはエラーとして分類できます。 詳細については、[Invoice capture ソリューションの構成グループ](invoice-capture-config-group.md)を参照してください。

警告メッセージおよびエラー メッセージは、メッセージ ウィンドウ、請求書ヘッダー、または請求明細行で無視できます。 メッセージが無視された後は、エラーまたは警告として表示されなくなり、請求書の検証に失敗することもありません。

- メッセージ ペインからのメッセージを無視するには、**無視** を選択します。 無視されたメッセージをリセットするには、**無視** を再度選択します。 そのタイプは、エラーまたは警告から情報に変更されます。
- 請求書ヘッダーまたは請求明細からのメッセージを無視するには、このフィールドで **無視** を選択します。 メッセージ記号が表示されなくなります。 ただし、メッセージがメッセージ ペインからリセットされた場合は再表示されます。

請求書ヘッダー フィールドに関連するメッセージの場合、メッセージ ペインでメッセージを選択すると、ヘッダー セクションの対応するフィールドにカーソルが移動します。

#### <a name="proofread-and-edit-fields"></a>フィールドの校正と編集

フィールドの値が光学式文字認識 (OCR) で元の請求書から読み取られる場合、フィールドに記号が表示されます。 記号を選択すると、ドキュメント ビューアーがズームインし、フィールド値の読み取りを行う場所を強調表示して、請求書データを検証できます。

ドキュメント ビューアーを元の拡大率にリセットするには、次のいずれかの手順に従います。

- 以前に選択した記号と同じ記号を選択します。
- ドキュメント ビューアーの右上隅にあるボタンを選択します。

必要に応じてフィールドを編集します。 カーソルがフィールドから移動すると、編集内容が自動的に保存されます。 フィールドの右側の記号は、そのフィールドが手動で更新された状態を示します。 ページが更新されると、記号が削除されます。

#### <a name="check-an-invoice-to-get-up-to-date-messages"></a>請求書をチェックして最新のメッセージを取得します

フィールドを編集するとフィールドの値は更新されますが、新しい検証メッセージは生成されません。 最新の検証メッセージを取得するには、**チェック** を選択します。 メッセージ ペイン、請求書ヘッダー、および請求明細行のメッセージが更新されます。

#### <a name="complete-the-review"></a>レビューの完了

レビューを完了するには、**レビューの完了** を選択します。 請求書が検証されます。 エラーが見つかった場合は、ドキュメントのステータスが **確認中** の状態で、メッセージ バーが表示されます。 メッセージ ペインおよび請求書ヘッダーおよび行のすべてのメッセージは、検証に失敗した原因に関する情報を提供するために自動的に更新されます。

すべてのブロック エラーを固定した時点で、レビューを完了できます。 ドキュメントのステータスが **レビュー済** に更新され、フィールドを編集できなくなります。 **レビューの開始** を再選択することで、レビューを再開できます。

#### <a name="generate-a-pending-vendor-invoice-in-finance"></a>Finance での保留中仕入先請求書の生成

接続されている Finance 環境に請求書ドキュメントを送信するには、**生成** を選択します。 請求書の生成が失敗した場合は、メッセージ バーにエラー メッセージが表示されます。

#### <a name="void-an-invoice"></a>請求書の無効化

請求書を無効するには、**無効** を選択します。 無効にした請求書は確認できないし、**請求書は手動レビューが必要です** リストに表示されません。

### <a name="validation-logic"></a>検証ロジック

並べて表示するビューアーの一部の重要なフィールドは、請求書のステージング データには存在しませんが、Finance で保留中の請求書を生成するために必要です。 これらのフィールドは、現在の請求書ステージング データと Finance からのマスター データの組み合わせから派生します。

派生する必要があるフィールドは、**法人**、**仕入先勘定**、および **品目番号** です。 これらの派生は次の順序で行う必要があります。 フィールドの派生に失敗すると、プロセスは停止します。

1. **法人** - 法人が最初に派生します。 有効なマッピング ルールが法人に対して検出された場合、法人は会社名と会社住所に基づいて派生します。
2. **仕入先勘定** - 次に、有効なマッピング ルールと、派生法人と仕入先の名前と仕入先の住所の組み合わせに基づいて仕入先勘定が派生します。
3. **品目番号** - 最後に、品目名は、次の 3 種類の情報に基づいてステージングから派生します。

    - 構成されたマッピング ルール
    - 派生した法人
    - 派生した仕入先勘定

検証を実行するには、並べて表示されたビューアーで **チェック** を選択します。 現在、検証では次のチェックを実行します。

- **必須のチェック** - このチェックでは、並べて表示されたビューアーの必須フィールドが検証されます。 ユーザーは、**構成の設定** ページで必須に設定するフィールドを選択できます。
- **信頼スコア** - ユーザーは、信頼スコアの警告のしきい値およびエラーのしきい値を設定できます。 このチェックでは、しきい値を下回っている OCR からの信頼スコアに焦点を当てることができます。 検証結果に基づいてエラー メッセージまたは警告メッセージが表示されます。
- **法人** - このチェックは、法人が Finance を使用している場合に検証します。 Finance 環境に法人が存在しない場合、検証は失敗します。

並べて表示するビューアーを初めて使用し、ユーザーが **確認** を選択すると、派生プロセスと検証プロセスが実行されます。 請求書が正確な場合は、ユーザーが **確認の完了** を選択すると検証プロセスが実行されます。 また、ユーザーが **仕入先請求書の生成** を選択した場合にも実行されます。

派生プロセスは検証プロセスの前に発生し、すべての警告またはエラーが検証プロセスから発生します。 警告とエラーは Finance に記録されます。
