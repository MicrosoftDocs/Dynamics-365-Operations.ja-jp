# <a name="set-up-lease-approval-workflows"></a>リース承認ワークフローの設定

[!include [banner](../includes/banner.md)]

このトピックでは、新しいリースが作成されたときに実行される承認ワークフローの設定方法について説明します。 ワークフローの使用方法の詳細については、[リース承認ワークフローの使用](use-create-lease-wrkflw.md)を参照してください。 

1. **資産リース \> 設定 \> リース ワークフロー** の順に移動します。
2. **リース ワークフロー** ページで、**新規** を選択します。
3. 表示されるダイアログボックスの **ワークフロー タイプ** で、**リース ワークフロー** リンクを選択します。

    アプリケーションが開きます。 実行されたら、Azure Active Directory (Azure AD) にサインインします。ワークフロー アプリケーションにリダイレクトされます。

4. **リース ワークフローの承認** 要素をワークフローにドラッグします。
5. 1 つのノードを **開始** から **リース ワークフローの承認** に関連付けます。 次に、**リース ワークフローの承認** を **終了** に関連付けます。
6. **リース ワークフローの承認** をダブルクリックします。
7. **プロパティ** をクリックし、**基本設定** でワークフローの名前を入力します。

    このページでは、ワークフローに対してさらにパラメーターを設定することもできます。 **自動アクション** が有効になっている場合、システムによって自動的に特定のアクションが実行されます。 通知は、**通知** タブで指定されている場合に送信できます。**詳細設定** タブでは、最終承認者の指定、制限時間の設定、および完了する必要がある特定のアクションの指定を行うことができます。

8. すべてのワークフロー パラメーターの設定が終了したら **閉じる** を選択します。
9. **ステップ 1** を選択し、次に **プロパティ** を選択します。
10. **基本設定** で、ステップの名前を入力し、承認の件名行を作成します。また、承認の指示を指定します。
11. **割り当て** ページで、割り当てタイプを選択します。
12. 特定のユーザーを承認に割り当てるには、**ユーザー** を選択し、リースを承認するユーザーを選択して、**閉じる** を選択します。
13. **保存して終了** を選択して、ワークフローを作成します。 次に、メッセージが表示されたら、**OK** を選択します。
14. **ワークフローの作成** ページで、**閉じる** を選択します。
14. 新しいワークフローを選択し、**バージョン** を選択します。 **有効にする** をオンにして、ワークフローが有効になっていることを確認します。
15. **閉じる** を選択します。 新しい有効なバージョンが表示されます。
