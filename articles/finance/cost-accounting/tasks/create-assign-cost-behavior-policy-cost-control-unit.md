# <a name="create-and-assign-a-cost-behavior-policy-to-a-cost-control-unit"></a>原価動作ポリシーの作成と原価管理単位への割り当て

[!include [banner](../../includes/banner.md)]

原価の動作は、固定費または変動費の分類です。 ポリシーと対応ルールは、有効になるポリシーの原価管理単位へ割り当てる必要があります。 ポリシーを作成し、原価管理単位へ割り当てるには、この手順を使用します。


## <a name="create-a-cost-behavior-hierarchy"></a>原価動作階層を作成
1. [原価会計] > [分析コード] > [分析コード階層] に移動します。
2. [新規] をクリックします。
3. [作成] をクリックします。
4. [分析コード階層名] フィールドでは、「原価動作階層」を入力します。
5. [分析コード] フィールドで、値を入力または選択します。
    * [原価要素] を選択します。  
6. [保存] をクリックします。
7. [階層の表示] をクリックします。
8. [新規] をクリックします。
9. [ノード名] フィールドで、値を入力します。
    * [固定費] を入力します。  
10. ツリーで、「原価動作階層」を選択します。
11. [新規] をクリックします。
12. [ノード名] フィールドで、値を入力します。
    * [変動費] を入力します。  
13. [保存] をクリックします。
14. ツリーで、「原価動作階層\固定費」を選択します。
15. [新規] をクリックします。
16. 一覧で、選択された行をマークします。
17. [移動元分析コード メンバー] フィールドで、値を入力または選択します。
    * 分析コード メンバーの範囲にギャップが生じる可能性がありますが、メンバーとは重複できません。  
18. [移動先分析コード メンバー] フィールドでは、値を入力または選択します。
    * 分析コード メンバーの範囲にギャップが生じる可能性がありますが、メンバーとは重複できません。  
19. ツリーで、「原価動作階層\変動費」を選択します。
20. [新規] をクリックします。
21. 一覧で、選択された行をマークします。
22. [移動元分析コード メンバー] フィールドで、値を入力または選択します。
    * 分析コード メンバーの範囲にギャップが生じる可能性がありますが、メンバーとは重複できません。  
23. [移動先分析コード メンバー] フィールドでは、値を入力または選択します。
    * 分析コード メンバーの範囲にギャップが生じる可能性がありますが、メンバーとは重複できません。  
24. [保存] をクリックします。

## <a name="create-the-policy-and-rules"></a>ポリシーとルールを作成
1. [原価会計] > [ポリシー] > [原価動作ポリシー] へ移動します。
2. [新規] をクリックします。
3. [ポリシー名] フィールドで、値を入力します。
4. [原価要素分析コード階層] フィールドで、値を入力または選択します。
    * 作成したポリシーの階層を選択します。  
5. [原価オブジェクト分析コード階層] フィールドで、値を入力または選択します。
    * [組織] を選択します。  
6. [保存] をクリックします。
7. [新規] をクリックします。
8. 一覧で、選択された行をマークします。
9. [原価要素分析コード階層ノード] フィールドで、値を入力または選択します。
    * 階層を展開し、[変動費] を選択します。  
10. [原価オブジェクト分析コード階層ノード] フィールドで、値を入力または選択します。
    * 既定では、変動率は 100 パーセントです。  
11. 原価管理単位の [ポリシーの割り当て] をクリックします。
12. [新規] をクリックします。
13. 一覧で、選択された行をマークします。
14. [会計日から有効] フィールドで、値を入力します。
    * ルールには有効期限がありますが、もし新しいバージョンが作成された場合、ユーザーまたはシステムがルールを失効させることが出来ます。  
15. [原価管理単位] フィールドで、値を入力または選択します。
16. [保存] をクリックします。
