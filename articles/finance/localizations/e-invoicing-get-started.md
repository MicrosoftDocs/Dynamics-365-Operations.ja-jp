# <a name="get-started-with-electronic-invoicing"></a>電子請求を開始する

[!include [banner](../includes/banner.md)]

このトピックでは、電子請求を使い始めるための情報を提供します。 このトピックでは、Regulatory Configuration Services (RCS) と Dynamics 365 Finance に共通の設定手順を説明し、ビジネス ドキュメントを送信し、処理結果をレビューするために従うべき手順を提供します。

## <a name="prerequisites"></a>必要条件

このトピックの手順を完了する前に、以下の前提条件を満たす必要があります :

- Microsoft Dynamics Lifecycle Services (LCS)、Regulatory Configuration Service (RCS)、および Microsoft Dynamics 365 Finance または Dynamics 365 Supply Chain Management 環境を設定します。 詳細については、[電子請求サービスの管理を開始する](e-invoicing-get-started-service-administration.md)を参照してください。
- 組織の構成プロバイダを作成します。 詳細については、[コンフィギュレーション プロバイダーを作成して、アクティブとマークする](../../fin-ops-core/dev-itpro/analytics/tasks/er-configuration-provider-mark-it-active-2016-11.md)を参照してください。

## <a name="import-an-electronic-invoicing-feature-from-the-microsoft-configuration-provider"></a>Microsoft 構成プロバイダから電子請求機能をインポートする 

1. 規制構成サービス (RCS) のアカウントにログインします。
2. **グローバリゼーション機能** ワークスペースの **機能** セクションで、**電子請求** タイルを選択します。
3. **インポート** を選択し、 **同期する** を選択します。
4. **構成プロバイダ** 列 を **Microsoft** という語句で フィルタ処理します。
5. テーブルから電子請求機能の名前を選択し、**インポート** を選択します。

## <a name="create-an-electronic-invoicing-feature-under-your-organization-provider"></a>組織のプロバイダー配下に電子請求書発行機能を作成します

1. RCS の **グローバリゼーション機能** ワークスペースの **機能** セクションで、**電子請求** タイルを選択します。
2. **追加** > **既存の機能をベースにする** を選択し、**名前** フィールドで、 電子請求機能の名前を入力します。
3. **説明** フィールドに、機能の説明を入力します。
4. **基本機能** フィールドで、Microsoft 構成プロバイダからインポートされた電子請求機能を選択します。
5. **機能の作成** を選択します。

## <a name="country-specific-configuration-for-electronic-invoicing-feature"></a>電子機能の国別設定

国またはリージョンによって、電子請求機能には特定の設定が必要となります。 

具体的な手順については、国または地域向け 「使い方」 ドキュメントを参照してください。

## <a name="country-specific-configuration-of-application-setup"></a>アプリケーション設定の国別コンフィギュレーション

国またはリージョンによって、アプリケーション設定に特定の設定が必要となります。 

具体的な手順については、国または地域向け 「使い方」 ドキュメントを参照してください。

## <a name="deploy-the-electronic-invoicing-feature-to-service-environment"></a>サービス環境に電子請求機能をデプロイする

1. **バージョン** タブで、デプロイする電子請求機能のバージョンを選択します。
2. **ステータスの変更** \> **完了** を選択します。
3. **状態の変更** \> **公開** を選択します。
4. **デプロイ** を選択します。
5. **接続されたアプリケーションへデプロイする** オプションを **いいえ** に設定します。
6. **サービスの環境にデプロイする** オプション を **はい** に設定します。
7. **サービス環境** フィールドで、電子請求機能をデプロイする電子請求サービス環境を選択します。
8. **開始日** フィールドで、電子請求で電子請求が有効となる必要がある日付を選択します。
9. **OK** を選択します。

## <a name="deploy-the-electronic-invoicing-feature-to-connected-application"></a>接続されたアプリケーションに電子請求機能をデプロイする

1. **バージョン** タブで、デプロイする電子請求機能のバージョンを選択します。
2. **デプロイ** を選択します。
3. **接続されたアプリケーションにデプロイする** オプション を **はい** に設定します。
4. **アプリケーションの接続** フィールドで、Finance および Supply Chain Management のインスタンスに関連付けられた接続を選択します。
5. **サービス環境へデプロイする** オプションを **いいえ** に設定します。
6. **OK** を選択します。

## <a name="issue-electronic-invoices"></a>電子請求書を発行する

1. **組織管理** \> **定期的** \> **電子ドキュメント** \> **電子ドキュメントの送信** に移動します。
2. **対象に含めるレコード** クイック タブで、**フィルター** を選択します。
3. **追加** を選択して、クエリのフィルターにテーブル名を追加します。
4. 請求書を含むテーブルを選択します。

    > [!NOTE]
    > テーブル名は、このトピック冒頭の [アプリケーション設定を構成する](#configure-the-application-setup) セクションに記載のテーブルに含まれている必要があります。

5. クエリするテーブルからフィールド名を選択します。
6. クエリを実行する基準のテーブル名とフィールド名を入力します。
7. 手順 5 と手順 6 を繰り返してクエリにフィールドと条件を追加し、**OK** を選択します。
8. **OK** を選択します。

## <a name="view-submission-logs"></a>送信ログの表示

1. **組織管理** \> **定期的** \> **電子ドキュメント** \> **電子ドキュメントの提出ログ** に移動します。
2. **ドキュメントのタイプ** フィールドで、請求書を含むテーブルを選択します。

    > [!NOTE]
    > テーブル名は、このトピック冒頭の [アプリケーション設定を構成する](#configure-the-application-setup) セクションに記載のテーブルに含まれている必要があります。

3. グリッドで請求書を選択し、**照会** \> **詳細の送信** を選択します。
