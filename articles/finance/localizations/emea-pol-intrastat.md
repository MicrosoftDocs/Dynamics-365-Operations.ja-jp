# <a name="polish-intrastat"></a>ポーランドのイントラスタット

[!include[banner](../includes/banner.md)]

**イントラスタット** ページは、欧州連合 (EU) の加盟国での取引に関するレポート情報を生成するために使用されます。 ポーランドのイントラスタット申告には、報告する商品の取引に関する情報が含まれています。

ポーランドのイントラスタット申告には次のフィールドが含まれます。 すべてのフィールドは、着荷と発送に含まれていますが、発送に含まれていない **RodzajTransportu** (配送モード) および **KrajPochodzenia** (原産国または地域) と、着荷に含まれていない **IdKontrahenta** (顧客の外国 VAT 番号) は除きます。

| フィールド名 | Description |
|-------------------------|-------------------------|
| Deklaracja データ | ドキュメントが作成される日付。 |
| Miesiac | 申告の参照月。 |
| Rok | 申告の参照年。 |
| Numer | 参照期間の申告番号。 |
| Wersja | 申告のバージョン番号。 |
| NrWlasny | 申告識別子。 この値は自動的に生成されます。 |
| Typ | レポートの方向。</br><li>着荷の"P" が印刷されます。</li><li>発送の場合は、"W" が印刷されます。</li> |
| Rodzaj | 申告のタイプ。 この値は、レポートが元の申告か訂正申告かを示します。 |
| UC | イントラスタット申告の宛先となる単位コード。 この値は、**対外貿易パラメーター** ページの **エージェント** タブの **売上税** セクションの **課税控除番号** フィールドで指定されます。 |
| Nazwa | 会社名です。 |
| Miejscowosc、UlicaNumer、KodPocztowy | 法人の完全な住所。 |
| Nip | ポーランドの税 ID 番号 (付加価値税 [VAT] ID)。 |
| Regon | ポーランドの統計 ID 番号 (企業番号)。 |
| LacznaWartoscFaktur | 請求書金額の合計。 |
| LacznaWartoscStatystyczna | 統計金額の合計。 |
| LacznaLiczbaPozycji | 商品品目の合計数。 |
| PozId | 指定された商品品目の連続した番号。 |
| OpisTowaru | 商品の取引名。 |
| KrajPochodzeniaWysylki | 取引先企業の国または地域の国際標準化機構 (ISO) コード。 |
| WarunkiDostawy | 配送条件のイントラスタット コード。 |
| RodzajTransakcji | トランザクションの性質を示すコード。 ポーランドの会社では、2 桁のトランザクション コードを使用しています。 |
| KodTowarowy | 結合分類に従った 8 桁の商品コード。 |
| RodzajTransportu | 配送モードのイントラスタット コード。 |
| KrajPochodzenia | 商品が生産または製造された国または地域の ISO コード。 |
| MasaNetto | 正味質量 (フル キログラム)。 |
| IloscUzupelniajacaJm | 補足測定単位の数量 (整数)。 |
| WartoscFaktury | 1 つの品目によってカバーされるすべてのトランザクションの請求金額。 |
| WartoscStatystyczna | 統計値。 |
| Wypelniajacy: NazwiskoImie、Telefon、Faks、Email | 申告を提出する人の姓名、電話番号、FAX 番号、およびメール アドレス。 |
| IdKontrahenta | EU 加盟国内の顧客の外国 VAT 番号。 |


## <a name="set-up-intrastat"></a>イントラスタットの設定

グローバル リポジトリから、次の電子申告 (ER) コンフィギュレーションの最新バージョンをインポートします。

   - イントラスタット モデル
   - イントラスタット レポート
   - イントラスタット (PL)

詳細については、[ER コンフィギュレーションをコンフィギュレーション サービスのグローバル リポジトリからダウンロードする](../../fin-ops-core/dev-itpro/analytics/er-download-configurations-global-repo.md)を参照してください。

## <a name="set-up-a-vat-id-and-an-enterprise-number-for-your-company"></a>会社の VAT ID と企業番号の設定

### <a name="create-registration-types-for-company-codes"></a>会社コードの登録タイプの作成

会社コードには、VAT ID (NIP コード) 用と企業番号 (Regon コード) 用の 2 つの登録タイプを作成する必要があります。

1. **組織管理** > **グローバル アドレス帳** > **登録タイプ** > **登録タイプ** の順に移動します。
2. アクション ウィンドウで、**新規** を選択して、VAT ID の登録タイプを作成します。
3. **登録タイプの詳細の入力** ダイアログ ボックスの **名前** フィールドに新しい登録タイプの名前を入力します。 たとえば、**NIP** と入力します。
4. **国/地域** フィールド で, **POL** を選択します。
5. **作成** を選択します。
6. アクション ウィンドウで、**新規** を選択して、企業番号の登録タイプを作成します。
7. **登録タイプの詳細の入力** ダイアログ ボックスの **名前** フィールドに新しい登録タイプの名前を入力します。 たとえば、**Regon** を入力します。
8. **国/地域** フィールド で, **POL** を選択します。
9. **作成** を選択します。

### <a name="match-the-registration-types-with-registration-categories"></a>登録タイプと登録カテゴリのマッチング

1. **組織管理** > **グローバル アドレス帳** > **登録タイプ** > **登録カテゴリ** の順に移動します。
2. アクション ウィンドウで、**新規** を選択して、作成した各登録タイプと登録カテゴリ の間にリンクを作成します。

    - VAT ID (NIP コード) の登録タイプで、**VAT ID** 登録カテゴリを選択します。
    - 企業番号 (Regon コード) の登録タイプで、**企業 ID (COID)** 登録カテゴリを選択します。

### <a name="set-up-a-vat-id-and-an-enterprise-number-for-your-company"></a>会社の VAT ID と企業番号の設定

1. **組織管理** > **組織** > **法人** の順に移動します。
2. グリッドで会社を選択します。
3. アクション ウィンドウで、**登録 ID** を選択します。
4. **登録 ID** クイック タブで、**追加** を選択します。
5. **登録タイプ** フィールドで、以前に作成した登録タイプのいずれかを選択します。
6. 前の手順で選択した登録タイプに応じて、会社の VAT ID (NIP コード) または企業番号 (Regon コード) を入力します。
7. 上で作成した他の登録タイプについて、手順 4 ～ 6 を繰り返します。

## <a name="set-up-a-company-address"></a>会社住所の設定

1. **組織管理** > **組織** > **法人** の順に移動します。
2. グリッドで会社を選択します。
3. **住所** タブで、**編集** を選択します。
4. **住所の編集** ダイアログ ボックスの **郵便番号** フィールドで、会社の郵便番号を選択します。
5. **住所** フィールドに、住所を入力します。
6. **市区町村** フィールドで、市区町村を選択します。

## <a name="set-up-foreign-trade-parameters"></a>対外貿易パラメーターの設定

1. **税** > **設定** > **税設定対外貿易パラメーター** の順に移動します。
2. **イントラスタット** タブの、**電子申告** クイック タブの **ファイル形式のマッピング** フィールドで、**イントラスタット (PL)** を選択します。
3. **レポート形式のマッピング** フィールドで **イントラスタット レポート** を選択します。
4. **商品コード階層** クイック タブの **カテゴリ階層** フィールドで、**イントラスタット** を選択します。
5. **トランザクション コード** フィールドで、プロパティ転送のトランザクション コードを選択します。 このコードは、補償 (金銭的またはその他) に対する資産の実際のまたは計画された譲渡を生み出すトランザクションに使用します。 また、修正に対して使用します。 ポーランドの会社では、2 桁のトランザクション コードを使用しています。
6. **訂正票** フィールドで商品の返品用のトランザクション コードを選択します。
7. **国/地域のプロパティ** タブの、**国/地域** フィールドで、会社が事業を行っているすべての国または地域を一覧表示します。 EU の加盟国ごとに、**国/地域のタイプ** フィールドで **EU** を選択して、国をイントラスタット レポートに表示します。 ポーランドの場合、**国/地域のタイプ** フィールドで、**国内** を選択します。
8. **エージェント** タブの **エージェント** クイック タブの **売上税** セクションの **課税控除番号** フィールドに、**420000** と入力して、イントラスタット申告で指定する単位コードを指定します。
9. **連絡先** タブで、申告を提出する人の名前、電話番号、FAX 番号、およびメール アドレスを入力します。
10. **番号順序** タブの **XML ファイル番号** 参照の **番号順序コード** フィールドで、最大 9 文字の連続しない番号順序を指定します。 このフィールドは、イントラスタット レポートの **申告識別子** フィールドの値を自動的に生成するために使用されます。

## <a name="set-up-product-parameters-for-the-intrastat-declaration"></a>イントラスタット申告の製品パラメーターの設定

1. **製品情報管理** > **製品** > **リリースされた製品** の順に移動します。
2. グリッドで製品を選択します。
3. **対外貿易** クイック タブの **イントラスタット** セクションにある **商品** フィールドで、商品コードを選択します。 商品の名前は、イントラスタット レポートの **商品の説明** フィールドに印刷されます。
4. **発生元** セクションの **国/地域** フィールドで、製品の原産国または地域を選択します。
5. **在庫管理** クイック タブの **正味重量** フィールドに、製品の重量をキログラムで入力します。

## <a name="set-up-compression-of-intrastat"></a>イントラスタットの圧縮の設定

-   **税** > **設定** > **対外貿易** > **イントラスタットの圧縮** に移動し、イントラスタット情報が集計されたとき比較するフィールドを選択します。 ポーランドのイントラスタットの場合、次のフィールドを選択します。

    - 商品
    - トランザクション コード
    - 原産国/地域
    - 輸送
    - 配送条件
    - 送信者の国/地域
    - 国/地域
    - 訂正
    - 課税控除番号
    - 方向
    - 請求書

## <a name="set-up-the-transport-method-and-delivery-terms"></a>配送方法と配送条件の設定

1.  配送コードを設定します。

    1. **税** > **設定** > **対外貿易** > **輸送方法** の順に移動します。
    2. アクション ウィンドウで、**新規** を選択します。
    3. **輸送** フィールドに、固有のコードを入力します。 ポーランドの会社では、1 桁の輸送コードを使用しています。

2.  配送イントラスタット コードを設定します。

    1. **調達** > **設定** > **配布** > **配送条件** の順に移動します。
    2. グリッドで、一連の配送条件を選択します。
    3. **一般** クイック タブの **イントラスタット コード** フィールドで、一意のコードを選択します。

## <a name="intrastat-transfer"></a>イントラスタットの転送

**イントラスタット** ページの操作ウィンドウで、**転送** を選択し、販売注文、自由書式の請求書、発注書、仕入先請求書、仕入れ先製品受領書、プロジェクト請求書、および転送オーダーから EU 内での貿易についての情報を自動的に転送できます。 EU 加盟国が宛先の国または地域または委託として扱っているドキュメントのみ転送されます。

さらに、アクション ウィンドウで **新規** を選択することで、トランザクションを手動で入力することもできます。

### <a name="generate-an-intrastat-report"></a>イントラスタット レポートの生成

1. **税** > **申告** > **対外貿易** > **イントラスタット** の順に移動します。
2. アクション ウィンドウで、**出力** &gt; **レポート** を選択します。
3. **イントラスタット レポート** ダイアログ ボックスで、次のフィールドを設定します。

    | フィールド | Description |
    |-------------------------|-------------------------|
    | 自 | レポートの開始日を選択します。 |
    | ファイルの生成 | イントラスタット レポートの .xml ファイルを生成するには、このオプションを **はい** に設定します。 |
    | ファイル名 | .xml ファイルの名前を入力します。 |
    | レポートの生成 | イントラスタット レポートの .xlsx ファイルを生成するには、このオプションを **はい** に設定します。 |
    | レポート ファイル名 | .xlsx ファイルの名前を入力します。 |
    | 方向 | EU 内の着荷に関してレポートするには **着荷** を選択します。</br>EU 内の発送に関してレポートするには **発送** を選択します。 |
    | 申告識別子 | ドキュメント ID は自動的に生成され、更新できます。 |
    | 申告のタイプ | 元の申告の **申告** を選択します。</br>以前に提出された元の申告または訂正申告を完全に置き換える訂正申告で、**訂正申告 – 置き換え** を選択します。 |
    | ドキュメント作成の市区町村 | イントラスタット申告の **Miejscowosc** フィールドに印刷する値を入力します。 |
    | ドキュメント作成の日付 | イントラスタット申告の **Deklaracja データ** フィールドに印刷する値を入力します。 |
    | 文書番号 | イントラスタット申告の **数値** フィールドに印刷する値を入力します。 |
    | ドキュメント バージョン | イントラスタット申告の **Wersja** フィールドに印刷する値を入力します。 |

4. **OK** を選択し、生成されたレポートを確認します。

## <a name="example"></a>例

この例は、**DEMF** 法人を使用することでイントラスタットの着荷と発送を投稿する方法を示しています。

### <a name="preliminary-setup"></a>予備設定

次の ER コンフィギュレーションの最新バージョンをインポートします。

   - イントラスタット モデル
   - イントラスタット レポート
   - イントラスタット (PL)

### <a name="set-up-a-company-address"></a>会社住所の設定

1. **組織管理** > **グローバル アドレス帳** > **アドレス** > **アドレスの設定** の順に移動します。
2. **市区町村** タブで、**新規** を選択します。
3. **国/地域** フィールド で, **POL** を選択します。
4. **市町村** フィールドで、**ワルシャワ** と入力します。
5. **郵便番号** タブで、**新規** を選択します。
6. **国/地域** フィールド で, **POL** を選択します。
7. **市町村** フィールドで、**ワルシャワ** を選択します。
8. **郵便番号** フィールドに、**00-844** と入力します。
9. **組織管理** > **組織** > **法人** に移動し、**DEMF** 法人を選択します。
10. **住所** クイック タブで、**編集** を選択します。
11. **国/地域** フィールド で, **POL** を選択します。
12. **郵便番号** フィールドで、**31-111** を選択します。
13. **住所** フィールドに、**Statystyczna 22/1** と入力します。
14. **市町村** フィールドで、**ワルシャワ** を選択します。
15.  **OK** を選択します。

## <a name="set-up-a-vat-id-and-an-enterprise-number-code-for-your-company"></a>会社の VAT ID と企業番号コードの設定

### <a name="create-registration-types-for-company-codes"></a>会社コードの登録タイプの作成

1. **組織管理** > **グローバル アドレス帳** > **登録タイプ** > **登録タイプ** の順に移動します。
2. アクション ウィンドウで、**新規** を選択して、VAT ID (NIP コード) の登録タイプを作成します。
3. **登録タイプの詳細の入力** ダイアログ ボックスの **名前** フィールドに、**NIP** と入力します。
4. **国/地域** フィールド で, **POL** を選択します。
5. **作成** を選択します。
6. アクション ウィンドウで、**新規** を選択して、企業番号 (Regon コード) の登録タイプを作成します。
7. **登録タイプの詳細の入力** ダイアログ ボックスの **名前** フィールドに、**Regon** と入力します。
8. **国/地域** フィールド で, **POL** を選択します。
9. **作成** を選択します。

### <a name="match-the-registration-types-with-registration-categories"></a>登録タイプと登録カテゴリのマッチング

1. **組織管理** > **グローバル アドレス帳** > **登録タイプ** > **登録カテゴリ** の順に移動します。
2. アクション ウィンドウで、**新規** を選択して、作成した各登録タイプと登録カテゴリ の間にリンクを作成します。

    - **NIP** 登録タイプで、**VAT ID** 登録カテゴリを選択します。
    - **Regon** 登録タイプで、**企業 ID (COID)** 登録カテゴリを選択します。

### <a name="set-up-a-vat-id-and-an-enterprise-number-for-your-company"></a>会社の VAT ID と企業番号の設定

1. **組織管理** > **組織** > **法人** の順に移動します。
2. グリッドで、**DEMF** を選択します。
3. アクション ウィンドウで、**登録 ID** を選択します。
4. **登録 ID** クイック タブで、**追加** を選択します。
5. **登録タイプ** フィールドで、**NIP** を選択します。
6. **登録番号** フィールドで、**1234567890** を入力します。
7. **追加** を選択します。
8. **登録タイプ** フィールドで、**Regon** を選択します。
9. **登録番号** フィールドで、**12345678901234** を入力します。

### <a name="set-up-a-number-sequence-code"></a>番号順序コードの設定

1. **組織管理** > **番号順序** > **番号順序** の順に移動します。
2. アクション ペインの、**番号順序** タブの、**新規** グループで、**番号順序** を選択します。
3. **ID** クイック タブ の **番号順序コード** フィールドに、**XML\_ファイル** と入力します。
4. **スコープ パラメーター** クイック タブの **スコープ** フィールドで、**会社** を選択します。
5. **会社** フィールドで、**DEMF** を選択します。

### <a name="create-a-purchase-order"></a>発注書の作成

1. **買掛金勘定** > **発注書** > **すべての発注書** の順に移動します。
2. アクション ウィンドウで、**新規** を選択します。
3. **発注書の作成** ダイアログ ボックスでは、**仕入先勘定** フィールドで **DE-001** を選択します。
4. **サイト** フィールドで、**1** を選択します。
5. **倉庫** フィールドで、**11** を選択します。
6.  **OK** を選択します。
7. **ヘッダー** タブの **配送** クイック タブで、**配送方法** フィールドが **10** に設定され、**配送条件** フィールドが **CIF** に設定されていることを確認します。
8. **明細行** タブの **購買注文明細行** クイック タブにある、**品目番号** フィールドから、**D0003** を選択します。 その後、**数量** フィールドに **6** と入力します。
9. **明細行の詳細** クイック タブの **対外貿易** タブで、**トランザクション コード** が **11** に設定され、**輸送** フィールドが **3** に設定され、**商品** フィールドが **100 200 30** に設定され、**原産国/地域** フィールドが **DEU** に設定されていることを確認します。
10. アクション ウィンドウの **購買** タブにある **アクション** グループから、**確認** を選択します。
11. アクション ペインの、**請求書** タブで、**生成** グループから **請求書** を選択します。
12. アクション ウィンドウで、**既定の取得元** を選択し、**明細行の既定の数量** フィールドで **注文済数量** を選択します。 その後、**OK** を選択します。
13. **仕入先請求書ヘッダー** クイック タブの **請求書 ID** セクションにある **番号** フィールドに、**00010** と入力します。
14. **請求日** セクションの **請求日** フィールドで、現在の日付を選択します。 この日付は、イントラスタット配送に使用されます。
15. **ドキュメント受領日** フィールドで、**10/18/2021** (2021 年 10 月 18 日) を選択します。
16. アクション ウィンドウで **転記** を選択して、請求書を転記します。
