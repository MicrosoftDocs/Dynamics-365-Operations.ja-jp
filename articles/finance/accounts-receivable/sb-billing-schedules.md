# <a name="create-billing-schedules"></a>請求スケジュールの作成

[!include [banner](../includes/banner.md)]

**請求スケジュール** ページでは、請求スケジュールを作成、削除、または編集できます。 また、請求スケジュールの一覧を確認できます。 請求スケジュールを作成する際の既定値は、請求書作成スケジュールに関連付けられている請求グループによって決まります。 追加情報は、**経常契約の請求書作成** パラメータ ページで設定します。

## <a name="create-a-billing-schedule"></a>請求スケジュールを作成する

請求スケジュールを作成するには、次の手順に従います。

1. **請求スケジュール** ページで、**新規** を選択します。
2. **請求スケジュールの作成** ダイアログ ボックスの **請求スケジュール グループ** フィールドで、請求スケジュール グループを選択します。
3. **顧客口座** フィールドで、顧客口座を選択します。
4. **開始日** フィールドで開始日を選択し、**期間数** フィールドに期間の数を入力します。 **終了日** フィールドは、入力した期間の数に基づいて更新されます。 **請求終了日** フィールドを更新すると、**期間数** フィールドが **0** (ゼロ) に更新されます。
5.  **OK** を選択します。
6. **請求スケジュール** ページの **全般** タブの **説明** フィールドに、請求スケジュールの説明を入力します。
7. **マイルストーン テンプレート** フィールドで、**マイルストーン請求** のマイルストーン テンプレートを選択します。

**請求元仕入先** や **通貨コード** などのフィールドは、顧客からの情報で更新されます。

**請求頻度** フィールドおよび **請求間隔** フィールドは、選択した請求スケジュール グループに基づいて自動的に設定されます。

8. 個別の請求書を作成するには、**個別に請求** オプションを **はい** に設定 します。
9. 最終請求期間後に請求書作成スケジュールを自動的に更新するには、**自動更新** オプションを **はい** に設定し、**更新ごとに追加する明細行** を設定します。

**パラメータ** フィールドは、**経常契約の請求書作成** ページの値に基づいて自動的に設定されます。

10. 請求スケジュールの金額を計算するには、**部分期間の支払** オプションを **はい** に設定します。
11. 請求スケジュールの詳細行を月の終わりに揃えるには、**月に調整** オプションを **はい** に設定します。
12. **契約の開始日** フィールドおよび **契約の終了日** フィールドに、契約の開始日と終了日を入力します。 これらの日付は情報専用です。

**支払** フィールドには、顧客からの顧客支払情報が表示されます。 明細行品目が保留または終了した場合は、支払情報を変更できません。

> [!NOTE]
> 品目別に請求書を連結する場合は、**支払期間**、**方法**、および **請求スケジュール** の各フィールドの値が一致する必要があります。 それ以外の場合、請求書を連結できます。

13. **住所** タブで、**配送先住所** フィールドと **請求先住所** フィールドを確認および更新できます。
14. **連絡先情報** タブで、エンド ユーザー アカウントを請求スケジュールに関連付けできます。
15. **連絡先情報** フィールド には、連絡先、メールアドレス、およびインターネット アドレスを入力できます。
16. パートナー コミッション情報を追跡するには、**パートナー アカウント** および **パートナー コミッション レート** フィールドを設定します。 これらのフィールドは情報専用です。
17. **合計** タブでは、請求スケジュールに対して計算された各種合計を表示できます。
18. **保留** タブでは、保留が削除された際に、いつ請求スケジュールが保留に設定されたかに関する監査情報を表示できます。
19. **終了** タブでは、請求スケジュールに適用または削除された終了の履歴を表示できます。
20. **保存** を選択します。
21. **請求スケジュール明細行** クイック タブで、**行の追加** を選択します。
22. **品目番号** フィールドで、品目番号を選択します。 追加される品目が収益分割の親品目である場合は、その子品目を使用して明細行が自動的に更新されます。 収益分割の親品目のみを編集できます。 これにより、すべての子品目が更新されます。
23. **品目タイプ** フィールドで、品目タイプを選択します。
24. 開始日と終了日を更新します。
25. 請求書を作成する前に、**請求頻度** フィールドを更新することで、請求頻度を変更できます。 請求スケジュールの最初の請求書を作成した後は、請求頻度を変更できません。
26. **単位** フィールドで、品目の測定単位を選択します。
27. **価格決定方法** フィールドで、品目の価格決定方法を選択します。

**単価** フィールドは、在庫から自動的に設定されます。 ただし、価格決定方法を **一律** に変更した場合は更新できます。

## <a name="remove-a-line-item"></a>明細行品目の削除

請求スケジュールから品目を削除するには、次の手順に従います。

1. **請求スケジュール** ページの **スケジュール番号** フィールドで、編集する請求スケジュールの番号を選択します。
2. **請求スケジュール明細行** クイック タブで、削除する明細行を選択し、**削除** を選択します。
3. **保存** を選択します。

このトピックでは、**請求スケジュール行** クイック タブの明細行に使用できるアクションと詳細について説明します。

## <a name="billing-schedule-line-actions"></a>請求スケジュール明細行アクション

**請求スケジュール明細行** クイック タブのボタンを使用すると、アクションを個々の明細行に適用できます。

| ボタン | Description |
|--------|-------------|
| 明細行の追加 | 明細行を請求スケジュールに追加します。 |
| 品目リストから追加 | 一覧で複数の品目を選択して、請求書作成スケジュールに追加します。 |
| 除去 | <p>選択した明細行を請求スケジュールから削除します。</p><p>収益分割の一部である品目の場合は、親品目のみを削除できます。 その後、関連付けられている子品目はすべて自動的に削除されます。</p> |
| 請求詳細の表示 | 請求明細を表示します。 |
| 退職 | <p>選択した行を終了します。 このボタンは、選択した明細行のステータスが **有効** の場合にのみ使用できます。</p><p>収益分割の一部である品目の場合は、親品目のみを終了できます。</p> |
| 終了の削除 | <p>選択した明細行から終了を削除します。 このボタンは、選択した明細行のステータスが **終了済み** の場合にのみ使用できます。</p><p>収益分割の一部である品目の場合は、親品目からの終了のみを削除できます。</p> |
| 保留中に設定する | <p>選択した行を保留に設定する詳細を選択します。</p><p>収益分割の一部である品目の場合は、親品目のみを保留にできます。</p> |
| 保留の解除 | <p>選択した明細行から保留を削除します。</p><p>収益分割の一部である品目の場合は、親品目からの終了のみを保留にできます。</p> |
| エスカレーションと割引 | 収益分割の一部である品目には使用できません。収益分割では **金額ゼロ** 配賦方法を使用する親品目以外は、このボタンは使用できません。 |
| 繰延 | <p>選択した行の繰延オプションを入力します。</p><p>このボタンは、収益分割の親品目には使用できません。</p> |
| マイルストーン配賦 | <p>選択した行のマイルストーン配賦情報を確認および更新します。</p><p>このボタンは、請求スケジュールの品目がマイルストーン品目の場合にのみ使用できます。</p> |
| サポートと更新 | <p>選択した行のサポートおよび更新情報を確認および更新します。</p><p>このボタンは、請求スケジュールの品目がサポートまたは更新品目の場合にのみ使用できます。</p> |
| 分析コードの表示 | **請求スケジュール行** グリッドに表示される分析コード列の表示/非表示を切り替えます。 |
| 単価の計算 | <p>顧客が契約金額を分割払い (たとえば、日次、月次、四半期、半年、年次など) できるように、品目の単価を計算します。 契約価格および価格頻度を選択できます。</p><p>変更の監査証跡を表示できます (古い契約価格と頻度、新しい契約価格と頻度、変更を行ったユーザー、変更の日時)。</p> |
| 調整日 | <p>変更後の品目の配置日を指定します。</p><p>**品目グループ** フィールドで品目グループを選択すると、すべての品目で、請求書作成スケジュールの品目グループの最初の品目の配置日が使用されます。 **品目グループ** フィールドが空白の場合は、すべての品目に使用する配置日を指定できます。</p><p>このボタンは、**請求頻度** フィールドが **年次** に設定されている場合にのみ使用できます。</p> |
| 未請求収益 | <p>未定収益機能を使用する品目を設定します。 その後、品目に対して未収収益および未収の割引勘定を指定できます。</p><p>このボタンは、**品目タイプ** フィールドが **標準** に設定されている品目に対してだけ使用できます。 これは、既存の請求スケジュールや請求書が既に作成されている請求スケジュール行では使用できません。</p> |
| 収益分割の子の追加 | <p>販売注文に追加する子品目を選択します。</p><p>このボタンは、収益分割の親品目にのみ使用できます。</p> |
| 詳細価格決定オプション | 品目の価格決定オプションを編集します。 |

## <a name="billing-schedule-line-details"></a>請求スケジュール明細行の詳細

**請求スケジュール明細行** クイック タブで明細行を選択すると、その明細行に固有の詳細を表示できます。 これらの詳細は、複数のタブに分かれています。

### <a name="general-tab"></a>[全般] タブ

**一般** タブでは、次の情報を利用できます。

| フィールドまたはセクション | Description |
|------------------|-------------|
| 用途 | <p>このセクションのトピックでは、使用品目について説明します。 これは次のフィールドが含まれています。</p><ul><li>**使用 ID** ― メートルまたは使用品目の ID。</li><li>**読み取り** オプション ― 読み取りオプション: **読み取り** または **消費**。</li><li>**消費の見積** ― 請求書が作成されていない期間を持つ使用品目の見積消費を指定します。 **請求書の詳細** ページで、消費予測の請求書の詳細行を確認できます。</li></ul> |
| 外部参照\* | ここでは、**外部** フィールドおよび **行番号** フィールドを含み、このフィールドで外部参照情報を指定できます。 |
| マイルストーン | <p>このセクションのトピックでは、マイルストーン品目について説明します。 品目の完了日を示す **見積完了日** フィールドが含まれています。</li></ul> |
| テキスト | 明細行のコメント。 このテキストは、顧客または法人の既定の言語に翻訳されます。 |
| 品目グループ | 行品目の品目グループ。 |
| 調整日 | 請求書作成スケジュールの配置日。 |

\* **請求書の生成** ページで品目別に請求書を連結する場合は、**外部参照** フィールドが一致している必要があります。 1 文字でも異なる場合、品目は請求書に連結されません。 **外部参照** セクションのいずれのフィールドでも検証チェックは行われず、**行番号** フィールドの値は正の整数である必要があります。

### <a name="address-tab"></a>[住所] タブ

**住所** タブでは、次の情報を利用できます。

| フィールド | Description |
|-------|-------------|
| 配送先住所 | <p>名作業品目の配達先住所を選択します。 既定の配送先住所は、**住所** クイック タブの基本配送先住所です。</p><p>住所を変更するときに、次の住所オプションを選択できます。</p><ul><li>**住所** ― 現在の顧客の住所を選択します。</li><li>**使用中** ― 現在の顧客に対して現在使用されている住所を選択します。</li><li>**その他の住所** ― 任意の顧客レコードの住所を選択します。</li></ul><p>収益分解を使用する品目の場合は、親品目の住所のみを編集できます。 子品目の住所は親の住所と一致し、個別に編集することができます。</p> |
| 請求先住所 | <p>明細行品目の請求先住所を選択します。 既定の配送先住所は、**住所** クイック タブの基本配送先住所です。 住所は、使用できる住所の目的に基づいて必要に応じて変更できます。</p><ul><li>どの住所にも **請求書** の目的がない場合、目的に関係なく、既定の請求先住所が顧客の基本住所になります。</li><li>1 つ以上の住所に **請求書** の目的がある場合、既定の請求先住所は、最新で入力された住所です。</li><li>1 つ以上の住所が **請求書** の目的で、1 つの請求先住所が基本住所として設定されている場合、既定の請求先住所は、**請求書** を目的として設定された住所で、基本住所として設定されます。</li><li>収益分解を使用する品目の場合は、親品目の住所のみを編集できます。 子品目の住所は親の住所と一致し、個別に編集することができます。</li></ul> |

### <a name="product-tab"></a>[製品] タブ

**製品** タブでは、次の情報を利用できます。

| フィールドまたはセクション | Description |
|------------------|-------------|
| 保管分析コード | <p>ここでは、品目の保管情報を示します。 このフィールドには、品目のシリアル番号を示す **シリアル番号** フィールドが含まれています。</p><p>シリアル番号は、サポート プロセスおよび変更プロセス中に最初の販売注文からコピーされます。 収益分解を使用する品目の場合は、親品目のシリアル番号がすべての子品目にコピーされます。 **定期処理契約の請求パラメータ** ページで **シリアル番号のコピー** オプションが **はい** に設定されている場合は、シリアル番号がコピーされます。</li></ul> |
| 製品分析コード | 品目および値の製品詳細は、請求スケジュール行に対して選択された **バリアント番号** の値に基づいて自動的に更新されます。 |

### <a name="account-tab"></a>[勘定] タブ

**勘定** タブでは、次の情報を利用できます。

| フィールド | Description |
|-------|-------------|
| 主勘定 | 販売行に作成される主要勘定。 既定値は販売注文から取得されます。 このフィールドは、空白のままにすることができます。 |
| 品目財務分析コード | <p>顧客レコードおよび品目レコードに基づく既定の財務分析コード値。</p><p>収益分解を使用する品目の場合、子品目は、前の説明に従って、顧客レコードおよび品目レコードの財務分析コード値を使用します。 子品目の財務分析コード値を更新する必要がある場合は、データ エンティティをインポートできます。</p> |

### <a name="renewals-tab"></a>[更新] タブ

**更新** タブでは、次の情報を利用できます。

| フィールド | Description |
|-------|-------------|
| 自動的に更新 | <p>次の請求期間について請求スケジュール行を自動的に更新するかどうかを示す値。</p><ul><li>**はい** ― 請求書を作成すると、次の請求期間について請求スケジュール行が自動的に更新されます。</li><li>**いいえ** ― 請求スケジュール行は自動的に更新されません。 請求書作成スケジュールを手動で更新する必要があります。</li></ul> |
| 更新ごとに追加する明細行 | 請求スケジュールの契約に追加する行数。 |

さらに、**更新** タブには次のボタンも表示されます。

| ボタン | Description |
|--------|-------------|
| 未請求収益仕訳入力の監査 | 未定収益機能を使用する品目のすべての変更を表示します。 |
| 更新期間の追加 | 品目の項目を変更する項目を追加します。 新しい更新期間の開始日は、前の期間の終了日の後の次の日付です。 **更新終了日**、**繰延開始日**、**繰延終了日**、**品目数量**、および **単価** の各フィールドを更新できます。 |
| 更新期間の変更 | <p>更新期間を変更します。 最初の期間では、最初の仕訳入力が作成される前に繰延の開始日と終了日を変更できます。 それ以降の期間については、開始日は変更できません。 これは常に前の期間の終了日の後の日付です。</p><p>変更する期間の後に更新期間が存在する場合は、その期間の日付は変更できません。 この場合、変更後の品目の **数量** フィールドおよび **単価** フィールドのみを更新できます。</p><p>たとえば、3 つの期間があります。 <ul><li>最初の期間は既に開始されているため変更できません。</li><li>2 番目の期間では、数量および単価のみを変更できます。</li><li>3 番目の期間では、開始日を除くすべての値を変更できます。 また、**テンプレートからスケジュール** オプションを使用すると、未定の収益品目のテンプレートに基づく繰延スケジュールを作成できます。 このオプションが **はい** に設定されている場合は、**テンプレート** フィールドで繰延テンプレートを選択し、必要に応じ、延期の開始日と終了日を変更します。 その後の更新期間では、同じ繰延テンプレートを使用します。 ただし、繰延テンプレートは変更できます。</p></li></ul> |

### <a name="termination-tab"></a>[終了] タブ

**終了** タブでは、次の情報を利用できます。

| フィールド | Description |
|-------|-------------|
| 退職日 | 請求スケジュール明細行が終了した日付。 既定値はヘッダーの **終了日** フィールドから取得されます。 必要に応じて値を変更することができます。 |
| 終了タイプ | 終了タイプ。 既定値はヘッダーの **終了タイプ** フィールドから取得されます。 |

### <a name="hold-tab"></a>[保留] タブ

収益および経費の繰延を使用する場合は、**保留** タブには繰延日が表示されます。

### <a name="escalation-and-discount-tab"></a>エスカレーションと割引タブ

**エスカレーションおよび割引** タブでは、次の情報を利用できます。

| フィールド | Description |
|-------|-------------|

### <a name="price-changes-tab"></a>[価格変更] タブ

**標準** 価格から **一律** 価格に変更された明細行の場合、**価格の変更** タブのグリッドに次の列が表示されます。

- 日付の変更
- ユーザーにより変更済み
- 標準価格
- 一律価格
- 価格の更新
