# <a name="subscription-billing-power-bi-content"></a>サブスクリプション請求管理の Power BI コンテンツ

[!include[banner](../includes/banner.md)]

このトピックでは、サブスクリプション請求管理の Microsoft Power BI コンテンツの内容について説明します。 Power BI レポートにアクセスする方法を説明し、コンテンツを作成するために使用したデータ モデルおよびエンティティについての情報を提供します。 

## <a name="overview"></a>概要

サブスクリプション請求管理の Power BI コンテンツは、サブスクリプション請求管理の担当者およびマネージャーのために作成されました。 請求スケジュールに対する毎月の経常収益 (MRR)、繰延スケジュールに対する定率法とウォーターフォールの情報など、サブスクリプション請求管理の主要メトリックスを提供します。 請求スケジュールおよび繰延スケジュールからのデータを使用して、経常利益と収益の概要を提供します。

Power BI コンテンツには、次のように合計 4 つの分析レポートを提供する 3 つのタブがあります。 

- **分析 - MRR** – このタブは、**毎月の定期請求** レポートと **請求スケジュールの詳細** レポートを提供します。
- **分析 - ウォーターフォール** – このタブは、**収益ウォーターフォール** レポートを提供します。
- **分析 - 定率法** – このタブは、**定率法** レポートを提供します。

## <a name="view-data-on-the-analytical-reports"></a>分析レポートのデータを表示する

分析レポートのデータを表示する前に、各レポートのレポート データを生成する定期処理を実行する必要があります。 レポートに必要なデータはデータベースに直接保存されているわけではないので、生成する必要があります。 

1. **サブスクリプション請求管理 \> 定期契約請求 \> 定期処理のタスク \> MRR 分析レポートのバッチ処理** に移動し、次の手順を実行します。

    1. 3 年以内の日付範囲を入力します。
    2. オプション: 定期的にデータを最新の情報に更新するために、定期的なバッチ処理ジョブを設定します。
    3.  **OK** を選択します。

2. バッチ ジョブの実行が終了した後、**システム管理 \> 設定 \> エンティティ格納** に移動し、**MRR レポート** の集計測定を最新の情報に更新します。 
3. **サブスクリプション請求管理 \> 収益および経費の繰延 \> 定期処理のタスク \> ウォーターフォール分析レポートのバッチ処理** に移動し、次の手順を実行します。

    1. 26 期以内の開始日と終了日を入力します。 
    2. 現在の期間で過去の期間の予測金額を表示する場合は、**現在の期間で過去の金額を予測する** オプションを **はい** に設定します。
    3. オプション: 定期的にデータを最新の情報に更新するために、定期的なバッチ処理ジョブを設定します。
    4.  **OK** を選択します。 

4. バッチ ジョブの実行が終了した後、**システム管理 \> 設定 \> エンティティ格納** に移動し、**ウォーターフォール レポート** の集計測定を最新の情報に更新します。
5. **サブスクリプション請求管理 \> 収益および経費の繰延 \> 定期処理のタスク \> 定率法分析レポートのバッチ処理** に移動し、次の手順を実行します。

    1. 26 期以内の開始日と終了日を入力します。 
    2. 現在の期間で過去の期間の予測金額を表示する場合は、**現在の期間で過去の金額を予測する** オプションを **はい** に設定します。
    3. オプション: 定期的にデータを最新の情報に更新するために、定期的なバッチ処理ジョブを設定します。
    4.  **OK** を選択します。

6. バッチ ジョブの実行が終了した後、**システム管理 \> 設定 \> エンティティ格納** に移動し、**定率法レポート** の集計測定を最新の情報に更新します。

## <a name="accessing-the-power-bi-content"></a>Power BI コンテンツへのアクセス

サブスクリプション請求管理の Power BI コンテンツは、**サブスクリプション請求管理** ワークスペースに表示されます。
