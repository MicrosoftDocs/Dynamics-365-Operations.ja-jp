# <a name="benefits-management-overview"></a>福利厚生の管理の概要

競争力を維持するには、最高の従業員を獲得し維持するために、充実したメリットを提供する必要があります。 医療や歯科などの標準的な利点に加えて、採用支援、レクリエーション プログラム、および衣料手当などの拡張サービスを提供することもできます。 Microsoft Dynamics 365 Human Resources の給付金の管理には、さまざまな給付金オプションをサポートする柔軟なソリューションが用意されています。 人事管理では、製品を紹介するための使いやすい従業員エクスペリエンスも含まれています。

- 拡張された給付金プランを使用すると、独自の給付金プランを作成および管理し、複雑な給付金のレート テーブルとネストされた階層をサポートできます。 より簡単な従業員エクスペリエンスのために、給付金プログラム、バンドル、および自動登録ルールを容易に作成できます。
- レックス クレジット プログラムでは、退職やその他のライフ イベントをサポートします。
- 広範な適格性ルールによって、適切な従業員が適切な利益を得ることができます。
- オンライン給付金の登録は、従業員にとって簡単なエクスペリエンスです。
- 認定されたライフ イベント処理は将来のライフ イベントをサポートします。

デモ データにアクセスする場合は、サンドボックス環境を再配置する必要があります。

> [!NOTE]
> これで、給付金管理ページをカスタマイズできます。 補償レートに関連するカスタム フィールドを、給付金計画の **補償オプション** ページに追加できます。 カスタム フィールドの操作についての詳細については、[カスタム フィールド](hr-developer-custom-fields.md) を参照してください。
>
> ![給付金管理カスタム フィールド](media/hr-benefits-management-custom-fields.png)

## <a name="enable-benefits-management"></a>給付金管理を有効にする

この記事では、Human Resources の機能を有効にする方法について説明します。 また、Human Resources の既存の機能が給付金の管理に置き換えられているか、給付金の管理を有効にした後に無効になっている機能についても説明します。

> [!IMPORTANT]
> **実稼働** 環境で福利厚生の管理を有効にした後、無効にすることはできません。 **実稼働** 環境で有効にする前に、**サンドボックス** 環境で福利厚生の管理を有効にしてテストすることをお勧めします。 従来の福利厚生機能と新しい福利厚生の管理機能には大きな違いがあり、追加の設定が必要であり、実稼働前にテストする必要があります。

詳細については [機能の管理](hr-admin-manage-features.md) を参照してください。

## <a name="process-overview"></a>プロセスの概要

福利厚生を構成するプロセスには次のタスクが含まれます。

1.  必要な福利厚生情報を設定します。
2.  オプションの福利厚生情報を設定します。
3.  福利厚生計画を設定します。
4.  フレックス クレジット プログラムを設定します (オプション)。
5.  必要な従業員情報を構成します。
6.  オプションの従業員情報を構成します。
7.  従業員を処理して適格性を決定します。
8.  従業員は、従業員セルフサービスを通じて計画を選択します (オプション)。
9.  従業員計画の選択を確認します。
10. ライフイベント処理 (オプション)。
11. レートを更新します (オプション)。

## <a name="set-up-required-benefit-information"></a>必要な福利厚生情報を設定する

従業員を計画に登録するには、次の複数のコンポーネントを設定する必要があります。

- **福利厚生の管理パラメータ** – これらの設定は、会社全体で共有されます。 既定の理由コードを設定し、**福利厚生の年間給与** オプションを有効にし、新規採用の既定の支払頻度を設定し、ライフ イベントを有効にできます。 詳細については、[福利厚生管理パラメーターの設定](hr-benefits-setup-parameters.md) を参照してください。
- **個人の連絡先適格性オプション** – 個人の連絡先は、設定された計画の依存または受益者です。 通常、子、配偶者、または信頼する組織です。 詳細については、[個人連絡先の適格性オプションコンフィグレーション](hr-benefits-setup-contact-eligibility-options.md) をご覧ください。
- **補充オプション** – 計画に使用できる補償のタイプを設定します。 具体的には、誰が対象とされるか、またはどの程度補償されるかを定義します。 詳細については、[補償オプションの作成](hr-benefits-setup-coverage-options.md) を参照してください。
- **計画タイプ** – 福利厚生計画を作成するときに使用できる計画のタイプを設定します。 計画のタイプの例は、**歯科**、**眼科** および **貯蓄** です。 計画タイプのいくつかの重要な設定により、福利厚生計画で使用できる設定が決まります。 詳細については、[計画タイプの作成](hr-benefits-setup-plan-types.md) を参照してください。
- **適格性ルール** – 従業員が計画に適格かどうかを判断するために使用されます。 すべての給付金プランには、少なくとも 1 つの適格性ルールを関連付ける必要があります。 詳細については、[適格性ルールとオプションを構成](hr-benefits-setup-eligibility-rules.md) をご覧ください。
- **支払頻度** – 福利厚生レートを構成する場合は、支払頻度が必要です。 レートで使用される支払頻度は、従業員や雇用主が週単位、月単位、または年単位で支払う必要がある金額を識別するのに役立ちます。 詳細については、[支払頻度の設定](hr-benefits-setup-payment-frequencies.md) 参照してください。
- **レート** – レートは、福利厚生が従業員または雇用主のいずれかにかかる費用を定義します。 従業員に金銭を配賦する必要がある場合 (たとえば、スポーツ クラブの会員資格へのクレジット)、負のレートが入力されます。 詳細については、[レートの構成](hr-benefits-setup-rates.md) を参照してください。
- **階層レート** – 階層レートは、一部の基準に基づいてレートを変更する必要がある場合に使用されます。 最も一般的な階層レートは、年齢に基づく 1 階層です。 ただし、性別、年齢、その他の基準に基づいてレートが変化する場合がある 2 重階層レートも設定できます。
- **控除** – 控除は、基本的に、福利厚生の控除を識別するために給与システムに渡されるヘッダー情報/コードです。 これらの控除は福利厚生計画で必要となるので設定する必要があります。 詳細については、[控除のコンフィギュレーション](hr-benefits-setup-deductions.md) を参照してください。
- **福利厚生期間** – 福利厚生期間は、従業員に福利厚生の補償がある期間です。 計画年とも呼ばれます。 ここで、オープン登録期間も設定します。

## <a name="set-up-optional-benefit-information"></a>オプションの福利厚生情報を設定する

次のコンポーネントは、福利厚生計画を作成するために設定する必要がありません。

- **プログラム** – プログラムは、同じ適格性ルールによって制御される一連の福利厚生です。 たとえば、販売部門の全員が携帯電話を取得する場合があります。
- **バンドル** – バンドルは福利厚生のグループで、追加の計画を選択するオプションを使用する前に、1 つの計画を選択する必要があります。 たとえば、高額控除可能な医療プランは、医療貯蓄口座 (HSA) プランにバンドルされる場合があります。
- **ライフ イベント タイプ** – ライフ イベントは、従業員の補償を変更できるイベントです。 ライフ イベント タイプは計画タイプにリンクされます。 たとえば、医療プランのタイプによって、出生や養子縁組、または配偶者の有無の変更のためにプランを変更できる場合があります。 ただし、保険プランのタイプでは、ライフ イベントによる変更は許可されない場合があります。 詳細については、[ライフ イベント タイプのコンフィギュレーション](hr-benefits-setup-life-event-types.md) を参照してください。
- **待機日数および待機期間** – 補償待機期間を福利厚生計画に設定できます。 たとえば、新入社員は、3 か月の雇用後にのみ 401(k) に登録できます。 この場合、待機期間は 3 か月です。 新規登録を処理し、月の特定の日にのみプロバイダに送信できる場合は、待機期間中に待機日数が使用されます。 たとえば、401(k) の登録を処理できるのは月の 15 日目、雇用 3 か月後、設定された待機期間は 3 か月、待ち時間は 15 日です。 詳細については、[待機日数のコンフィギュレーション](hr-benefits-setup-waiting-days.md) および[待機期間のコンフィギュレーション](hr-benefits-setup-waiting-periods.md) を参照してください。
- **理由コード** – 理由コードは、従業員の福利厚生が変化する理由を説明するために使用されます。 詳細については、[理由コードの設定](hr-benefits-setup-reason-codes.md) を参照してください。

## <a name="set-up-benefit-plans"></a>福利厚生計画の設定

福利厚生計画を設定する際、従業員を登録する前に次の手順を完了する必要があります。

- 福利厚生を割り当てます。
- 補償オプションを関連付けます。
- **一般** タブで、有効な開始日付と有効な終了日付を設定します。
- 適格性ルールを少なくとも 1 つ割り当てます。
- **設定** タブの **登録の許可/続行** フィールドを設定します。

福利厚生計画の設定方法の詳細については、[福利厚生計画の設定](hr-benefits-plans-setup.md) を参照してください。

## <a name="set-up-flex-credit-programs-optional"></a>フレックス クレジット プログラムを設定する (オプション)

フレックス クレジット プログラムを使用して、所定のフレックス クレジットの数に基づいて従業員を給付金に登録できます。 従業員は、フレックス クレジットが割り当てられる方法を選択できます。 たとえば、従業員がすでに配偶者の健康保険プランの対象となっている場合、健康保険にクレジットを使用する必要はありません。 したがって、代わりに他の福利厚生に使用することもできます。 フレックス クレジット プログラムの詳細については、[フレックス クレジット プログラムの設定](hr-benefits-plans-flex-credit-programs.md) を参照してください。

## <a name="configure-required-employee-information"></a>必要な従業員情報を構成する

従業員を福利厚生に登録する前に、必要な情報を提供する必要があります。 

従業員には、**職位** が割り当てられている必要があります。 **職位** は、**従業員** ページまたは **職位** ページで **従業員の割り当て** を更新することで、従業員に割り当てることができます。 

続いてに、従業員は入社日に固定報酬制度に加入しているか、**年間の給付給与** 額が設定されている必要があります。 従業員に **固定報酬** を割り当てる前に、**職位** を割り当てる必要があります。 

> [!NOTE] 
> **固定報酬の開始日** を **職位の割り当て日** より前にすることはできません。

代替手段として、コミッションなどの補足報酬を受け取る従業員がいる場合は、従業員レコードから **福利厚生の年間給与** 額を追加できます。 Human Resources では、補償範囲額を決定する際に、**固定報酬の年額** ではなく、**福利厚生の年間給与額** を使用します。 **福利厚生の年間給与** は、従業員の開始日または受給期間の開始日のいずれか遅い方の日付で有効である必要があります。 ただし、職位には **福利厚生年次給与** を割り当てるが必要はありません。 **福利厚生年次給与** 機能を有効にするには、**福利厚生管理** タブの **人事管理共有パラメータ** ページに移動します。この機能は既定で無効になっています。

> [!IMPORTANT]
> 従業員に対して **固定報酬** と **福利厚生の年間給与額** の両方が記録されている場合、**福利厚生の年間給与** が補償範囲額の決定に使用されます。 **作業者** ページの **雇用の詳細** セクションで、**福利厚生支払頻度** フィールドで値を選択する必要があります。

## <a name="configure-optional-employee-information"></a>オプションの従業員情報を構成する
性別または年齢に基づくレートを使用する給付金プランを作成する場合、従業員が福利厚生コストを計算するには、生年月日と性別を入力する必要があります。

## <a name="process-employees-to-determine-eligibility"></a>従業員を処理して適格性を決定する
従業員をプランに登録する前に、適格性処理を実行して、対象となるプランを決定します。 適格性処理の結果は、**プロセス結果ビューアー** で表示できます。 詳細については、[登録適格性の処理](hr-benefits-process-enrollment-eligibility.md) を参照してください。

## <a name="employees-select-plans-using-employee-self-service-optional"></a>従業員は、**従業員セルフサービス** を使用して計画を選択します (オプション)

オープン登録が行われると、従業員が新たに新しく雇用された場合や、ライフ イベントが発生した場合、従業員は **従業員セルフ サービス** を通じて福利厚生を選択または更新できます。 詳細については、[従業員セルフ サービスのコンフィギュレーション](hr-benefits-setup-employee-self-service.md) を参照してください。

## <a name="confirm-employee-plan-selections"></a>従業員計画の選択を確認する

従業員が登録されたと見なされる前に、従業員が選択する福利厚生を確認する必要があります。 従業員に代わって福利厚生を選択することもできます。 福利厚生を選択または確認するには、**従業員** ページの **福利厚生** タブで、**作業者の福利厚生計画** を選択します。 複数の従業員の福利厚生を選択または確認するには、**作業者の福利厚生計画の一括更新** ページを使用します。

## <a name="life-event-processing-optional"></a>ライフ イベント処理 (オプション)

従業員のライフサイクル中に、各従業員は、結婚、雇用の変更、扶養家族や受益者の変更など、さまざまなライフ イベントを経験する可能性があります。 ライフ イベントを使用するには、**人事管理共有パラメーター** ページでライフ イベントを有効にする必要があります。 計画タイプのライフ イベント タイプおよびライフ イベント オプションを設定します。

ライフ イベントを処理するには、採用期間中に、少なくとも一度はオープン登録を実行しておく必要があります。 米国では、オープン登録を通常年に 1 回行います。 米国以外では、オープン登録は入社時に実行される場合があります。 ライフ イベント処理では、作業者が福利厚生計画を選択する必要がありません。 ただし、作業者は、オープン登録処理に含まれている必要があります。 詳細については、次のトピックを参照してください。

- [ライフ イベントの処理](hr-benefits-process-life-events.md)
- [ライフ イベントの変更の処理](hr-benefits-process-life-event-changes.md)
- [ライフ イベントの適格性の処理](hr-benefits-process-life-event-eligibility.md)

ライフ イベント処理が完了し、ライフ イベントの登録期間がオープンである限り、従業員はライフ イベントから影響を受ける計画オプションを変更できます。 管理者は、従業員の代わりに変更を行うことができます。 登録期間が終了し、ライフ イベント トランザクションに関連する確認済みの計画タイプがない場合、トランザクションは終了します。

ライフ イベントの影響を受けるすべての計画を選択または免除し、確認する必要があります。 計画が選択、免除されていないために未確認の場合、ライフ イベント トランザクションは終了されません。

管理者は、ライフ イベント トランザクションを選択して **閉じる** を選択すると、必要に応じて手動で閉じることができます。 トランザクションに未確認の計画があり、管理者がそれを閉じようとすると、ライフ イベントのクローズが計画の編集を制限する場合があります。

終了したライフ イベントは削除できます。

管理者は、ライフ イベント トランザクションを選択して **再度開く** を選択すると、必要に応じて再度開くことができます。

## <a name="rate-updates-optional"></a>レートを更新する (オプション)

計画期間中に福利厚生のレートが変更される場合があります。 計画に既に登録されている従業員のレートを更新するには、レートの変更を処理する必要があります。 詳細については、[レート変更の処理](hr-benefits-process-rate-changes.md) を参照してください。
