# <a name="configure-life-event-types"></a>ライフ イベント タイプのコンフィギュレーション

Microsoft Dynamics 365 Human Resources は、ライフ イベント タイプを使用して、従業員福利厚生登録の更新に有効なイベントを定義します。 たとえば、結婚や子供の出産があります。 各ライフ イベント タイプ ID は、1 つのライフ イベント タイプにのみ関連付けることができます。 たとえば、ライフ イベント タイプ従業員の住所変更に関連付けられた住所変更というライフ イベント ID を作成する場合、別の ID を作成して従業員の住所変更のラベルを付けて、ライフ イベント タイプ従業員の住所変更と関連付けることはできません。 

ライフ イベント タイプを作成した後、それらを計画タイプに関連付ける必要があります。 詳細については、[計画タイプの作成](hr-benefits-setup-plan-types.md) を参照してください。

   > [!NOTE]
   > ライフ イベントを作成する場合、それを計画タイプに関連付ける必要があります。 詳細については、[計画タイプの作成](hr-benefits-setup-life-event-types.md) を参照してください。

## <a name="create-a-life-event-type"></a>ライフ イベント タイプの作成

1. **設定** の **給付金管理** ワーク スペースで、**ライフ イベント タイプ** を選択します。

2. **新規** を選択します。

3. 次のフィールドの値を指定します。

   | フィールド | 説明 |
   | --- | --- |
   | **ライフ イベント タイプ ID** | ライフ イベント タイプの固有識別子。 |
   | **説明** | ライフ イベント タイプの説明。 |
   | **ライフ イベント タイプ** | 従業員の給付金登録を更新する触媒。 ライフ イベントの一覧については、以下の [ライフ イベント](hr-benefits-setup-payment-frequencies.md?life-events) を参照してください。 |

4. **保存** を選択します。

## <a name="view-attached-plans"></a>添付計画の表示

選択されたライフ イベント タイプに関連付けられた計画の一覧を確認できます。 ライフ イベントは計画タイプに関連付けられ、計画タイプは計画に関連付けられます。 

1. **設定** の **給付金管理** ワーク スペースで、**ライフ イベント タイプ** を選択します。

2. **操作** を選択します。

3. **添付計画** を選択します。

## <a name="life-events"></a>ライフ イベント

| --- | --- | --- |

| **雇用状態の変更** | <ul><li>作業者 > 雇用</li><li>職歴ページ</li></ul> | 雇用状態の変更 |
| **従業員住所の変更** | <ul><li>作業者 > プロファイル > 住所 </li><li>作業者 > 個人情報 > 個人の連絡先 > 住所</li></ul> 住所の追加、更新、または削除 |
| **被扶養者の変更** | <ul><li>作業者 > プロファイル > 個人情報 > 個人の連絡先 > 被扶養者の追加または削除</li><li>従業員セルフ サービス</li></ul> | 被扶養者は追加または削除されます。 個人の連絡先関係は、子供、配偶者、ドメスティック パートナー、または以前の配偶者である必要があります。 **発効** 日を更新することによりライフ イベントをトリガーします。 日付を更新しない場合、ライフ イベントはトリガーされません。 |

| **補償の喪失 (配偶者 / ドメスティック パートナー)** | 作業者 > プロファイル > 個人情報 > 個人の連絡先 > 被扶養者の詳細 > 補償の喪失 | **有効日** と共に、**補償の喪失** が個人の連絡先に対して選択されます |
| ドメスティック パートナー雇用の変更 | 作業者 > プロファイル >個人情報 > 個人の連絡先 > 被扶養者の詳細 > 雇用済。 | <ul><li>被扶養者の詳細レコードの作成と **雇用された個人の連絡先** ボックス = はい</li><li>**雇用された個人情報** ボックスの変更 (はいまたはいいえ)</li></ul> |
| **休暇 (配偶者/ドメスティック パートナー)** | 作業者 > プロファイル > 個人情報 > 個人の連絡先 > 被扶養者の詳細 > 休暇 | <ul><li>被扶養者の詳細レコードの作成され、**EhrLOAEffectiveDate** が入力されます</li><li>**personPrivateDetails.EhrIsLOA** が変更されます (はいまたはいいえ)</li><li>**personPrivateDetails.EhrLOAEffectiveDate** が変更されます</li></ul> |
| **補償の変更 (職位)** | <ul><li>作業者 > 職位割り当て > 作業者の職位の割り当て</li><li>職位 > 職位</li></ul> | <ul><li>作業者の職位割り当てレコードで職位に変更します</li><li>職位で作業者割り当ての変更</li></ul> |
| **メディケア (従業員 / 被扶養者)** | 作業者 > プロファイル > 個人情報 > 個人の連絡先 > 被扶養者の詳細 > 休暇> メディケア有効日 | 個人の連絡先が有効日を入力した場合、自動的にはトリガーされません。 |
| **裁判所の命令によるサポート** | 作業者 > プロファイル > 個人情報 > 個人の連絡先 > 被扶養者 > 裁判所の命令によるサポート (QMSCO/QDRO と有効日 | 自動更新はトリガーされません。 適格性に影響されずに、ライフ イベントを記録します。 |
| **死亡** | 作業者 > プロファイル > 個人情報 > 死亡日 | 死亡日が入力されます |
| **保険の証拠** | <ul><li>作業者 > 作業者 > バージョン > 職歴 > 日付マネージャー > 給付金の詳細</li><li> 作業者 > 雇用 > 給付金の詳細 > 検証日</li></ul> | <ul><li>作業者は検証日を入力します</li><li>作業者は EvidenceOfInsurability フィールドを **はい** に設定します</li></ul> |

| **作業者の適格性の変更 (米国固有ではありません)** | <ul><li>作業者 > 雇用</li><li>作業者 > 作業者 > バージョン > 職歴</li></ul> | <ul><li>従業員タイプ、雇用カテゴリ、または 5 人のユーザー定義可能な適格性フィールドが変更されます</li><li>**HcmEmploymentDetail.EhrEmploymentType** の変更は (*新規* 雇用レコードに対してではなく、再雇用や退職などの、*変更した* 雇用詳細レコードに対してのみ処理されます)</li></ul> |
