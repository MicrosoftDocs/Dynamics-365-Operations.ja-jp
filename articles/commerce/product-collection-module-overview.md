# <a name="product-collection-modules"></a>製品収集モジュール  

[!include [banner](includes/preview-banner.md)]
[!include [banner](includes/banner.md)]

このトピックでは、Microsoft Dynamics 365 Commerce の製品収集モジュールの概要を示します。

## <a name="overview"></a>概要

製品の検出は、小売業者が E コマース Web サイト上で顧客を確保するために使用する主要なツールです。 製品収集モジュールは、すばやく製品収集を作成するのに使用できる直感的なビジュアル インターフェースを提供することにより、小売業者が魅力的なショッピング体験を構築するのに役立ちます。

製品収集モジュールは、Web サイトの物理的な製品とサービスを表します。 通常、製品収集モジュールは、顧客が製品またはサービスを購入したり、詳細情報を取得できる詳細ページにリンクされます。 

製品収集のソースには、3 つのタイプのリストがあります。

- 製品または製品リストに対して関連のある製品として Dynamics 365 Retail 内で手動で定義される製品の編集リスト
- 新製品、ベストセラー、またはトレンド製品のリストなどの、アルゴリズム リスト
- 機械学習に基づく推奨リスト

次の図は、E コマース サイトで使用されている異なるタイプの製品収集を示しています。

![E コマース サイトの異なるタイプの製品収集の例](./media/ProductCollectionsAcrossTheSiteUseProductPlacement.png)

> [!NOTE]
> 常に製品収集モジュールを使用して、類似のタイプまたはテーマの製品グループを表示します。

## <a name="product-collection-modules-and-types"></a>製品収集モジュールおよびタイプ

次の表で、Dynamics 365 Commerce におけるさまざまなタイプの製品収集モジュールについて説明します。

| 製品収集モジュール  | 型 | 説明 |
|----------------------------|------|-------------|
| カテゴリの参照            | 編集 | このタイプの製品収集モジュールは、小売業者が小売チャネルに対して作成したナビゲーション カテゴリ階層を使用し、特定のサイト カテゴリで提供される製品の参照フローを示します。 |
| 検索結果             | 検索クエリ | このタイプの製品収集モジュールは、顧客が入力した検索クエリに最も一致する製品のリストを表示します。 |
| 関連製品           | 編集 | このタイプの製品収集モジュールは、販売促進マネージャーが、作成者が選択した関係タイプに対して、Retail で関連製品としてコンフィギュレーションした製品のリストを表示します。 |
| キュレーション製品の一覧      | 編集 | このタイプの製品収集モジュールは、販売促進担当者および編集者が Retail で作成したカスタム リストを表示します。 |
| 新規                        | アルゴリズム | このタイプの製品収集モジュールは、チャネルおよびカタログに対して類別された最新の製品のリストを表示します。 |
| ベストセラー               | アルゴリズム | このタイプの製品収集モジュールは、最大販売注文数でランク付けされている製品のリストを表示します。 |
| トレンド                   | アルゴリズム | このタイプの製品収集モジュールは、指定期間において業績が最上位の製品のリストを表示します。 |
| よく一緒に購入される品目 | 人為的知能/機械学習 | このタイプの製品収集モジュールは機械学習を使用して消費者の購買パターンを分析し、指定された製品と共によく購入される関連品目を勧めます。 |
| その他のお勧め           | 人為的知能/機械学習 | このタイプの製品収集モジュールは機械学習を使用して消費者の購買パターンを分析し、指定された製品に関連する品目を勧めます。 |

## <a name="add-a-product-collection-module-to-a-category-page"></a>カテゴリ ページへの製品収集モジュールの追加

カテゴリ ページへ製品収集モジュールを追加するには、次の手順に従います。

1. Dynamics 365 Commerce でサイトに移動し、既定のカテゴリ ページと同じテンプレートを使用するページを作成します。
1. ページ アウトラインで**サブ フッター** スロットを選択し、省略記号ボタン (**...**) を選択してから、**モジュールの追加**を選択します。
1. **モジュールの追加**ダイアログ ボックスで**コンテナー**を選択し、**OK** をクリックします。
1. コンテナー モジュールで省略符号ボタンを選択し、**モジュールの追加**を選択します。
1. **モジュールの追加**ダイアログ ボックスで**製品収集**を選択し、**OK** をクリックします。
1. 適切なデータ ソースを選択し、製品収集に対して入力することにより、設定をコンフィギュレーションします。
1. 製品収集モジュールのプロパティ ウィンドウで、**製品リストの追加**を選択します。
1. **製品リストのコンフィギュレーションの選択**ダイアログ ボックスで、リストのタイプを選択し、品目の数を入力してから、リスト タイプに使用可能な他のオプションを選択します。 リスト タイプの詳細については、次の表を参照してください。 
1. **OK** を選択します。
1. ページを保存してチェックインする。

次の表に、**製品リストのコンフィギュレーションの選択**ダイアログ ボックスの選択に使用可能なリスト タイプを示します。
   
| 型                       | 説明 | 一般的なプラクティス | ページ コンテキストから派生できるコンテキスト | 作成者がページ コンテキストをオーバーライドできるコンテキスト |
|----------------------------|-------------|------------------|-------------------------------------|-----------------------------------------------|
| カテゴリ別製品       | 指定されたカテゴリに属する製品のリスト。 このカテゴリは、ページ コンテキストまたは作成者が提供するコンテキストから決定されます。 | リッチ カテゴリ ページ、ホーム ページ、チェックアウトとカートのページ、および製品ページ | カテゴリ | 作成者によって決定されたカテゴリ |
| 関連製品           | 販売促進マネージャーが、関係するタイプに対して Retail で関連製品としてコンフィギュレーションした製品のリスト。 | 製品ページ、チェックアウトとカートのページ、欲しい物のリストのページ、および顧客アカウントのページ | 製品、関係のタイプ (必須)  | 製品、関係のタイプ |
| キュレート                    | 販売促進担当者および編集者が Retail で作成したカスタム リスト。 | リッチ カテゴリ ページ、ホーム ページ、チェックアウトとカートのページ、および製品ページ | 適用できません | リスト選択 |
| アルゴリズム                | <ul><li>**新規** – チャネルおよびカタログに対して類別された最新の製品のリスト。</li><li>**ベストセラー** – 最大販売注文数でランク付けされている製品のリスト。</li><li>**トレンド** – 指定期間において業績が最上位の製品のリスト。</li></ul> | ホーム ページ、リッチ カテゴリ ページ、およびチェックアウトとカートのページ | カテゴリ | 作成者によって決定されたカテゴリ |
| よく一緒に購入される品目 | 消費者の購買パターンを分析し、指定された製品と共によく購入される関連品目を勧めるために、機械学習を使用するリスト。 | 製品ページ、およびチェックアウトとカートのページ | 製品、カート | カートを含める |
| その他のお勧め           | 消費者の購買パターンを分析し、指定された製品に関連する品目を勧めるために、機械学習を使用するリスト。 | 製品ページ、およびチェックアウトとカートのページ | 製品、カート | 適用できません |
