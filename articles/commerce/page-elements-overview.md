# <a name="page-model-glossary"></a>ページ モデルの用語集


[!include [banner](includes/banner.md)]

このトピックでは、Microsoft Dynamics 365 Commerce サイトのページで使用されるさまざまな要素について説明します。

## <a name="page-element-definitions"></a>ページ要素の定義

次の表で、サイトの概観、およびコンテンツを変更する際に理解しておく必要がある条件の概要を示します。 さらに詳細な説明および手順については、リンクを参照してください。

| 相談 | 説明およびメモ |
|------|-----------------------|
| [モジュール](work-with-modules.md) | <p>**定義:** モジュールは、Web ページのスケルトンの作成および構成をするための構成要素です。 ヘッダー、ヒーロー、およびカルーセル モジュールがその例です。</p><p>**選択される場所:** テンプレート、レイアウト、ページ、およびフラグメントの作成ステージなど、サイト作成のワークフローのさまざまなステージで、配置したモジュールを選択し、コンフィギュレーションすることができます。</p><p>**編集されている場所:** カスタム モジュールは、ソフトウェア開発キット (SDK) を使用し、コード内で作成されます。 次に、選択内容が利用可能になるサイトにアップロードされます。</p> |
| モジュールのプロパティ | <p>**定義:** モジュールのプロパティは、モジュールごとに定義される特定の設定です。 E コマース作成ツールで編集できます。 たとえば、モジュールのプロパティは、バナー モジュールのヘッダーと背景イメージを設定するために使用されます。</p><p>**コンフィギュレーションされる場所:** モジュールのプロパティは、テンプレート、レイアウト、ページ、フラグメント、およびアプリ設定の作成環境 (エディター) に表示されるプロパティ ウィンドウで選択し、コンフィギュレーションされます。</p> |
| [テンプレート](templates-layouts-overview.md) | <p>**定義:** テンプレートは、ページのカテゴリ (たとえば、マーケティング ページ、カテゴリ ページ、および製品ページ) に使用するモジュールの組み合わせおよびオプションを定義します。</p><p>**選択される場所:** ページまたはレイアウトの作成ワークフローの中でテンプレートを選択できます。</p><p>**編集される場所:** テンプレートはテンプレート エディターで作成します。 作成または変更にコードは必要ありません。</p> |
| [レイアウト](templates-layouts-overview.md) | <p>**定義:** レイアウトは、親テンプレートのオプション セットからの、最終的なモジュールの選択および配置を定義します。 レイアウトは、1 ページに対してコンフィギュレーションすることができ (*カスタム レイアウト*)、また複数のページごとに共有することもできます (*プリセット レイアウト*)。</p><p>**選択される場所:** レイアウトは、新しいページの作成時か、または既存のページに異なるレイアウトが必要な場合に、選択できます。</p><p>**編集される場所:** レイアウトはレイアウト エディターで作成します。 作成または変更にコードは必要ありません。</p> |
| [ページ インスタンス](modify-existing-page.md) | <p>**定義:** ページ インスタンスは、1 ページに対する最終的な、ページ固有のローカライズされたコンテンツを定義します。 このコンテンツは、モジュール プロパティの値から派生します。</p><p>**選択される場所:** ページは、URL が割り当てられる時に選択されます。</p><p>**編集される場所:** ページはページ エディターで編集します。 作成または変更にコードは必要ありません。</p> |
| [テーマ](select-site-theme.md) | <p>**定義:** テーマは、カスケード スタイル シート (CSS) を定義し、ページでレンダリングされるモジュールの概観を決定します。</p><p>**選択される場所:** テーマが Microsoft Dynamics Lifecycle Services (LCS) を使用してアップロードされた後、ページ コンテナー モジュールのプロパティとして選択できます。</p><p>**編集される場所:** 現在、テーマは SDK を使用して作成および編集できます。 次に、LCS を使用してサイトにアップロードされます。</p> |
| [フラグメント](work-with-fragments.md) | <p>**定義:** フラグメントは、コンテンツをローカライズし、完全にコンフィギュレーションしたモジュールで、再利用および複数のページ間での 1 か所の更新ができます。 たとえば、ヘッダー モジュールから作成されたフラグメントは、すべてのテンプレート内およびサイト間のすべてのページ上で使用でき、1 か所で更新できます。</p><p>**選択される場所:** フラグメントは、モジュールを選択できる任意の場所で選択できます。 モジュールが、再利用可能な一括作成を通して効率を向上させるために、代用することができます。</p><p>**編集される場所:** フラグメントはフラグメント エディターで編集します。 作成または変更にコードは必要ありません。</p> |
| [URL](create-page-URL.md) | <p>**定義:** ユニフォーム リソース ロケータ (URL) は、Web ページまたは他の URL をポイントするアドレスです。</p><p>**選択される場所:** URL は、ページ間のリンクが必要な場合に選択されます。</p><p>**編集される場所:** URL は URL エディターで編集します。 作成または変更にコードは必要ありません。</p> |
| 資産 | <p>**定義:** 資産とは、.jpg、.docx、.pdf、または .mpg などの拡張子を持つファイル バイナリです。</p><p>**選択される場所:** 資産は、必要とされるモジュールに対して、モジュール プロパティとして選択されます。</p><p>**編集される場所:** 資産はアップロードされ、関連するメタ データは資産マネージャーで編集されます。</p> |
