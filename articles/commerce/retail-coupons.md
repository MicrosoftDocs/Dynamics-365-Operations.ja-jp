> <span data-ttu-id="1ef12-116">クーポンが割引にリンクされると、コマースの割引ページのいくつかのフィールドは、クーポンの設定で管理されているため読み取り専用になります。</span><span class="sxs-lookup"><span data-stu-id="1ef12-116">After a coupon is linked to a discount, several fields on the discount page in Commerce become read-only, because they are managed by the coupon's settings.</span></span> <span data-ttu-id="1ef12-117">これらのフィールドには、ステータスと標準の日付範囲のフィールドが含まれます。</span><span class="sxs-lookup"><span data-stu-id="1ef12-117">These fields include the fields for the status and standard date ranges.</span></span>

### <a name="limited-use-coupons"></a><span data-ttu-id="1ef12-118">専用クーポン</span><span class="sxs-lookup"><span data-stu-id="1ef12-118">Limited-use coupons</span></span>

<span data-ttu-id="1ef12-119">専用クーポンとして、クーポンを構成することができます。</span><span class="sxs-lookup"><span data-stu-id="1ef12-119">Coupons can be configured as limited-use coupons.</span></span> <span data-ttu-id="1ef12-120">使用制限は、顧客またはチャンネルごと、またはグローバルな制限として定義できます。</span><span class="sxs-lookup"><span data-stu-id="1ef12-120">The usage limit can be defined per customer or channel, or as a global limit.</span></span> <span data-ttu-id="1ef12-121">POS または販売注文入力時にコードまたはバーコードを入力またはスキャンする際に、この制限が適用されます。</span><span class="sxs-lookup"><span data-stu-id="1ef12-121">This limit is enforced when the code or bar code is entered or scanned in POS or during sales order entry.</span></span>

<span data-ttu-id="1ef12-122">制限は、クーポンのクーポン コードごとに適用されます。</span><span class="sxs-lookup"><span data-stu-id="1ef12-122">The limit is enforced per coupon code on a coupon.</span></span> <span data-ttu-id="1ef12-123">たとえば、2 つのクーポン コードを持つ 1 回のみ使用のクーポンは、各クーポン コード 1 回で 2 回使用できます。</span><span class="sxs-lookup"><span data-stu-id="1ef12-123">For example, a single-use coupon that has two coupon codes can be used two times: one time for each coupon code.</span></span> <span data-ttu-id="1ef12-124">クーポンの各コードは、個別に有効に設定できます。</span><span class="sxs-lookup"><span data-stu-id="1ef12-124">Each code on a coupon can be independently set to active.</span></span>

<span data-ttu-id="1ef12-125">クーポンはすべての販売チャンネルで使用することができますが、コールセンターで注文を行う場合は、通話センターの **注文完了** 設定が有効になっている通話センターの注文に対してのみ、限定使用のクーポンを使用できます。</span><span class="sxs-lookup"><span data-stu-id="1ef12-125">The coupons can be used across any selling channel however, for call center orders, the limited use coupons can be used for only those call centers orders where the **Order completion** setting on the call center is enabled.</span></span> <span data-ttu-id="1ef12-126">これが有効になっていない場合、限定されない使用タイプのクーポンのみをコール センター オーダーで使用できます。</span><span class="sxs-lookup"><span data-stu-id="1ef12-126">If this is not enabled, then only non-limited use type coupons can be used in call center orders.</span></span>

> [!NOTE]
> <span data-ttu-id="1ef12-127">クーポン コードが使用制限に達した後、システムが自動的にクーポン コードの状態を „使用済み„ に変更することは *ない* です。</span><span class="sxs-lookup"><span data-stu-id="1ef12-127">After a coupon code has reached its usage limit, the system does *not* automatically change the status of the coupon code to “Used".</span></span> <span data-ttu-id="1ef12-128">ただし、このクーポン コードは使用制限に達しているため使用できません。</span><span class="sxs-lookup"><span data-stu-id="1ef12-128">However that coupon code has reached its usage limit and cannot be used.</span></span> <span data-ttu-id="1ef12-129">クーポン コードの状態が手動で **有効** 以外のもに設定されている場合、すべてのチャネルでこのクーポン コードを使用できません。</span><span class="sxs-lookup"><span data-stu-id="1ef12-129">If the status of a coupon code is manually set to anything other than **Active**, then this coupon code cannot be used in any channel.</span></span>

## <a name="system-setup-for-coupons"></a><span data-ttu-id="1ef12-139">クーポンのシステム設定</span><span class="sxs-lookup"><span data-stu-id="1ef12-139">System setup for coupons</span></span>

<span data-ttu-id="1ef12-140">クーポンを設定する前に、クーポンのバーコードおよび 2 つのクーポン番号順序を設定する必要があります。</span><span class="sxs-lookup"><span data-stu-id="1ef12-140">Before you can set up a coupon, you must set up the coupon bar code and two coupon number sequences.</span></span>

## <a name="the-effect-of-partial-updates-on-coupons"></a><span data-ttu-id="1ef12-156">クーポンに対する部分的な更新の影響</span><span class="sxs-lookup"><span data-stu-id="1ef12-156">The effect of partial updates on coupons</span></span>

<span data-ttu-id="1ef12-157">クーポン機能は、複数の異なる機能で構成されています。</span><span class="sxs-lookup"><span data-stu-id="1ef12-157">Coupon functionality comprises multiple distinct features.</span></span> <span data-ttu-id="1ef12-158">コマース バックオフィス (HQ) とチャンネルは、コンポーネント間で部分的に更新することができます。</span><span class="sxs-lookup"><span data-stu-id="1ef12-158">Commerce Headquarters (HQ) and the channel can be partially updated across components.</span></span> <span data-ttu-id="1ef12-159">したがって、部分的な更新が全体としてのクーポン機能にどれほど影響するのか理解することが重要です。</span><span class="sxs-lookup"><span data-stu-id="1ef12-159">Therefore, it's important that you understand how partial updates affect coupon functionality as a whole.</span></span>

- <span data-ttu-id="1ef12-160">**HQ は部分的に更新されますが、Commerce Scale Unit と POS は更新されません。**</span><span class="sxs-lookup"><span data-stu-id="1ef12-160">**HQ is partially updated, but Commerce Scale Unit and POS aren't updated.**</span></span> <span data-ttu-id="1ef12-161">HQ 更新では、クーポンと割引ページが更新され、コマース価格エンジンも更新されます。</span><span class="sxs-lookup"><span data-stu-id="1ef12-161">In an HQ update, the coupon and discount pages are updated, and the commerce price engine is also updated.</span></span> <span data-ttu-id="1ef12-162">これら 2 つのコンポーネントの一方だけが更新された場合、コマースの一部のページで価格計算データが一致しなくなります。</span><span class="sxs-lookup"><span data-stu-id="1ef12-162">If only one of those two components is updated, some pages in Commerce won't match the price calculation data.</span></span> <span data-ttu-id="1ef12-163">そのため、予期しない割引の計算やエラーが、割引の計算時に発生する可能性があります。</span><span class="sxs-lookup"><span data-stu-id="1ef12-163">Therefore, unexpected discount calculations or errors might occur during discount calculations.</span></span>

- <span data-ttu-id="1ef12-169">**HQ は更新されませんが、Commerce Scale Unit と POS は更新されます (N-1)。**</span><span class="sxs-lookup"><span data-stu-id="1ef12-169">**HQ isn't updated, but Commerce Scale Unit and POS are updated (N+1).**</span></span> <span data-ttu-id="1ef12-170">Commerce Scale Unit で更新された価格エンジンで価格の計算時にレガシ割引コードを処理できるので、このシナリオでは更新による機能の影響はありません。</span><span class="sxs-lookup"><span data-stu-id="1ef12-170">Because the updated price engine in Commerce Scale Unit can handle legacy discount codes during price calculations, the update should have no functional impact in this scenario.</span></span>
