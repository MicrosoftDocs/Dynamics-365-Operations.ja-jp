<span data-ttu-id="8d9b4-106">コンテナー モジュールは、その中の他のモジュールをホストするモジュールです。</span><span class="sxs-lookup"><span data-stu-id="8d9b4-106">A container module is a module that hosts other modules inside it.</span></span> <span data-ttu-id="8d9b4-107">コンテナー モジュールの主な目的は、それに対して設定されているプロパティを通じて、内部に含まれるモジュールのレイアウトを定義することです。</span><span class="sxs-lookup"><span data-stu-id="8d9b4-107">The primary purpose of a container module is to define, through the properties that are set for it, the layout of the modules that it contains.</span></span> <span data-ttu-id="8d9b4-108">たとえば、それらのモジュールは 2 列、3 列、4 列、または 6 列のレイアウトで左右に並んで表示できます。</span><span class="sxs-lookup"><span data-stu-id="8d9b4-108">For example, those modules can appear side by side in a two-column, three-column, four-column, or six-column layout.</span></span> <span data-ttu-id="8d9b4-109">また、コンテナーの幅に限定することも、画面全体に表示することもできます。</span><span class="sxs-lookup"><span data-stu-id="8d9b4-109">They can also be limited to the width of the container, or they can fill the screen.</span></span> <span data-ttu-id="8d9b4-110">ヘッダーは、各コンテナー モジュールに追加することもできます。</span><span class="sxs-lookup"><span data-stu-id="8d9b4-110">A heading can also be added to every container module.</span></span>
