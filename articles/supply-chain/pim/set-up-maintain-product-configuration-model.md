# <a name="set-up-a-product-configuration-model"></a>製品コンフィギュレーション モデルを設定します。

[!include[banner](../includes/banner.md)]


この記事は、製品コンフィギュレーション モデルを設定および作成するステップについて説明します。

| タスク                                                        | 説明                                                                                                                                                                                                                                                                                                                                                                                        |
|-------------------------------------------------------------|----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| 製品マスターを作成します。                                    | **製品マスター**一覧から製品マスターを作成します。 すべての関連会社に製品マスターをリリースします。 製品コンフィギュレーション モデルのバージョンや、サブコンポーネントとして使用する製品マスターには、**制約ベースのコンフィギュレーション** をコンフィギュレーション テクノロジとして選択する必要があります。また、コンフィギュレーション分析コードは、製品分析コード グループにのみ選択される必要があります。 |
| コンポーネントを作成します。                                          | **コンポーネント** ページでコンポーネントを作成します。 コンポーネントは、製品コンフィギュレーション モデルの構成要素であり、複数の製品コンフィギュレーション モデルで再利用できます。                                                                                                                                                                                                                      |
| 属性タイプを作成します。                                     | **属性タイプ** ページで属性タイプを作成します。 属性タイプは、製品コンフィギュレーション モデルで使用されているすべての属性のデータ型のセットを指定します。 **ブール値**属性、固定リストのある**テキスト**属性、範囲タイプのある**整数**属性は、製品コンフィギュレーション モデルに基づいて製品バリアントのコンフィギュレーション時に使用できる一連の値を一覧します。       |
| 製品コンフィギュレーション モデルを作成します。                       | **新しい製品コンフィギュレーション モデル** ページで、製品コンフィギュレーション モデルを作成します。                                                                                                                                                                                                                                                                                                              |
| 製品コンフィギュレーション モデルに属性を追加します。            | **制約ベースの製品コンフィギュレーション モデルの詳細**ページの**属性**クイック タブで、属性を作成します。 属性は、製品コンフィギュレーション モデルの機能を表します。                                                                                                                                                                                                       |
| 製品コンフィギュレーション モデルに制約を追加します。           | **制約ベースの製品コンフィギュレーション モデルの詳細**ページの**制約**クイック タブで、制約を作成します。 制約とは、製品コンフィギュレーションで満たす必要がある制限です。 制約は、式の制約またはテーブルの制約です。                                                                                                                                 |
| 製品コンフィギュレーション モデルにサブコンポーネントを追加します。         | **制約ベースの製品コンフィギュレーション モデルの詳細**ページの**サブコンポーネント**クイック タブで、サブコンポーネントを作成します。 サブコンポーネントはコンポーネントに関連付けられ、サブコンポーネントを表す品目にリンクされます。                                                                                                                                                                       |
| 製品コンフィギュレーション モデルにユーザー要件を追加します。     | サブコンポーネントに似ていますが、ユーザー要件は品目を参照しません。 ユーザー要件は、ファントム BOM と考えることができます。 ユーザー要件の下に直接設定されている BOM 明細行や工順工程は、親コンポーネントに折りたたまれます。                                                                                                                       |
| 製品コンフィギュレーション モデルに BOM 明細行を追加します。             | **制約ベースの製品コンフィギュレーション モデルの詳細**ページの**BOM 明細行**クイック タブで、BOM 明細行を作成します。 BOM 明細行は、製品バリアントの作成に必要な部品を表します。                                                                                                                                                                                                 |
| 製品コンフィギュレーション モデルに工順工程を追加します。      | **制約ベースの製品コンフィギュレーション モデルの詳細**ページの**工順工程**クイック タブで、工順工程を作成します。 工順工程は、製品バリアントの作成に実行される手順の順序で手順を表します。                                                                                                                                                    |
| 製品コンフィギュレーション モデルに有効なバージョンを作成します。 | **バージョン**ページで、製品コンフィギュレーション モデルに有効なバージョンを作成します。 バージョンとは、製品コンフィギュレーション モデルと製品マスター間の関係です。 有効なバージョンを持つ製品コンフィギュレーション モデルは、販売注文、販売見積、発注書、または製造オーダーからコンフィギュレーションできます。                                                               |
| 製品コンフィギュレーション モデルをテストします。                         | **制約ベースの製品コンフィギュレーション モデルの詳細**ページまたは**製品コンフィギュレーション モデルの一覧**ページから、製品コンフィギュレーション モデルをテストします。 製品コンフィギュレーション モデルをテストすると、注文の処理中に発生するプロダクト モデルのコンフィギュレーション プロセスがシミュレーションされます。                                                                                                |
| 製品コンフィギュレーション モデル テンプレートを作成します。                | **コンフィギュレーション テンプレート** ページで、製品コンフィギュレーション モデルのテンプレートを作成します。 コンフィギュレーション テンプレートには、製品コンフィギュレーション モデルの属性値が含まれます。 **コンフィギュレーション明細** ページで、属性値を選択します。 製品モデルのコンフィギュレーション中に、製品モデルのコンフィギュレーション テンプレートを読み込むように選択できます。                                                   |
| 品目をコンフィギュレーションします。                                          | 製品コンフィギュレーション モデルは、販売注文、販売見積、発注書、または製造オーダーからコンフィギュレーションできます。                                                                                                                                                                                                                                                                           |
