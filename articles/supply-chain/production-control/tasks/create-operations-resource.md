# <a name="create-an-operations-resource"></a>運営リソースの作成

[!include [banner](../../includes/banner.md)]

プロジェクト活動または生産プロセスを実行する運営リソース。 この手順では、運営リソースを定義する方法について説明します。 デモ データ会社 USMF または独自のデータを使用してこの手順の説明を見ることができます。

1. [リソース] に移動します。
2. [新規] をクリックします。
3. [リソース] フィールドに値を入力します。
4. [説明] フィールドに値を入力します。

## <a name="define-capacity-and-consumption-parameters"></a>能力と消費パラメータの定義
1. [操作] セクションを展開します。
2. [仕損率] フィールドに数値を入力します。
3. [段取りカテゴリ] フィールドで、値を入力または選択します。
    * 設定の原価を計算する方法を定義する原価カテゴリを指定します。  
4. [実行時間カテゴリ] フィールドで、値を入力または選択します。
    * 実行時間の原価を計算する方法を定義する原価カテゴリを指定します。  
5. [数量カテゴリ] フィールドで、値を入力または選択します。
    * 出来高数量に基づいてリソース コストを説明する方法を定義する原価カテゴリを指定します。  
6. [能力単位] フィールドで、オプションを選択します。
    * 運営リソース能力の表示単位を指定します。  
7. [能力] フィールドに数値を入力します。
8. [効率] フィールドに、数値を入力します。
    * その運営リソースから期待される効率を指定します。 効率はリソースのスループットを調整します。これにより、運営リソースに対して予約される時間に影響を与えます。  
9. [工程のスケジューリング率] フィールドに数値を入力します。
    * 工程のスケジューリングで使用する運営リソース能力の最大割合を指定します。  
10. [有限能力] フィールドで [はい] を選択します。
    * 利用可能な実際の能力に基づいてスケジュールを立てる場合、かつ既存の確保済能力を考慮する場合は、このオプションを [はい] にします。 このオプションが [いいえ] に設定されている場合、運営リソースには無限の能力があるとされ、リソースが予約超過になることがあります。  
11. [ボトルネック リソース] フィールドで、[はい] を選択します。

## <a name="define-working-times"></a>作業時間の定義
1. [カレンダー] セクションを展開します。
2. [追加] をクリックします。
3. [カレンダー] フィールドで値を入力または選択します。
    * リソースの能力 (時間) で定義した作業時間カレンダーを指定します。  
4. 一覧で、目的のレコードを見つけ、選択します。
5. 一覧で、選択された行のリンクをクリックします。

## <a name="define-resource-capabilities"></a>リソース能力の定義
1. [処理能力] セクションを展開します。
2. [追加] をクリックします。
    * 能力とは、運営リソースが特定の活動を実行する能力です。 スケジューリング エンジンは、運営リソースの利用可能な能力と各活動のリソース要件が一致するようにリソースを割り当てます。  
3. [能力] フィールドで、値を入力または選択します。
4. [レベル] フィールドに数値を入力します。
    * リソースが能力を処理する実力レベルを指定します。  
5. [優先順位] フィールドに数値を入力します。
    * 能力に基づいて、運営リソースの優先順位を指定します。 優先順位のスケジューリングの際は、最も優先順位の高い運営リソース (最小の数字) が最初に選択されます。  

## <a name="assign-resource-to-resource-group"></a>リソース グループへのリソース割り当て
1. [リソース グループ] セクションを展開します。
2. [追加] をクリックします。
    * リソース グループとは、運営リソースのサイト、生産単位、および倉庫の環境です。 運営リソースは、リソース グループに割り当てられている時、かつそのリソース グループが置かれているサイトでのみスケジューリングできます。  
3. [リソース グループ] フィールドで、値を入力または選択します。
4. [入庫場所] フィールドで、値を入力または選択します。
    * 運営リソースが消費する材料を格納している倉庫の場所を指定します。
