# <a name="define-production-flow-models"></a>生産フロー モデルの定義

[!include[task guide banner](../../includes/task-guide-banner.md)]

生産フロー モデルは、リーン生産作業セルの能力の、計算および管理方法を示しています。 したがって、生産フロー モデルの定義は、作業セルの定義の前提条件となります。 この手順の作成に使用するデモ データの会社は USMF です。


## <a name="define-a-production-flow-model"></a>生産フロー モデルを定義します。 
1. [生産管理] > [設定] > [リーン生産フロー] > [生産フロー モデル]に移動します。
2. [新規] をクリックします。
3. [生産フロー モデル] フィールドで、生産フロー モデルの ID を入力します。
4. [モデル タイプ] フィールドで、オプションを選択します。
    * 2 つのモデル タイプ [スループット タイプ] と [時間タイプ] があります。 [スループット タイプ] では、この生産フロー モデルを使用する作業セルの能力は、製品数量で表され、算出されます。 [時間タイプ] では、この生産フロー モデルを使用する作業セルの能力は、時間で表され、算出されます。 このプロパティは、既存の生産フロー モデルに変更できないことに注意してください。 作業セルが確保済能力を保持したら、生産フロー モデル タイプは変更できません。  
5. [EPE サイクル] に日数を入力します。
    * 間隔は、生産フローまたは作業セルによって生産されるすべてのパーツが少なくとも一度生産される期間について説明します。 EPE サイクルが短ければ短いほど、生産プロセスはより柔軟になります。 EPE サイクルが 0 の場合、すべての需要を同じ日に作成することができます。 EPE サイクルは、リーン プロセス ジョブをスケジュールするのに使用されます。 EPE サイクルが 5 の作業セルのジョブをスケジュールするときは、スケジューリング プロセスは、製品がそのサイクルで生産できるように、[期日 – EPE サイクル] (期日より 5 日先) の作業セルの能力を検索します。 在庫の製品リード タイムは、通常、関連する生産プロセスの EPE サイクルと等しいか、またはより大きくなります。  
6. [計画期間タイプ] フィールドで、オプションを選択します。
    * 作業セルが確保済能力を保持したら、計画期間タイプは変更できません。 この特定の作業セルには、同じ期間タイプの生産フロー モデルのみ選択できます。  
7. [計画タイム フェンス] フィールドで、数値を入力します。
    * 計画タイム フェンスは、関連する作業セルに能力の予約を作成できる日数について説明します。 [計画タイム フェンス] で、日数を入力します。   この期間外にあるかんばんプロセス ジョブは、自動計画では計画されません。 通常、計画タイム フェンスは、生産フローまたは作業セルで生産した製品の在庫の平均リード タイムの 2 倍です。 EPE サイクルが計画タイム フェンスの半分を超えることはできません。     
8. [能力不足に対する反応] フィールドで、オプションを選択します。
    * オプションには次のものが含まれます:   [延期] - 利用可能なスループットで、次に作業可能な生産日のスケジューリング イベントの全需要を延期します。 [キャンセル] - スケジューリング イベントの自動計画を終了し、関連するジョブを予定外にしておきます。   [要求された日に追加] - 要求された期間に要求されたジョブを計画します。 これによりこの日のセルはオーバーロードし、プランナーは手動インタラクションを確認する必要があります。   [作業可能な期間への配分] - 最初の作業可能日から始めて、スケジューリング イベントのさまざまなジョブをすべての作業可能な生産日に対して配分します。 最少配分数量は、かんばん作業数量です。 この配分により、最少計画数量 (かんばん数量) が十分な利用可能のスループットで各日に割り当てられます。
