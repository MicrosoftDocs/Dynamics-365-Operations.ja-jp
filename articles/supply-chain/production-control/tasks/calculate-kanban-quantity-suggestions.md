# <a name="calculate-kanban-quantity-suggestions"></a>かんばん数量修正候補の計算

[!include [banner](../../includes/banner.md)]

この手順では、かんばん数量計算を使用して、特定のかんばんルールに対してかんばんのサイズおよび数量を最適化することに焦点をあてます。 この手順の作成に使用するデモ データの会社は USMF です。 この手順は、バリュー ストリーム マネージャーを対象としています。 「かんばん数量計算ポリシーをかんばんルールへ追加する」という手順を事前に完了しておく必要があります。


## <a name="create-a-kanban-quantity-calculation"></a>かんばん数量計算の作成
1. [生産管理] > [定期処理タスク] > [かんばん数量の計算] > [かんばん数量の計算] の順に移動します。
2. [新規] をクリックします。
3. [名前] フィールドに、「Speaker2016」と入力します。
4. [名前] フィールドで、ドロップ ダウン ボタンをクリックし、ルックアップを開きます。
    * 「かんばん数量計算ポリシーをかんばんルールへ追加する」という手順に基づいて作成されたポリシーを選択します。 たとえば、「Speaker2016」です。  
5. 一覧で、選択された行のリンクをクリックします。
6. [現在の日付で有効なルール] フィールドで、日時を「2012-12-17T08:00:00」と設定します
    * この日付は、かんばん数量計算に含まれている固定かんばんルールを特定する基準として使用されます。  
7. [履行済需要期間の開始日] フィールドで、日時を「2012-11-17T09:00:00」と設定します
    * 過去の需要トランザクションを含めてかんばん数量を計算する開始日です。  
8. [履行済需要期間の終了日] フィールドで、日時を「2012-12-17T07:59:59」と設定します。
    * 過去の需要トランザクションを含めてかんばん数量を計算する最終日です。  
9. [需要期間の開始日] フィールドで、日時を「2012-12-17T08:00:00」と設定します
    * 現在の需要トランザクションを含めてかんばん数量を計算する開始日です。  
10. [需要期間の最終日] フィールドで、日時を「2013-01-16T07:59:59」と設定します
    * 現在の需要トランザクションを含めてかんばん数量を計算する最終日です。  

## <a name="generate-kanban-quantity-proposal"></a>かんばん数量提案に生成
1. [保存] をクリックします。
2. [生成] をクリックします。
    * これにより、かんばんルール 000020 のかんばん数量提案明細行が生成されます。  

## <a name="run-kanban-quantity-calculation"></a>かんばん数量計算の実行
1. [計算] をクリックします。
    * これにより、かんばん数量の提案が計算されます。  
2. [OK] をクリックします。
3. 一覧で、選択された行をマークします。
    * 提案されたかんばん数量が 2.であることに注意します。  

## <a name="change-product-quantity-and-calculate-again"></a>製品数量および再計算の変更
1. 製品数量を「5」と設定します。
2. [計算] をクリックします。
3. [OK] をクリックします。
    * かんばん数量 5 からかんばん数量 4 に変更することが推奨されています。  
    * これは、より少ない製品数量においては、その需要を満たすためにさらなるかんばんが必要となることが原因です。  

## <a name="update-kanban-rule"></a>かんばんルールの更新
1. [ルールの有効日] フィールドに日時を入力します。
    * 「現在の日付で有効なルール」を将来の日付に設定します。 たとえば、今日 + 1 年です。  
2. [更新] をクリックします。
3. [OK] をクリックします。
4. ページを閉じます。

## <a name="validate-change-on-kanban-rule"></a>かんばんルールの変更の検証
1. [製品情報管理] > [リーン生産] > [かんばんルール] の順に移動します。
2. 一覧で、選択された行のリンクをクリックします。
    * 以前のサブタスクで作成されたかんばんルールを選択します。 これは、数字で並べ替えた一覧にある最初のかんばんルールです。  
3. [詳細] セクションの展開を切り替えます。
    * 有効日に注意します。この日付までこのルールが有効化されないことを意味します。  
4. [数量] セクションの展開を切り替えます。
    * これはかんばん数量計算に入力した既定の数量であることを確認します。  
    * これは、かんばん数量計算から算出した固定かんばん数量 4 です。  
5. ListPanel タブをクリックします。
