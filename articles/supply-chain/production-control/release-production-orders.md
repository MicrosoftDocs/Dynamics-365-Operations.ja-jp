# <a name="release-production-orders"></a>製造オーダーのリリース

[!include [banner](../includes/banner.md)]

リリースされた製造オーダーは、生産が承認された注文です。 生産の作業現場と倉庫でのプロセス実行に使用でき製造オーダーには、製造オーダーのライフ サイクルの状態を説明するのに "リリース済" という語句が使用されます。

## <a name="characteristics-of-the-released-state"></a>リリース済ステータスの特性

**リリース済** ステータスは、製造オーダーのライフ サイクルの 1 つの状態です。 **リリース済** 状態にある製造オーダーは、生産の作業現場と倉庫でのプロセス実行に使用できます。 **リリース済** ステータスは次の特徴があります。

- 製造オーダーは、製造オーダーから、またはバッチ処理を使用して、**リリース済** ステータスに変更できます。 また、製造オーダーは、**マスター プラン** ページの **確定タイム フェンス** フィールドを使用して確定される計画製造オーダーにより、自動的に更新することもできます。
- **リリース済** ステータスは、作業現場の操作員に、作業現場での生産ジョブの実行を開始してもらうための合図です。
- 工順カード、工順ジョブ、ジョブ カードなど、生産ジョブについての情報を提供する生産文書を発行することができます。
- 現物引当済である材料の場合、倉庫の作業が生成され、製造オーダーに対する材料のピッキングを行います。

## <a name="releasing-jobs-to-the-shop-floor"></a>作業現場へのジョブのリリース

製造オーダーがリリースされた後、注文に関連付けられる生産ジョブが表示され、登録が実行できるようになります。 操作員は、**ジョブ カード ターミナル** ページまたは **ジョブ カード デバイス** ページのいずれかで、開始、停止、完了などのジョブ登録を行うことができます。 登録された時間と数量は、登録ページから生産仕訳帳に自動的に転送され、消費時間と数量が記録されます。

## <a name="route-cards"></a>工順カード

工順カードには、工順と工程の設定から、および開始工程とジョブのスケジュール方法から取得された情報の概要が記載されます。

## <a name="route-jobs"></a>工順ジョブ

工順ジョブには工程の各ジョブの詳細が一覧表示され、段取り時間、処理時間、待ち時間、配送時間などが含まれます。 たとえば、塗装などの工程では、段取り時間、塗装プロセスの実行時間、塗装が乾くまでの待ち時間などの、個々のジョブが必要になる場合があります。

## <a name="job-cards"></a>ジョブ カード

ジョブ カードには、特定の工程を示す個々のジョブ番号が一覧表示されます。 各ページに 1 種類のジョブが表示されます。 ジョブ カードに含まれるジョブとその見積時間は、工順および工程の設定情報から取得されます。 ジョブ カードから、**生産仕訳帳明細行**、**ジョブ カード** ページを開くことができます。 運営リソースの運用担当者は、生産プロセスに関するフィードバックを提供できます。 消費統計や、エラー数量などの情報を入力できるフィールドがあります。

## <a name="warehouse-work-for-raw-material-picking"></a>原材料ピッキングの倉庫作業

原材料ピッキングの作業がリリース時に生成されます。 注文がリリースされる前に、製造オーダーのために引当された現物品目の数量に対してのみ作業が生成されます。 原材料ピッキングの倉庫作業を生成するには、以下の設定が必要です。

- 材料をピッキングする倉庫の場所を決定する、原材料ピッキングの場所のディレクティブ
- 倉庫作業の実行ポリシーを構成する、原材料のウェーブ テンプレート
- 材料をプットする場所を決定する、生産入庫の場所

ライセンス プレートで管理された場所を使用する場合、原材料のピッキング作業の処理中に、品目の注文済数量または使用可能な全数量のピッキングを行うかどうかを選択できます。 このオプションを設定するには、次のようにします。

1. **製品情報管理 \> 製品 \> リリース済製品** の順に移動して、関連品目を開きます。
1. **倉庫** クイック タブを展開します。
1. **ライセンス プレートの場所での材料ピッキング** フィールドで、次のいずれかのオプションを選択します。
    - *注文ピッキング*: 注文数量のみを選択する必要があります。
    - *ステージング*: 可能な限り、ライセンス プレートで入手可能な全数量を選択する必要があります。 作業員がライセンス プレートの全数量を選択するには、ライセンス プレートに混合品目を含めず、分析コードが混合しているべきではありません。 ライセンス プレートに分析コードや品目が混合して含まれている場合、ピッキングは *注文ピッキング* に設定されている場合と同様に処理されます。
