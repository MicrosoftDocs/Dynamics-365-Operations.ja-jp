# <a name="running-average-cost-price"></a>移動平均原価価格

[!include [banner](../includes/banner.md)]

在庫原価計算プロセスでは、品目の品目モデル グループで選択した在庫評価方法に基づいて、払出トランザクションを受入トランザクションへと決済します。 ただし、在庫決算を実行する前に、システムは、払出トランザクションの転記時に通常使用される移動平均原価価格を計算します。

システムは、次の式を使用して、品目に対するこの移動平均原価価格を見積もります。

見積価格 = (現物金額 + 財務金額) ÷ (現物数量 + 財務数量)

## <a name="default-item-cost"></a>既定の品目原価

リリースされた製品の既定の品目原価は、**リリースされた製品の詳細** ページで次のいずれかの方法で構成できます。

- アクション ウィンドウの **原価の管理** タブで、**設定** グループから **品目価格** を選択して、標準原価を作成します。 この方法を使用する場合は、標準原価バージョンを使用する必要があります。また、原価を有効化する必要があります。
- リリースされた製品の既定の品目原価価格を定義するには、**原価管理** クイック タブの **価格** フィールドに値を入力します。

価格の入力または作成に加えて、**リリースされた製品の詳細** ページの **原価の管理** クイック タブで **最新の原価価格を使用** チェック ボックスをオンにすることもできます。 この場合、財務更新を転記すると、**価格** フィールドが自動的に更新されます。 たとえば、発注請求書を転記する場合、このフィールドは請求書からの購買価格に設定されます。

有効な原価バージョンに原価価格を入力した場合、**原価の管理** クイック タブに価格を入力すると、システムは有効な原価バージョンの価格を使用して、その価格が **原価の管理** クイック タブで定義された価格を使用します。

## <a name="using-the-running-average-cost-price"></a>移動平均原価価格の使用

次の表では、システムが在庫トランザクションの転記に、移動平均原価価格を使用する場合と、品目マスタ レコードで定義されている原価価格を代わりに使用する場合を示します。

| 条件 | システムは見積もられた移動平均原価価格を使用 | システムは既定の品目原価価格を使用します |
| --- | --- | --- |
| 分子\* と分母\*\* が両方とも正です。 | あり | 番号 |
| 分子\* と分母\*\* のどちらか一方または両方が負です。 | 無 | 有 |
| 分母\*\* が 0 (ゼロ) です。 | 無 | 有 |

\*分子 = (現物金額 + 財務金額)  
\*\*分母 = (現物数量 + 財務数量)

> [!NOTE]
> 品目に対して **現物価格を含める** オプションが選択されていない場合、システムは現物金額と現物数量の両方に 0 (ゼロ) を使用します。 このオプションの詳細については、「[現物価格を含める](include-physical-value.md)」を参照してください。

## <a name="avoiding-pricing-amplification"></a>価格決定増幅の回避

あまりないことですが、システムが、価格の基になる入庫が十分な数になる前に複数の払出の価格を決定する場合があります。 このシナリオは、移動平均原価価格の見積を過剰に高くします。 ただし、この価格決定増幅を回避する方法、または問題が発生した場合にその影響を減らす方法があります。

**シナリオ**: 次のトランザクションは、**現物価格を含める** オプションを選択した品目に対して発生します。

1. USD 100.00 で 100 個を財務で入庫します。
2. 200 個を財務で払い出します。
3. USD 202.00 で 101 個を現物で入庫します。

品目に対して見積もられた移動平均原価価格を調べる際には、原価価格を USD 1.51 と予想します。 代わりに、USD 102.00 と見積もられた移動平均が次の式に基づいていることに気付きます。

見積価格 = \[202 + (-100)\] ÷ \[101 + (-100)\] = 102 ÷ 1 = 102

この価格決定増幅は、ステップ 2 で 200 品目が財務的に払い出された際、システムが、対応する入庫が発生する前に品目の価格を強制的に 100 に決める必要があります。 この条件によってマイナス在庫が発生します。 システムは単価を USD 1.00 と見積もり、これは予想した値です。 しかし、対応する 100 個の入荷があり、その単価はそれぞれ USD 2.00 です。

> [!NOTE]
> この払出により在庫は負になりますが、払出価格の計算時には在庫は正です。 このため、品目マスタの価格ではなく、移動平均原価価格が使用されます。 この時点で、システムには USD 100.00 の在庫金額相殺があります。 それぞれの単位相殺は USD 1.00 である 100 個に対する相殺が累積しましたが、現在在庫は 1 個になりました。 そのため、その相殺の USD 100.00 が、このただ 1 個に割り当てられます。 結果は、過剰に高い見積もり原価価格になります。
>
> 比較のため、このシナリオでステップ 2 と 3 が逆になったとすると、200 個の品目が単価 USD 1.51 で払出され、1 個が単価 USD 1.51 で残ることがわかります。 この価格決定増幅シナリオは、負の在庫が関係するときに発生する可能性があるので、次の場合には防ぐのが困難です。
>
> - 手持の金額と数量で払出価格を見積もる必要があります。
> - 手持の金額と数量を払出と入庫で調整する必要があります。
> - ビジネス モデルで、手持より多い個数の発送または価格決定が許可されています。
> - 送信されたすべての入庫金額と数量を受け入れる必要があります。

ただし、ビジネス モデルが以下の方法を許している場合は、価格決定増幅シナリオの可能性がある負の数量を避けるのに有効です。

- 品目に **現物価格を含める** オプションを選択している場合、**品目モデル グループ** ページの **現物マイナス在庫** チェック ボックスをクリアにします。
- 品目に **現物価格を含める** オプションを選択して **いない** 場合、**品目モデル グループ** ページの **財務マイナス在庫** オプションをオフにします。

また、現物在庫金額の最大相殺は、現物トランザクションの数および現物価格と財務価格の差によって制限されることに留意してください。 すべての現物トランザクションが最終的に財務的に更新されれば、現物金額が非常に高くなることはありません。 最後に、累積された相殺がただ 1 つではなく、いくつかの手持に対して分配されると、増幅の影響は大幅に減少することに注意してください。

## <a name="avoid-a-zero-cost-price-on-issues"></a>払出時の原価価格のゼロを回避する

**品目モデル グループ** ページで **物価格を含める** オプションが選択されていない場合、在庫の入庫が財務的に更新されていない場合、在庫からの入庫の原価価格をゼロにできます。 このシナリオを回避するようにするには、次のオプションを考慮してください。

- **品目モデル グループ** ページで、品目の **現物価格を含める** チェック ボックスをオンにします。 入庫が物理的に更新される場合、このオプションを使用すると、入庫の原価価格がゼロに設定されません。 マイナスの在庫を許可しない場合、物理的に更新されたトランザクションの移動平均が発行によって計算されます。
- 標準原価の原価バージョンを有効化して既定の品目原価価格を作成するか、**リリースされた製品の詳細** ページの **原価の管理** クイック タブで価格を入力します。 **品目モデル グループ** ページで **物価格を含める** オプションが選択されていない場合は、システムには常に使用する代替価格が設定されるので、このオプションが最適です。
- **リリース済製品の詳細** ページの **原価の管理** クイック タブで **最新の原価価格を使用** オプションを選択します。 このオプションを選択すると、入庫を財務的に更新するごとに **価格** フィールドが最新の状態に維持されます。 このオプションを選択したが、既定の価格を入力したり、標準原価バージョンで原価を有効化したりしない場合でも、問題に対する原価がゼロである場合があります。

原価がゼロの発行トランザクションがある場合は、*在庫決算および調整* プロセスで、入庫が財務更新された後に調整を作成することで原価価格が修正されます。 この更新が発生した財務期間は、品目が物理的に受け入れた、または発行された財務期間と異なる場合があります。
