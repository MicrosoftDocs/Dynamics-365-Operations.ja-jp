# <a name="procurement-catalogs"></a><span data-ttu-id="8bc96-104">調達カタログ</span><span class="sxs-lookup"><span data-stu-id="8bc96-104">Procurement catalogs</span></span>

[!INCLUDE [banner](../includes/banner.md)]

<span data-ttu-id="8bc96-105">この記事は、購買担当者が調達カタログを設定および管理する方法について高レベルで説明します。</span><span class="sxs-lookup"><span data-stu-id="8bc96-105">This article describes, at a high level, how purchasing professionals can set up and maintain procurement catalogs.</span></span> <span data-ttu-id="8bc96-106">調達カタログは、会社の従業員が内部で使用するために注文する品目およびサービスを定義します。</span><span class="sxs-lookup"><span data-stu-id="8bc96-106">Procurement catalogs define the items and services that company employees can order for internal use.</span></span>

<span data-ttu-id="8bc96-107">購買担当者は、組織内で使用する品目およびサービスを購入する際に使用できるカタログの作成および管理を行えます。</span><span class="sxs-lookup"><span data-stu-id="8bc96-107">Purchasing professionals can create and maintain catalogs of the items and services that can be purchased for internal use in an organization.</span></span> <span data-ttu-id="8bc96-108">カタログを設定した後、会社の従業員は、購買要求を作成してそこから注文できます。</span><span class="sxs-lookup"><span data-stu-id="8bc96-108">After catalogs are set up, company employees can create purchase requisitions to order from them.</span></span> <span data-ttu-id="8bc96-109">カタログは、購買ポリシーを実施して、従業員が購買側法人で許可されるサービスや品目のみ注文できるように設定できます。</span><span class="sxs-lookup"><span data-stu-id="8bc96-109">The catalogs can be used to enforce purchasing policies, so that employees can order only the items and services that are allowed for their buying legal entity.</span></span> <span data-ttu-id="8bc96-110">調達カタログを作成する場合は、次の作業を考慮することをお勧めします。</span><span class="sxs-lookup"><span data-stu-id="8bc96-110">When you create a procurement catalog, you should consider the following tasks:</span></span>

-   <span data-ttu-id="8bc96-111">カタログを作成する前に、調達カテゴリ階層を構成します。</span><span class="sxs-lookup"><span data-stu-id="8bc96-111">Configure your procurement category hierarchy before you create the catalog.</span></span>
-   <span data-ttu-id="8bc96-112">従業員が注文できる製品を決定します。</span><span class="sxs-lookup"><span data-stu-id="8bc96-112">Determine which products you want your employees to be able to order.</span></span> <span data-ttu-id="8bc96-113">カタログ ノード内の特定の製品を表示または非表示にすることも、ノード内のすべての製品を表示または非表示にすることもできます。</span><span class="sxs-lookup"><span data-stu-id="8bc96-113">You can show or hide specific products in a catalog node, or you can show or hide all the products in a node.</span></span>
-   <span data-ttu-id="8bc96-114">必要な調達カタログの数を決定します。</span><span class="sxs-lookup"><span data-stu-id="8bc96-114">Determine how many procurement catalogs you require.</span></span> <span data-ttu-id="8bc96-115">調達カタログへのアクセス許可は、従業員に割り当てられている法人および作業単位にコンフィギュレーションしたカタログ ポリシー ルールによって決まります。</span><span class="sxs-lookup"><span data-stu-id="8bc96-115">Access to a procurement catalog is determined by the catalog policy rule that you configure for the legal entity and operating unit that an employee is assigned to.</span></span>

<span data-ttu-id="8bc96-116">一部の要素は、購買要求の作成時に、従業員が注文できる製品、および従業員が使用できる調達カテゴリを決定します。</span><span class="sxs-lookup"><span data-stu-id="8bc96-116">Several factors determine the products that employees can order and the procurement categories that they can use when they create purchase requisitions:</span></span>

-   <span data-ttu-id="8bc96-117">購入ポリシーのカテゴリ アクセス ポリシー ルールによって、購買要求に使用できる調達カテゴリが決まります。</span><span class="sxs-lookup"><span data-stu-id="8bc96-117">The category access policy rule in the purchasing policy determines which procurement categories are available in the purchase requisition.</span></span> <span data-ttu-id="8bc96-118">ルールが定義されていない場合は、すべての調達カテゴリを利用できます。</span><span class="sxs-lookup"><span data-stu-id="8bc96-118">If no rule is defined, all procurement categories are available.</span></span>
-   <span data-ttu-id="8bc96-119">従業員は、組織の有効な調達カタログにあり、有効なノードで非表示になっていない場合にのみ、製品を注文できます。</span><span class="sxs-lookup"><span data-stu-id="8bc96-119">Employees can order a product only if it's in the active procurement catalog for your organization, and if it's in an enabled node and not hidden.</span></span> <span data-ttu-id="8bc96-120">また、製品は、カテゴリ アクセス ポリシー ルールに従って特定の従業員のアクセスを許可するカテゴリにある必要があります。</span><span class="sxs-lookup"><span data-stu-id="8bc96-120">The product must also be in a category that a particular employee has access to according to the category access policy rule.</span></span>
-   <span data-ttu-id="8bc96-121">カタログ ポリシー ルールは、使用するカタログを指定します。</span><span class="sxs-lookup"><span data-stu-id="8bc96-121">The catalog policy rule specifies the catalog that is used.</span></span> <span data-ttu-id="8bc96-122">カタログ ポリシー ルールが定義されていない場合は、カテゴリ アクセス ルール単独で、従業員の購買要求で注文できる製品を決定します。</span><span class="sxs-lookup"><span data-stu-id="8bc96-122">If no catalog policy rule is defined, the category access rule alone determines the products that an employee can order on the requisition.</span></span>
