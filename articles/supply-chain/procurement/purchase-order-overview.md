<span data-ttu-id="6be4a-112">発注書、製品の受領書、および仕入先請求書に関する情報を表示することが可能な多くのレポートがあります。</span><span class="sxs-lookup"><span data-stu-id="6be4a-112">There are many reports where you can view information about POs, product receipts, and vendor invoices.</span></span> <span data-ttu-id="6be4a-113">これらのレポートは、**調達** と **買掛金勘定** モジュールで確認できます。</span><span class="sxs-lookup"><span data-stu-id="6be4a-113">These reports are found in the **Procurement and sourcing** and **Accounts payable** modules.</span></span>

data-ttu-id="6be4a-115">そこでは実行しなければならないアクションの概要も表示されます。</span><span class="sxs-lookup"><span data-stu-id="6be4a-115">They also provide a summary of the actions that must be taken.</span></span> <span data-ttu-id="6be4a-116">**発注書の準備** ワークスペースは、発注書の作成およびレビュー、注文の承認処理、仕入先との確認に焦点を合わせています。</span><span class="sxs-lookup"><span data-stu-id="6be4a-116">The **Purchase order preparation** workspace is focused on PO creation and review, processing of the order through approval, and confirmation with the vendor.</span></span> <span data-ttu-id="6be4a-117">**発注書入庫とフォローアップ** ワークスペースは、PO に対する商品またはサービスの入庫処理に焦点を当てています。</span><span class="sxs-lookup"><span data-stu-id="6be4a-117">The **Purchase order receipt and follow-up** workspace is focused on processing the receipt of goods or services against POs.</span></span> <span data-ttu-id="6be4a-118">期限が過ぎている入庫、またはまもなく仕入先が納品する予定の入庫の分析に役立つリストが含まれています。</span><span class="sxs-lookup"><span data-stu-id="6be4a-118">It includes lists that give insight into receipts that are overdue, or that will soon be due for delivery by the supplier.</span></span> <span data-ttu-id="6be4a-119">これらのワークスペースは倉庫で行われる受入の活動に関連するタスクの実行には使用されません。</span><span class="sxs-lookup"><span data-stu-id="6be4a-119">These workspaces aren't used to perform the related receipt activities that are done in the warehouse.</span></span> <span data-ttu-id="6be4a-120">これらの活動は、**在庫管理** にあるページと **倉庫管理** モジュールを使用して実行されます。</span><span class="sxs-lookup"><span data-stu-id="6be4a-120">Those activities are performed by using pages in the **Inventory management** and **Warehouse management** modules.</span></span> <span data-ttu-id="6be4a-121">仕入先請求書の処理は **仕入先請求書入力** ワークスペースを使用し、また支払は **仕入先支払** ワークスペースを使用して処理する必要があります。</span><span class="sxs-lookup"><span data-stu-id="6be4a-121">Processing of vendor invoices should be done by using the **Vendor invoice entry** workspace, and payments should be done by using the **Vendor payments** workspace.</span></span>

## <a name="purchase-order-statuses"></a><span data-ttu-id="6be4a-145">発注書の状態</span><span class="sxs-lookup"><span data-stu-id="6be4a-145">Purchase order statuses</span></span>
<span data-ttu-id="6be4a-146">発注書には、注文の進行状況を示すいくつかの状態フィールドが含まれます。</span><span class="sxs-lookup"><span data-stu-id="6be4a-146">POs include several status fields that indicate the progress of the order.</span></span> <span data-ttu-id="6be4a-147">これらすべてのフィールドは、注文の **ヘッダー** ビューに表示され、それらのいくつかは、すべての注文のグリッドの概要にも表示されます。</span><span class="sxs-lookup"><span data-stu-id="6be4a-147">All these fields are visible in the **Header** view of the order, and a few of them are also visible in the grid overview of all orders.</span></span> <span data-ttu-id="6be4a-148">**ステータス** フィールドは、注文の数量のステータスを表示します。</span><span class="sxs-lookup"><span data-stu-id="6be4a-148">The **Status** field show the status for quantities on the order.</span></span> <span data-ttu-id="6be4a-149">使用可能な値は次のとおりです。</span><span class="sxs-lookup"><span data-stu-id="6be4a-149">The following values are available:</span></span>

-   <span data-ttu-id="6be4a-150">**オープン注文** – 注文が作成されて、数量が注文済です。</span><span class="sxs-lookup"><span data-stu-id="6be4a-150">**Open order** – Orders have been created, and quantities are on order.</span></span>
-   <span data-ttu-id="6be4a-151">**受取済** – 数量の一部を受取ましたが、まだ請求されていません。</span><span class="sxs-lookup"><span data-stu-id="6be4a-151">**Received** – Some of the quantities have been received, but they haven't been invoiced yet.</span></span>
-   <span data-ttu-id="6be4a-152">**請求済** – 注文の全数量が請求されました。</span><span class="sxs-lookup"><span data-stu-id="6be4a-152">**Invoiced** – The full quantity on the order has been invoiced.</span></span> <span data-ttu-id="6be4a-153">**注:** 注文が *部分的に* 請求されている場合、**受取済** ステータスも **請求済** ステータスも適切な状態ではありません。</span><span class="sxs-lookup"><span data-stu-id="6be4a-153">**Note:** If an order has been *partially* invoiced, neither **Received** status nor **Invoiced** status is appropriate.</span></span> <span data-ttu-id="6be4a-154">そのため、注文は **オープン注文** の状態のままになります。</span><span class="sxs-lookup"><span data-stu-id="6be4a-154">Therefore, the order will still have a status of **Open order**.</span></span>
-   <span data-ttu-id="6be4a-155">**キャンセル済** – 注文は確認済みでしたが、後でキャンセルされました。</span><span class="sxs-lookup"><span data-stu-id="6be4a-155">**Canceled** – An order was confirmed but later canceled.</span></span> <span data-ttu-id="6be4a-156">そのため、この状態は注文のオープン数量が存在しないことを示します。</span><span class="sxs-lookup"><span data-stu-id="6be4a-156">Therefore, this status indicates that there are no longer any open quantities on order.</span></span>

<span data-ttu-id="6be4a-157">**ドキュメントの状態** フィールドを使用して、処理されたドキュメントの注文の進行状況を確認できます。</span><span class="sxs-lookup"><span data-stu-id="6be4a-157">The **Document status** field helps you quickly review the order's progress in terms of documents that have been processed.</span></span> <span data-ttu-id="6be4a-158">注文の完了した最新のドキュメントのステータスが表示されます。</span><span class="sxs-lookup"><span data-stu-id="6be4a-158">It shows the status of the most recent document that has been completed for the order.</span></span> <span data-ttu-id="6be4a-159">使用可能な値は次のとおりです。</span><span class="sxs-lookup"><span data-stu-id="6be4a-159">The following values are available:</span></span>

-   <span data-ttu-id="6be4a-160">**なし** – 注文のドキュメントが処理されていません。</span><span class="sxs-lookup"><span data-stu-id="6be4a-160">**None** – No document has been processed for the order yet.</span></span>
-   <span data-ttu-id="6be4a-161">**購買照会** – 購買照会が生成されましたが、注文は仕入先からのフィードバックを待っています。</span><span class="sxs-lookup"><span data-stu-id="6be4a-161">**Purchase inquiry** – A purchase inquiry has been generated, and the order is awaiting feedback from the vendor.</span></span>

-   <span data-ttu-id="6be4a-164">**請求書** – 注文の請求書が転記されました。</span><span class="sxs-lookup"><span data-stu-id="6be4a-164">**Invoice** – An invoice has been accounted with the order.</span></span>

<span data-ttu-id="6be4a-165">**承認ステータス** フィールドは、発注書の確認プロセスやワークフローを実行するときに使用します。</span><span class="sxs-lookup"><span data-stu-id="6be4a-165">The **Approval status** field is used when a PO goes through a review process or workflow.</span></span> <span data-ttu-id="6be4a-166">使用可能な値は次のとおりです。</span><span class="sxs-lookup"><span data-stu-id="6be4a-166">The following values are available:</span></span>

-   <span data-ttu-id="6be4a-170">**外部レビュー中** – このステータスは、仕入先が発注書の条件を確認するために、購買の照会が仕入先に送信される、といったシナリオに使用されます。</span><span class="sxs-lookup"><span data-stu-id="6be4a-170">**In external review** – This status is used in scenarios where a purchase inquiry is sent to the vendor, so that the vendor can confirm terms of the PO.</span></span> <span data-ttu-id="6be4a-171">このステータスは、**確認要求** アクションにより開始されるプロセスにも使用されます。</span><span class="sxs-lookup"><span data-stu-id="6be4a-171">This status is also used in the process that is initiated by the **Confirmation request** action.</span></span> <span data-ttu-id="6be4a-172">このプロセスでは、システムに接続し、注文を確認または拒否するかどうかを登録することにより、仕入先が発注書の条件を確認することが求められます。</span><span class="sxs-lookup"><span data-stu-id="6be4a-172">For this process, the vendor is asked to confirm terms of the PO by connecting to your system and registering whether it confirms or rejects the order.</span></span>
