# <a name="configure-wave-processing"></a><span data-ttu-id="fb086-103">ウェーブ処理のコンフィギュレーション</span><span class="sxs-lookup"><span data-stu-id="fb086-103">Configure wave processing</span></span>

[!include [banner](../../includes/banner.md)]

<span data-ttu-id="fb086-104">このガイドでは、ウェーブが処理されるときに倉庫に対して生成される作業、およびウェーブが手動または自動で処理されるか判断する基準の設定方法について説明します。</span><span class="sxs-lookup"><span data-stu-id="fb086-104">This guide describes how to set up the criteria that determine what work is generated for a warehouse when a wave is processed, and whether waves are processed manually or automatically.</span></span> <span data-ttu-id="fb086-105">販売注文、製造オーダー、またはかんばんオーダーのリリース済み明細行のあるウェーブと一致する、ウェーブ テンプレートおよびクエリを設定し、基準を指定します。</span><span class="sxs-lookup"><span data-stu-id="fb086-105">You specify the criteria by setting up wave templates and queries that match a wave with released lines in sales orders, production orders, or kanban orders.</span></span> <span data-ttu-id="fb086-106">ウェーブ処理は倉庫管理モジュールで機能を使用する倉庫で使用され、在庫管理モジュールで機能を使用する倉庫では使用されません。</span><span class="sxs-lookup"><span data-stu-id="fb086-106">Wave processing is used in warehouses that use the functionality in the Warehouse management module, and not those that use the functionality in the Inventory management module.</span></span> <span data-ttu-id="fb086-107">デモ データの会社 USMF でこの手順を確認できます。</span><span class="sxs-lookup"><span data-stu-id="fb086-107">You can run this procedure in demo data company USMF.</span></span>

2. <span data-ttu-id="fb086-109">**新規** をクリックします。</span><span class="sxs-lookup"><span data-stu-id="fb086-109">Click **New**.</span></span>
3. <span data-ttu-id="fb086-110">**ウェーブ テンプレート名** フィールドで、値を入力します。</span><span class="sxs-lookup"><span data-stu-id="fb086-110">In the **Wave template name** field, type a value.</span></span> <span data-ttu-id="fb086-111">ウェーブ テンプレートを設定する際は、販売注文、製造オーダー、かんばんのリリース済明細行と照合するテンプレートの順番を指定します。</span><span class="sxs-lookup"><span data-stu-id="fb086-111">When you set up a wave template, you specify the sequence in which the templates will be matched to released lines on sales orders, production orders, or kanbans.</span></span> <span data-ttu-id="fb086-112">明細行が倉庫または製造にリリースされる際、基準に合致する最初のウェーブ テンプレートが使用されます。</span><span class="sxs-lookup"><span data-stu-id="fb086-112">When a line is released to the warehouse or to production, it uses the first wave template that it meets the criteria for.</span></span> <span data-ttu-id="fb086-113">最も限定的な基準のテンプレートをリストの最初に置くことをお勧めします。</span><span class="sxs-lookup"><span data-stu-id="fb086-113">We recommend that you put templates with the most specific criteria at the top of the list.</span></span> <span data-ttu-id="fb086-114">基準の範囲が広いほど、明細行が基準と合致する確率が高くなり、この結果、間違ったウェーブに明細行が割り当てられることになります。</span><span class="sxs-lookup"><span data-stu-id="fb086-114">The broader the criteria, the more likely it is for a line to meet the criteria, and this could lead to lines being assigned to the wrong wave.</span></span>

6. <span data-ttu-id="fb086-118">**倉庫** フィールドで、値を入力または選択します。</span><span class="sxs-lookup"><span data-stu-id="fb086-118">In the **Warehouse** field, enter or select a value.</span></span> <span data-ttu-id="fb086-119">USMF を使用している場合は、倉庫 24 を選択できます。</span><span class="sxs-lookup"><span data-stu-id="fb086-119">If you're using USMF, you can select warehouse 24.</span></span>  
7. <span data-ttu-id="fb086-120">**ウェーブの作成を自動化** フィールドを **はい** に設定します。</span><span class="sxs-lookup"><span data-stu-id="fb086-120">Set the **Automate wave creation** field to **Yes**.</span></span> <span data-ttu-id="fb086-121">販売注文、製造オーダー、またはかんばんが倉庫にリリースされるときに、自動的にウェーブを作成するには、このオプションを選択します。</span><span class="sxs-lookup"><span data-stu-id="fb086-121">Select this option to automatically create a wave when a sales order, production order, or kanban is released to the warehouse.</span></span>

9. <span data-ttu-id="fb086-124">**ウェーブを自動的にリリースするオプション** を **はい** に設定します。</span><span class="sxs-lookup"><span data-stu-id="fb086-124">Set the **Automate wave release option** to **Yes**.</span></span> <span data-ttu-id="fb086-125">自動的にウェーブをリリースするには、このオプションを選択します。</span><span class="sxs-lookup"><span data-stu-id="fb086-125">Select this option to automatically release the wave.</span></span> <span data-ttu-id="fb086-126">ピッキング作業が作成され、モバイル デバイス上で利用できます。</span><span class="sxs-lookup"><span data-stu-id="fb086-126">The picking work is created and made available on mobile devices.</span></span>  
10. <span data-ttu-id="fb086-127">**未処理のウェーブへの割り当てオプション** を **はい** に設定します。</span><span class="sxs-lookup"><span data-stu-id="fb086-127">Set the **Assign to open waves option** to **Yes**.</span></span> <span data-ttu-id="fb086-128">明細行はウェーブ テンプレートのクエリ フィルターに基づいてウェーブに割り当てられます。</span><span class="sxs-lookup"><span data-stu-id="fb086-128">Lines are assigned to waves based on the query filter for the wave template.</span></span>  
11. <span data-ttu-id="fb086-129">**しきい値に達したときにウェーブを自動処理するオプション** を **はい** に設定します。</span><span class="sxs-lookup"><span data-stu-id="fb086-129">Set the **Process wave automatically at threshold option** to **Yes**.</span></span> <span data-ttu-id="fb086-130">値が [ウェーブのしきい値] フィールド グループで指定されている重量、出荷、明細行のしきい値に達したときに自動的にウェーブを処理するには、このオプションを選択します。</span><span class="sxs-lookup"><span data-stu-id="fb086-130">Select this option to automatically process the wave when its values reach the thresholds for weight, shipment, and lines specified in the Wave thresholds field group.</span></span> <span data-ttu-id="fb086-131">このオプションは、[ウェーブ テンプレート タイプ] フィールドで [出荷] が選択されている場合にのみ使用できます。</span><span class="sxs-lookup"><span data-stu-id="fb086-131">This option is available only if Shipping is selected in the Wave template type field.</span></span>

13. <span data-ttu-id="fb086-135">**メソッド** セクションを展開します。</span><span class="sxs-lookup"><span data-stu-id="fb086-135">Expand the **Methods** section.</span></span>

    - <span data-ttu-id="fb086-136">ウェーブ テンプレート メソッドによって各ウェーブの処理時に行われる一連の活動を制御できます。</span><span class="sxs-lookup"><span data-stu-id="fb086-136">Wave template methods allow you to control the sequence of activities that each wave is going through when it's processed.</span></span> <span data-ttu-id="fb086-137">たとえば、ウェーブの補充のメソッドがある場合があります。</span><span class="sxs-lookup"><span data-stu-id="fb086-137">For example, you might have a method for wave replenishment.</span></span> <span data-ttu-id="fb086-138">メソッドを追加すると、一連の手順の適切な場所に自動的に一覧表示されます。</span><span class="sxs-lookup"><span data-stu-id="fb086-138">When you add a method, it's automatically listed in the appropriate location in the sequence of steps.</span></span> <span data-ttu-id="fb086-139">補充作業のリリースの自動化 オプションを はい に設定した場合、ここに補充メソッドを追加する必要があります。</span><span class="sxs-lookup"><span data-stu-id="fb086-139">If you've set the Automate replenishment work release option to Yes, you need to add the replenish method here.</span></span>
