# <a name="set-up-manual-packing-february-2016--may-2016"></a>手動梱包の設定 (2016 年 2 月 & 2016 年 5 月)

[!include [task guide banner](../../includes/task-guide-banner.md)]

梱包プロセスで、製品を検証してコンテナーに梱包できます。 このプロセスでは、倉庫作業者が製品を保管場所から集荷し、品目の数量および種類をチェックする梱包ステーションに移動して、適切なコンテナーに割り当てます。 コンテナーが完全に梱包され、閉じてから出荷ドックに移動されると、製品を出荷する準備が整います。 この手順では、USMF というデモ会社を使用します。 この手順は、Dynamics 365 for Operations の 2016 年 ２ 月 & 2016 年 5 月バージョンのみです。


## <a name="set-up-location-profiles"></a>場所プロファイルを設定します
1. [倉庫管理] > [設定] > [倉庫] > [場所プロファイル] の順に移動します。
2. [新規] をクリックします。
    * 場所プロファイルは梱包ステーションに使用され、場所の情報およびルールが含まれます。  
3. [場所プロファイル ID] フィールドに値を入力します。
4. [名前] フィールドに値を入力します。
5. [場所形式] フィールドで、値を入力または選択します。
6. [場所タイプ] フィールドで、値を入力または選択します。
7. [混合保管を許可] フィールドで [はい] を選択します。
8. [混合在庫状態を許可] フィールドで [はい] を選択します。
9. [バッチ日数ルールの上書き] フィールドで [はい] を選択します。
10. ページを閉じます。

## <a name="set-up-warehouse-management-parameters"></a>倉庫管理パラメーターの設定 
1. [倉庫管理] > [設定] > [倉庫管理パラメーター] の順に移動します。
2. [梱包] タブをクリックします
3. [梱包場所のプロファイル ID] フィールドで値を入力するか選択します。
    * 梱包に使用する場所プロファイルを選択します。  
4. ページを閉じます。

## <a name="set-up-container-types"></a>コンテナー タイプの設定
1. [倉庫管理] > [設定] > [コンテナー] > [コンテナー タイプ] の順に移動します。
2. [新規] をクリックします。
    * 使用するコンテナー タイプを定義することができます。 風袋重量、最大重量、最大容積、長さ、幅と重量を含むコンテナーの現物分析コードを指定できます。  属性はコンテナー タイプの予備の分析コードを追加できるようにカスタマイズされたフィールドです。     
3. [コンテナー タイプ コード] フィールドに値を入力します。
4. [説明] フィールドに値を入力します。
5. [風袋重量] フィールドに数値を入力します。
6. [最大重量] フィールドに数値を入力します。
7. [容積] フィールドに数値を入力します。
8. [長さ] フィールドに数値を入力します。
9. [幅] フィールドに数値を入力します。
10. [高さ] フィールドに数値を入力します。
11. [保存] をクリックします。
12. ページを閉じます。

## <a name="set-up-packing-profiles"></a>梱包プロファイルの設定
1. [倉庫管理] > [設定] > [梱包] > [梱包プロファイル] の順に移動します。
2. [新規] をクリックします。
3. [梱包プロファイル ID] フィールドに値を入力します。
4. [説明] フィールドに値を入力します。
5. [コンテナーを閉じるプロファイル ID] フィールドで値を入力するか選択します。
    * コンテナーを閉じるプロファイル ID はオプションで、この梱包プロファイルの既定の終了コンテナー プロファイルです。  
6. [コンテナー ID モード] フィールドで、オプションを選択します。
    * このオプションにより、コンテナーの作成時またはコンテナー ID を手動で作成した場合に、コンテナー ID が自動的に生成されるかどうかが決まります。  
7. [コンテナー タイプ] フィールドで、値を入力または選択します。
    * コンテナー タイプは、新しいコンテナーの作成時に既定で使用されます。  
8. [コンテナーを閉じる時にコンテナーを自動作成] チェック ボックスをオンにします。
9. [保存] をクリックします。
10. ページを閉じます。

## <a name="set-up-container-closing-profiles"></a>コンテナー終了プロファイルの設定
1. [倉庫管理] > [設定] > [コンテナー] > [コンテナー終了プロファイル] の順に移動します。
    * コンテナーを閉じるプロファイルは、コンテナーを締める際の処理を定義します。 複数の終了コンテナー プロファイルを設定できます。       
2. [新規] をクリックします。
3. [コンテナーを閉じるプロファイル ID] フィールドで値を入力します。
4. [説明] フィールドに値を入力します。
5. [マニフェスト予定日時] フィールドで、オプションを選択します。
    * コンテナー終了時や出荷の確認時にマニフェストを行うかどうかを指定します。 マニフェストには輸送管理が必要であることに注意してください。 マニフェストを動作させるためには輸送エンジンに実装する必要があります。  
6. [倉庫] フィールドで、値を入力または選択します。
7. [最終出荷の既定の場所] フィールドで値を入力するか選択します。
    * これは、コンテナーを閉じた後に移動する場所です。 この場所には、倉庫パラメーターで定義された場所プロファイルが必要です。  
8. [重量単位] フィールドで、値を入力または選択します。
9. [保存] をクリックします。
