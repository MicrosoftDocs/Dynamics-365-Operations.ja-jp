# <a name="page-module"></a><span data-ttu-id="36df2-103">ページ モジュール</span><span class="sxs-lookup"><span data-stu-id="36df2-103">Page module</span></span>

[!include [banner](../../../../includes/banner.md)]

<span data-ttu-id="36df2-104">IPage インターフェイスは、さまざまなプロパティ、ライフ サイクル、およびワークスペース内のページに関連付けられているイベント フックをカプセル化します。</span><span class="sxs-lookup"><span data-stu-id="36df2-104">The IPage interface encapsulates the various properties, life cycle and event hooks associated with a page in a workspace.</span></span>
### <a name="page-data-synchronization"></a><span data-ttu-id="36df2-105">ページ データ同期</span><span class="sxs-lookup"><span data-stu-id="36df2-105">Page data Synchronization</span></span>
<span data-ttu-id="36df2-106">変更を送信できるページ (アクションとも呼ばれる) は、サーバーと完全に同期される前にさまざまなステージを通過します。</span><span class="sxs-lookup"><span data-stu-id="36df2-106">Pages that can submit changes (also referred to as actions) go through various stages before they're completely in sync with the server.</span></span>
<span data-ttu-id="36df2-107">ページが [送信](../interfaces/view-model-ipage-ipage.md#submit) されるとすぐに、起こり得る結果が 3 つあります。</span><span class="sxs-lookup"><span data-stu-id="36df2-107">As soon as a page is [submitted](../interfaces/view-model-ipage-ipage.md#submit), there are three possible consequences:</span></span>
* <span data-ttu-id="36df2-108">クライアント側の検証に失敗する場合: クライアント ロジックによってページを送信できない可能性があります。</span><span class="sxs-lookup"><span data-stu-id="36df2-108">It fails client-side validation: The client logic might prevent the page from being submitted.</span></span>
* <span data-ttu-id="36df2-109">クライアントはオンラインです。アプリケーション全体の同期キューがクリアされるとすぐに、送信が処理されます。</span><span class="sxs-lookup"><span data-stu-id="36df2-109">The client is online: The submission is processed as soon as the application-wide sync queue is cleared.</span></span>
* <span data-ttu-id="36df2-110">クライアントはオフラインです。送信はアプリケーション全体の同期キューに追加され、クライアントがオフラインである限り、そこにとどまります。</span><span class="sxs-lookup"><span data-stu-id="36df2-110">The client is offline: The submission is added to the application-wide sync queue and stays there as long as the client is offline.</span></span>

<span data-ttu-id="36df2-111">送信の同期の待機中は、以下のいずれかの状態にあります。</span><span class="sxs-lookup"><span data-stu-id="36df2-111">While a submission is waiting to be synchronized, it can be in one of these states:</span></span>
* <span data-ttu-id="36df2-112">保留中: 送信が保留中ですが、編集することはできます。</span><span class="sxs-lookup"><span data-stu-id="36df2-112">Pending: The submission is still pending and can still be edited.</span></span>
* <span data-ttu-id="36df2-113">処理: 送信は現在同期中です。</span><span class="sxs-lookup"><span data-stu-id="36df2-113">Processing: The submission is currently being synchronized.</span></span> <span data-ttu-id="36df2-114">この状態では、ページをこれ以上編集できません。</span><span class="sxs-lookup"><span data-stu-id="36df2-114">In this state, any further edits are not allowed on the page.</span></span>

<span data-ttu-id="36df2-115">送信がサーバーに送られた後、次のいずれかの状態になります。</span><span class="sxs-lookup"><span data-stu-id="36df2-115">After a submission goes through to the server, it can be in one of these states:</span></span>
* <span data-ttu-id="36df2-116">同期済み: 送信はサーバーに受け入れられて同期されます。</span><span class="sxs-lookup"><span data-stu-id="36df2-116">Synchronized: The submission is accepted by the server and is synchronized.</span></span>
* <span data-ttu-id="36df2-117">エラー: サーバーは、送信を拒否およびページがエラー状態を入力します。</span><span class="sxs-lookup"><span data-stu-id="36df2-117">Error: The server rejects the submission and the page enters an error state.</span></span>

<span data-ttu-id="36df2-118">特定のページおよびコンテキストにおいて保留中の送信が複数あり、ページが冪等であるとみなされる場合、集められる (かつ送信される) 場合があります。</span><span class="sxs-lookup"><span data-stu-id="36df2-118">Multiple pending submissions for a given page and context might be clubbed (and sent) together if the page is deemed to be idempotent.</span></span>
<span data-ttu-id="36df2-119">これは、サーバーがすべての送信をさまざまな順序で処理する必要はなく、サーバー上のページの設計に依存することを意味します。</span><span class="sxs-lookup"><span data-stu-id="36df2-119">This implies that the server need not process all submissions in any order and depends on the design of the page on the server.</span></span>
