# <a name="identify-and-resolve-conflicts-in-segregation-of-duties"></a>職務分掌の競合の識別と解決

[!include [task guide banner](../../includes/task-guide-banner.md)]

異なるユーザーが実行する必要があるタスクを分割するルールを設定できます。 この概念は職務分掌と呼ばれます。 セキュリティ ロールの定義またはユーザーのロール割り当てがルールに違反する場合、競合がログに記録されます。 すべての競合は、管理者が解決する必要があります。 競合を識別および解決するには、次の手順を実行します。 この手順の作成に使用するデモ データの会社は USMF です。


## <a name="verify-whether-user-role-assignments-comply-with-new-rules-for-segregation-of-duties"></a>ユーザー ロールの割り当てが職務分掌の新しいルールに準拠しているかどうかの確認
1. [システム管理] > [セキュリティ] > [職務分掌] > [ユーザー ロール割り当ての遵守を確認] の順に移動します。
2. [OK] をクリックします。
    * 通知には検証結果が表示されます。     競合がある場合は、[ユーザー] ページを開き、ユーザーのロールの割り当てを変更できます。 また、競合は、[職務分掌の競合] ページでログに記録されます。     検証プロセスをバッチ ジョブとして実行するには、バッチ処理を選択し、その他のバッチ パラメーターを設定します。 バッチ ジョブの実行後、[職務分掌の競合] ページで競合を確認できます。  

## <a name="view-and-resolve-conflicting-user-role-assignments"></a>競合しているユーザー ロールの割り当ての表示と解決
1. [システム管理] > [セキュリティ] > [職務分掌] > [職務分掌の競合] の順に移動します。
    * 競合を選択し、次のボタンの 1 つをクリックします。[割り当ての拒否] – 追加したセキュリティ ロールへのユーザーの割り当てを拒否します。 自動ロール割り当てを拒否すると、ユーザーはロールから除外としてマークされます。 除外されたユーザーは、ロールと関連付けられているアクセス権を付与されません。また、管理者が除外を削除するまでそのユーザーをロールに割り当てることはできません。     [割り当ての許可] – 競合を上書きし、ユーザーが両方のセキュリティ ロールに割り当てられることを許可します。 競合を上書きする場合は、[上書きの理由] フィールドに理由を入力する必要があります。  
2. ページを閉じます。
3. [システム管理] > [セキュリティ] > [職務分掌] > [職務分掌による未解決の競合] の順に移動します。
    * 競合を選択し、次のボタンの 1 つをクリックします。[割り当ての拒否] – 追加したセキュリティ ロールへのユーザーの割り当てを拒否します。 自動ロール割り当てを拒否すると、ユーザーはロールから除外としてマークされます。 除外されたユーザーは、ロールと関連付けられているアクセス権を付与されません。また、管理者が除外を削除するまでそのユーザーをロールに割り当てることはできません。     [割り当ての許可] – 競合を上書きし、ユーザーが両方のセキュリティ ロールに割り当てられることを許可します。 競合を上書きする場合は、[上書きの理由] フィールドに理由を入力する必要があります。    
4. ページを閉じます。

## <a name="verify-whether-existing-roles-comply-with-new-rules-for-segregation-of-duties"></a>既存のロールが職務分掌の新しいルールに準拠しているかどうかの確認
1. [システム管理] > [セキュリティ] > [職務分掌] > [職務分掌ルール] の順に移動します。
    * ルールを選択します。  
2. [職務とロールの検証] をクリックします。
    * 既存のロールが選択したルールに違反している場合、ロールの名前と競合する職務を示すメッセージが表示されます。 管理者は、職務分掌のルールに違反しないように、セキュリティ リスクの軽減策を示すか、ロールの変更をする必要があります。     選択したルールに違反するロールがない場合、すべてのロールがルールに遵守していることを示すメッセージが表示されます。
