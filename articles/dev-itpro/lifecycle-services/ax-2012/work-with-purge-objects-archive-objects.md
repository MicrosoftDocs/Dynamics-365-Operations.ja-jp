# <a name="work-with-purge-and-archive-objects"></a>オブジェクトの削除とアーカイブを使用する

[!include [banner](../../includes/banner.md)]

<a name="addedit-rules"></a>ルールの追加/編集
--------------

ルールは、削除タスクまたはアーカイブ タスクの実行時に、レコードをフィルター処理するために使用される基準です。 このコマンドを使用すると、削除オブジェクトまたはアーカイブ オブジェクト内のルールを操作できます。 ルールは一般的にドライバー テーブルに適用されますが、関連する子テーブルにも適用できます。 たとえば、削除オブジェクト **ProdJournalTable** には、**ProdTable** のルールが含まれ、ドライバー テーブルではありません。 これらのルールを作成する方法に関係なく、"where" 句はドライバー テーブルに常に適用されます。 どのようなルールを適用する場合も、適用前にルールの削除オブジェクトまたはアーカイブ オブジェクトへの影響について理解する必要があります。 ツールバーで、**ルールの追加/編集**をクリックして、**ルールの追加/編集**ウィンドウを開きます。

### <a name="navigation-of-the-addedit-rules-window"></a>ルールの追加/編集ウィンドウのナビゲーション

次のテーブルで、**ルールの追加/編集** ウィンドウのコントロールについて説明します。
#### <a name="panes"></a>ウィンドウ

| ウィンドウ                          | 説明                                                                                                                                                                           |
|-------------------------------|---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| **ルール コレクション**           | 削除オブジェクトまたはアーカイブ オブジェクト内のルールのリスト、および新しいルールの作成、既存のルールへの式の追加、またはルールまたは式の削除のコマンドが含まれています。     |
| **削除のルールのコンフィギュレーション** | ルールの名前、説明、および条件を入力または変更する領域を提供します。 また、**ルール コレクション**ウィンドウの一覧に、ルールを追加、または更新するコマンドが提供されます。 |
| **条件付き情報**   | 削除オブジェクトの条件または選択基準を指定できます。                                                                                                         |

| ボタン             | 説明                                                                                                                                                                  |
|--------------------|------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|

| **式を追加** | 新しい式を作成します。 式は既存のルールに追加される条件です。 式は SQL ステートメントの「where」句に「or」の条件を作成します。 |
| **削除**         | 選択したルールまたは式を削除します。                                                                                                                                      |

#### 

#### <a name="configure-rules-pane"></a>ルール ウィンドウのコンフィギュレーション

|------------|-------------------------------------------------------------------------------------|
| **追加**    | **ルール コレクション** ウィンドウの一覧に新しいルールまたは式を追加します。         |
| **更新** | **ルール コレクション** ウィンドウの一覧で選択したルールまたは式を更新します。 |
| **キャンセル** | 追加、または更新アクションをキャンセルします。                                                    |

#### 

#### <a name="fields-across-all-panes"></a>フィールド (すべてのウィンドウ間)

| フィールド                | 説明                                                                                                                                                                                        |
|----------------------|----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| **ルール名**        | ルールまたは式の名前。                                                                                                                                                                |
| **ルール**             | テーブル名、フィールド名、および条件付き演算子に基づいて作成する条件。 この条件は、SQL ステートメントの「where」句を構成します。                                   |
| **ルールの説明** | ルールの説明。 新しい説明を入力またはここに既存の説明を変更します。                                                                                                         |
| **テーブル名**       | 削除オブジェクトまたはアーカイブ オブジェクトのすべてのテーブルのリストから選択したテーブル。                                                                                                       |
| **フィールド名**       | 選択したテーブルのすべてのフィールドのリストから選択するフィールドの名前。                                                                                                          |
| **条件**        | ユーザーがテーブルとフィールドに配置する条件。 フィールドの選択に応じて、条件リストが変更されます。                                                                            |
| **値**            | **値** フィールドは、テキスト ボックスとリストを切り替えます。 リストから値を選択することができますが、ここではテキスト ボックスに値を入力することができません。 削除タスクを作成する場合には、値を入力します。 |

### 

### <a name="walkthrough-add-or-modify-a-rule-in-a-purge-object"></a>チュートリアル: 削除オブジェクトへのルールの追加または削除

このセクションでは、削除オブジェクトのルールを追加または変更するためのチュートリアルを提供します。

| **注意**                                                                                                                                                                                                                                                                                                                                                                                           |
|-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| このチュートリアルでは、学習を容易にするためにパージ オブジェクトから既存のルールを削除し、再作成します。 ルールを扱うには、データベース設計、データ フロー、プロセス フロー、および Microsoft Dynamics AX アプリケーションの機能について明確に理解している必要があります。 エラーは完全なデータベースおよびアプリケーションの復旧を必要とするデータの破損やアプリケーションのダウンタイムを引き起こす可能性があります。 |

1.  **構成** &gt; **削除テンプレート/削除オブジェクト** &gt; **PurchParmTable** とクリックし、**PurchParmTable** 削除オブジェクトを開きます。 このチュートリアルでは、既定のパージ テンプレートから作成されたパージ オブジェクトを使用して作業することを前提としています。 任意の方法で削除オブジェクトを変更した場合は、**復元**をクリックして削除オブジェクトを元のバージョンに復元します。
2.  ツールバーで、**ルールの追加/編集**をクリックして、**ルールの追加/編集**ウィンドウを開きます。
3.  **ルール コレクション** ウィンドウで、データ グリッドの最初の行をクリックして **PurchParmTable.ParmJobStatus = Executed** のルールを選択します。 **削除** をクリックします。 **削除**ダイアログ ボックスで、**OK** をクリックします。
4.  **新しいルール**をクリックします。 **削除のルールを構成**で、次のフィールド値を設定します。
    1.  **ルール名**フィールドに、「クリーンアップ」と入力します。
    2.  **ルールの説明**フィールドに、「どれをクリーンアップする必要がありますか」と入力します。
    3.  **テーブル名**リストで、**PurchParmTable** を選択します。
    4.  **フィールド名**リストで、**ParmJobStatus** を選択します。
    5.  **条件**リストで、**=** を選択します。
    6.  **値**フィールドで、リストから**実行済**を選択します。

5.  **追加**をクリックし、ルールが**ルール コレクション** ウィンドウのデータグリッドに追加されていることを確認します。
6.  **保存** をクリックします。 **ルール** ダイアログ ボックスで **OK** をクリックして続行します。
7.  **閉じる**をクリックしてウィンドウを閉じ、削除オブジェクトに戻ります。

### <a name="walkthrough-add-or-modify-a-rule-in-an-archive-object"></a>チュートリアル: アーカイブ オブジェクトへのルールの追加または削除

このセクションでは、アーカイブ オブジェクトのルールを追加または変更するためのチュートリアルを提供します。
1.  **構成** &gt; **アーカイブ テンプレート/アーカイブ オブジェクト** &gt; **SalesTable** とクリックして **SalesTable** アーカイブ オブジェクトを開きます。 このチュートリアルでは、既定のアーカイブ テンプレートから作成されたアーカイブ オブジェクトを使用して作業することを前提としています。 アーカイブ オブジェクトを変更した場合は、ツールバーの**バージョンを表示**コマンドを使用して、データ管理フレームワークに含まれている元のアーカイブ テンプレートに戻します。
2.  ツールバーで、**ルールの追加/編集**をクリックして、**ルールの追加/編集**ウィンドウを開きます。
3.  **ルール コレクション** ウィンドウで、データ グリッドの 2 行目をクリックして **SalesTable.SalesStatus In Canceled, Invoiced** のルールを選択します。 **削除** をクリックします。 **削除**ダイアログ ボックスで、**OK** をクリックします。
4.  **ルール コレクション**ウィンドウで、**拡張機能の追加**をクリックし、次のフィールド値を設定します。
    1.  **ルール名**フィールドに、「ステータス」と入力します。
    2.  **ルールの説明**フィールドに、アーカイブ用とみなされるステータスの値を入力します。

5.  **条件付き情報**ウィンドウで、次のフィールド値を設定します。
    1.  **テーブル名**リストで、**SalesTable** を選択します。
    2.  **フィールド名**リストで、**SalesStatus** を選択します。
    3.  **条件**リストで、**In** を選択します。
    4.  **値**フィールドで、リストから**キャンセル済および請求済**を選択します。

6.  **追加**をクリックし、ルールが**ルール コレクション** ウィンドウのデータグリッドに追加されていることを確認します。
7.  **保存** をクリックします。 **ルール** ダイアログ ボックスで **OK** をクリックして続行します。
8.  **閉じる**をクリックしてウィンドウを閉じ、アーカイブ オブジェクトに戻ります。

## <a name="add-relations"></a>リレーションの追加
このコマンドを使用すると、削除オブジェクトまたはアーカイブ オブジェクトに手動でテーブルを追加し、関係を確立できます。 Microsoft Dynamics AX アプリケーションにアプリケーション オブジェクト ツリー (AOT) のメタデータ リレーションシップなしのカスタム テーブルがある場合、リレーションシップの手動の追加が必要な場合があります。 リレーションを追加するには、リレーションシップ ツリー ダイアグラムでテーブルをクリックして選択します。 選択したテーブルが親テーブルになり、追加するテーブルがリレーションシップの子になります。 ツールバーで、**関係を追加**をクリックして、**関係を追加**ウィンドウを開きます。
### <a name="navigation-of-the-add-relations-window"></a>関係の追加ウィンドウのナビゲーション

| **関係のコンフィギュレーション** | 関係の名前、説明、および条件を入力または変更する領域を提供します。 また、**リレーションシップ**ウィンドウの一覧にリレーションの追加、一覧内での更新や、追加または更新アクションをキャンセルしたりするためのコマンドも用意されています。 |

### <a name="walkthrough-add-a-relation-in-a-purge-object"></a>チュートリアル: 削除オブジェクトへのリレーションの追加

このセクションでは、削除オブジェクトにリレーションを追加するためのチュートリアルを提供します。
1.  **構成** &gt; **削除テンプレート/削除オブジェクト**とクリックして、ドロップダウン リストから **ProjJournalTable** を選択します。
2.  削除オブジェクトで、レベル 0 の **ProjJournalTable** テーブルをクリックしてから、ツール バーで**関係の追加**をクリックします。
3.  **関係の追加**ダイアログ ボックスで、**新しい関係**をクリックします。
4.  **関係名**フィールドに、関係に対する有効な名前を入力します。
5.  **テーブル リレーション**領域で、これらの手順に従って関係を追加します。
    1.  **フィールド名**の一覧から、**JournalId** を選択します。
    2.  **関連テーブル**の名前リストから **JournalError** を選択します。
    3.  **関連フィールド名**リストから **JournalId** を選択します。

6.  **追加** をクリックします。
7.  **関係の追加**ダイアログ ボックスで、**新しい関係**をクリックして、今追加した関係に対する条件を追加します。
8.  **条件付き情報**領域で、これらの手順に従って条件を追加します。
    1.  **テーブル名**リストから条件で使用するテーブルを選択します。
    2.  **フィールド名**リストから条件で使用するフィールドを選択します。
    3.  **条件**リストから、条件を選択します。
    4.  **値**リストから値を選択します。

9.  条件をデータ グリッドに追加するには、**追加**をクリックします。
10. **保存** をクリックします。 **OK** をクリックして続行します。
11. 削除オブジェクトで、追加した JournalError テーブルがレベル 1 の子エンティティで表示されていることを確認します。
12. 削除オブジェクトを保存します。

### <a name="walkthrough-add-a-relation-in-an-archive-object"></a>チュートリアル: アーカイブ オブジェクトへのリレーションの追加

1.  **構成** &gt; **アーカイブ テンプレート/アーカイブ オブジェクト**とクリックして、ドロップダウン リストから **SalesTable** を選択します。
2.  アーカイブ オブジェクトで、レベル 1 の **DocuRef** テーブルをクリックしてから、ツール バーで**関係の追加**をクリックします。
3.  **関係の追加**ダイアログ ボックスで、**新しい関係**をクリックします。
4.  **関係名**フィールドに、関係に対する有効な名前を入力します。
5.  **テーブル リレーション**領域で、これらの手順に従って関係を追加します。
    1.  **フィールド名**リストから **ValueRecId** を選択します。
    2.  **関連テーブル名**リストから **DocuValue** を選択します。
    3.  **関連フィールド名**リストから **RecId** を選択します。
    4.  **追加** をクリックします。

6.  **関係の追加**ダイアログ ボックスで、**新しい関係**をクリックして、今追加した関係に対する条件を追加します。
7.  条件付き**情報**領域で、これらの手順に従って条件を追加します。
    1.  **テーブル名**リストから条件で使用するテーブルを選択します。
    2.  **フィールド名**リストから条件で使用するフィールドを選択します。
    3.  **条件**リストから、条件を選択します。
    4.  **値**リストから値を選択します。
    5.  条件をデータ グリッドに追加するには、**追加**をクリックします。

8.  **保存** をクリックします。 **OK** をクリックして続行します。
9.  アーカイブ オブジェクトで、追加した **DocuValue** テーブルがレベル 1 の子エンティティで表示されていることを確認します。
10. アーカイブ オブジェクトを保存します。

## <a name="export"></a>輸出
このコマンドを使用すると、削除オブジェクトまたはアーカイブ オブジェクトを XML 形式のファイルにエクスポートできます。 削除オブジェクトをエクスポートするには、これらの手順に従います。
1.  **構成** &gt; **削除テンプレート/削除オブジェクト**とクリックして、リストから **PurchParmTable** を選択します。
2.  ツールで、**エクスポート**をクリックして、**エクスポート オブジェクト**ダイアログ ボックスを開きます。
3.  既定では、ファイル名はドライバー テーブルで、この場合は **PurchParmTable** です。 場所に移動してから、**保存**をクリックします。 **OK** をクリックして続行します。
4.  保存されたファイルを検索し、Web ブラウザーで開きます。 スキーマと、削除オブジェクトのグラフィカル表示とのその関係を理解するファイルを確認します。 **注記:** 同様の手順でアーカイブ オブジェクトをエクスポートできます。 **注記:** ツールバーの**インポート**コマンドは、**エクスポート**コマンドの前に表示されます。

## <a name="import"></a>インポート元
このコマンドを使用すると、削除オブジェクトまたはアーカイブ オブジェクトを XML 形式でインポートできます。 次のチュートリアルでは、変更されたパージ オブジェクトをエクスポート時の状態に復帰させる実践的な指示が提供されています。
1.  **構成** &gt; **削除テンプレート/削除オブジェクト**とクリックして、ドロップダウン リストから **PurchParmTable** を選択します。
2.  **詳細** フィルターを使用して、**テーブル削除** ウィンドウでデータ グリッドをフィルター処理し、レベル 1 のテーブルのみを表示します。
3.  データ グリッドですべてのテーブルを選択し、**削除** をクリックします。
4.  リレーションシップ ツリーに **PurchParmTable** のみが含まれていることを確認します。
5.  ツール バーで、**保存**をクリックして削除オブジェクトを保存します。 プロンプトに応答し、オブジェクトを上書きします。
6.  ツール バーで、**インポート**をクリックします。 **有効な XML ファイルを選択**ダイアログ ボックスで、前のセクションからファイルの場所に移動してから**開く**をクリックします。 **OK** をクリックして続行します。
7.  削除オブジェクトは、前のセクションでエクスポートしたときと同じ状態になりました。 **注記:** 同様の手順でアーカイブ オブジェクトをインポートできます。

## <a name="save"></a>保存
このコマンドを使用すると、削除オブジェクトを保存したり、新しいバージョンのアーカイブ オブジェクトを保存できます。 削除タスクで使用する前に削除オブジェクトとして削除テンプレートを保存する必要があります。 アーカイブ タスクで使用する前に、アーカイブ オブジェクトとしてアーカイブ テンプレートを保存する必要があります。 アーカイブ機能では、既存のアーカイブ オブジェクトを上書きすることはできません。 アーカイブ テンプレートまたはアーカイブ オブジェクトを保存するときは、アーカイブ オブジェクトの新しいバージョンをかならず作成してください。 アーカイブ タスクは常に、保存しているアーカイブ オブジェクトの最新のバージョンを使用します。 異なるバージョンのアーカイブ オブジェクトの作業については、このトピックの後半で説明されます。

## <a name="show-versions"></a>バージョンの表示
このコマンドを使用すると、さまざまなバージョンのアーカイブ オブジェクトで作業できます。 この機能を理解するのにには、次のチュートリアルを使用します。
1.  **構成** &gt; **アーカイブ テンプレート/アーカイブ オブジェクト** &gt; **BankDeposit** とクリックして **BankDeposit** アーカイブ オブジェクトを開きます。
2.  このアーカイブ オブジェクトに含まれるリレーションシップの階層構造、テーブル、およびルールが、Microsoft Dynamics AX の実装に適用されることを確認するアーカイブ オブジェクトを確認します。
3.  ツール バーの **保存** をクリックします。 **名前を付けて保存**ダイアログ ボックスで、アーカイブ オブジェクトの名前を入力します。 このバージョンの Archive Object に、新しく名前を指定する必要があります。 入力した名前が既にアーカイブ オブジェクトの別のバージョンで使用されている場合、エラー メッセージが表示されます。 BankDeposit\_1 と入力し、**保存**をクリックします。
4.  **OK** をクリックして続行します。
5.  ツール バーで、**バージョンの表示**をクリックします。 **バージョン履歴**ダイアログ ボックスで、**BankDeposit** を**アーカイブ テンプレート** リストから選択します。 アーカイブ オブジェクトの一覧には、保存したアーカイブ オブジェクトのさまざまなバージョンが含まれています。 リストには最新のバージョンが一番上に表示されているので、リストの一番上に BankDeposit\_1 が表示されます。
6.  すべてのバージョンを選択し、**アーカイブ オブジェクトの一覧に表示** をクリックし、アーカイブ オブジェクトのバージョンを操作します。 このバージョンを変更して保存すると、保存されたバージョンは最新バージョンになり、実行時にアーカイブ タスクによって使用されます。
7.  **閉じる**をクリックして、**バージョン履歴**ダイアログ ボックスを閉じます。
8.  ツールバーで、**アーカイブ テンプレート/アーカイブ オブジェクト** &gt; **BankDeposit** をクリックして開きます。 **注記:** データ管理フレームワークは、保存したアーカイブ オブジェクトの最新バージョンを開きます。 アーカイブ タスクは実行時に、アーカイブ オブジェクトの最新のバージョンを使用します。

## <a name="validate-all"></a>すべて検証
このコマンドを使用すると、すべてのテンプレートとオブジェクトをプログラムで検証できます。 Data Management Framework に含まれている削除テンプレートとアーカイブ テンプレートは、標準の Microsoft Dynamics AX アプリケーションの機能検証に基づいて作成されます。 インストールに、同じライセンス、コンフィギュレーション、およびセキュリティ キーが含まれていないことがあります。 結果として、アーカイブ オブジェクトまたは削除オブジェクトとして使用する前に、既定テンプレートを実装に対して検証する必要があります。 このコマンドを使用すると、すべての削除テンプレートまたはすべてのアーカイブ テンプレートをシングル クリックでプログラムで検証できます。 この機能は、まだ検証されていないテンプレートでのみ動作し、以前に検証されたテンプレートは無視されます。 妥当性確認のプロセス中に、Data Management Framework はこれらの特性を持つすべてのテンプレートとオブジェクトからテーブルを削除することもできます。
-   Microsoft Dynamics AX 実装の実稼働データベースにはありません。
-   実稼働データベースに無効な構成キーがあるフィールドに、リレーションシップまたはルールがあります。
-   0 (ゼロ) のレコード数があります。

ツール バーで、**すべて検証**をクリックして、**すべてのテンプレートとオブジェクトの検証**ウィンドウを開きます。
