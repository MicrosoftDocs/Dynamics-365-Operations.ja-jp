# <a name="goods-and-services-tax-gst-integration-for-cash-registers-for-india"></a>インド向けキャッシュ レジスターの商品及びサービス税 (GST) 統合

[!include [banner](../includes/banner.md)]

このトピックでは、Microsoft Dynamics 365 for Retail に関連するフィーチャのチュートリアルを提供します。 また、さまざまな種類の小売業務取引に対する GST の影響をハイライトして、レシートが販売時点管理 (POS) で印刷される小売り取引の会計および転記を示します。

> [!NOTE]
> このトピックは、Microsoft Dynamics 365 for Finance and Operations および Microsoft Dynamics 365 for Retail の両方に適用されます。

## <a name="prerequisites"></a>必要条件 

- Finance and Operations でのインド向け GST を設定します。 詳細については、[インドの商品及びサービス税 (GST)](../../financials/localizations/apac-ind-gst.md) を参照してください。
- 小売チャネル コンポーネントを構成します。 インド固有の機能を有効にするには、小売チャネル コンポーネントの拡張機能を構成する必要があります。 詳細については、[配置ガイドライン](./apac-ind-loc-deployment-guidelines.md)を参照してください。

## <a name="validate-tax-information-for-the-retail-store"></a>小売店舗の税情報の検証

小売店舗の税情報は、選択した小売倉庫から取得されます。 この倉庫は倉庫マスターで定義されます。 店舗からの構成済み税情報は POS レシートに印刷されます。 また、財務転記のために小売用バック オフィスの小売販売注文で更新されます。

以下の手順に従い、小売店舗の税情報を表示します。

1. **小売** \> **チャンネル** \> **小売店舗** \> **すべての小売店舗**の順に移動します。
2. 小売店舗を選択します。
3. **税金情報** クイックタブを選択します。

## <a name="configure-language-texts-and-custom-fields"></a>言語テキストおよびカスタム フィールドのコンフィギュレーション

POS レシート形式で使用される言語テキストおよびカスタム フィールドを構成することができます。 レシート設定を作成するユーザーの既定の会社は、言語テキスト設定が作成された法人と同じである必要があります。 または、ユーザーの既定の会社、および設定の作成対象の店舗の法人の両方で、同じ言語テキストを作成する必要があります。

### <a name="set-up-the-pos-language-text"></a>POS 言語テキストの設定

1. **小売** \> **チャネル設定** \> **POS 設定** \> **POS プロファイル** \> **言語テキスト** の順に移動します。
2. **POS** タブの、**POS 言語テキスト** クイックタブで、テキストの言語 ID を選択します。 言語はユーザーの優先言語と一致する必要があります。
3. **テキスト ID** フィールドに、**900001** 異常の一意の ID を入力します。
4. **テキスト**フィールドで、言語テキストを入力します。

![言語テキスト](media/apac-ind-gst-language-text.png)

### <a name="create-custom-fields"></a>カスタム フィールドの作成

カスタム フィールドを作成するときは、**キャプション テキスト ID** フィールドの値が、**言語テキスト** ページの **テキスト ID** フィールドに入力した値と一致する必要があります。

1. **小売** \> **チャネル設定** \> **POS 設定** \> **POS プロファイル** \> **カスタム フィールド** の順に移動します。
2. フィールドの名前を入力します。
3. フィールド タイプを選択します。
4. **キャプション テキスト ID** フィールドで、**言語テキスト** ページの言語テキストのいずれかに **テキスト ID** 値を入力します。

![カスタム フィールド](media/apac-ind-gst-custom-fields.png)

## <a name="create-the-receipt-format"></a>受領書フォーマットの作成

受領書フォーマット デザイナーを使用して、適切な領収書セクションにカスタム フィールドを追加することができます。 詳細については、[レシートのテンプレートおよび印刷](../receipt-templates-printing.md)を参照してください。

1. **小売** \> **チャネル設定** \> **POS 設定** \> **POS プロファイル** \> **受領書フォーマット**の順に移動します。
2. **レシート** 領収書タイプのために領収書書式を選択して、必要な変更を加えます。

![受領書フォーマットのデザイン](media/apac-ind-gst-receipt-format.png)

## <a name="update-receipt-profiles"></a>領収書プロファイルの更新

領収書フォーマットの作成すると、領収書プロファイルにフォーマットを割当てることができます。

以下の手順を実行して、領収書プロファイルを更新します。

1. **小売** \> **設定** \> **POS** \> **レシート プロファイル** の順に移動します。
2. 更新するレシート プロファイルを選択します。
3. **編集**を選択します。
4. リスト内のレシート タイプごとに、受領書フォーマットを選択します。

## <a name="update-the-pos-invoice-number"></a>POS 請求書番号の更新

顧客の小売取引の請求書番号を使用して POS レシート番号を調整することができます。 **小売パラメータ** ページの **転記** タブで **POS 請求書番号を更新** オプションに **はい** をセットする場合、対応する小売販売注文の **取引 ID** フィールドに POS 請求書番号が入力されます。

既存の領収書番号書式に店舗番号と端末番号の両方が含まれる場合にのみ、**POS 請求書番号を更新** オプションに **はい** をセットすることができます。  次の図は、レシート番号付けに店舗番号と端末番号が含まれる、POS 機能プロファイルを示します。

![POS 機能プロファイル](media/apac-ind-gst-pos-functionality-profile.png)

## <a name="run-a-distribution-schedule"></a>配送スケジュールの実行

小売用バックオフィスから POS データベースに Tax Engine (GTE) データを同期するには、ジョブを **配送スケジュール** ページに追加する必要があります。

以下の手順に従い、ジョブが存在していることを検証して、ジョブを実行します。

1. **小売** \> **定期処理** \> **データ配送** \> **配送スケジュール** の順に移動します。
2. 新しいジョブ、**1180** が **汎用税エンジン** のために追加されていることを検証します。
3. すべてのジョブを実行します (**9999**)。

## <a name="example-scenarios"></a>シナリオ例

次のシナリオ例はインドの取引サンプルについて説明します。

- シナリオ 1: 登録済の顧客に対する販売
- シナリオ 2: 顧客に対する課税対象商品の販売
- シナリオ 3: GSTが価格に包まれる、匿名の顧客への課税対象商品の販売
- シナリオ 4: 免税品の販売
- シナリオ 5: GST を含む取引の返品

### <a name="scenario-1-sell-to-a-registered-customer"></a>シナリオ 1: 登録済の顧客に対する販売

登録済み顧客への販売は *企業間* (B2B) 販売と呼ばれます。 店舗の場所と供給の位置 (つまり、顧客の住所) が同じ州にある場合、取引は *イントラスタット販売* で、中央 GST (CGST) および州 GST (SGST) が支払われる必要があります。 店舗の場所および供給の位置が異なる州にある場所、取引は *州間販売* で、統合された GST (IGST) が支払われる必要があります。

> [!NOTE]
> これらの取引の場合、顧客の住所および登録番号のフィールドが必要です。

1. POS へのサインイン
2. 品目を入力してから、**入力** を選択します。

    ![POS 顧客注文](media/apac-ind-gst-pos-customer-order.png)

3. GST 計算を検証します。 税の設定で定義されているレートを検討します。

    | 品目/サービス | 単価 | 税率          | CGST     | SGST      |
    |--------------|------------|--------------------|----------|-----------|
    | M0001        | 10,000.00  | CGST 12%、SGST 11% | 1,200.00 | 1,100.00  |
    | M0002        | 5,000.00   | CGST 10%、SGST 10% | 500.00   | 500.00    |
    | S0001        | 1,200.00   | CGST 11%、SGST 5%  | 132.00   | 60.00     |
    |              | 16,200.00  |                    | 1,832.00 | 1,660.00  |
    | 合計金額 |            |                    |          | 19,692.00 |

4. **注文** \> **顧客注文の作成** の順に選択します。
5. **顧客の追加** を選択して、顧客アカウントを選択します。
6. **すべて集荷** を選択します。
7. 店舗および集荷日を選択します。
8. **OK** を選択します。

    ![POS 顧客注文](./media/apac-ind-gst-pos-customer-order2.png)

    > [!NOTE]
    > この例では、店舗場所の州が Delhi で、顧客住所の州も Delhi です。 州が同じであるため、イントラスタット GST が計算されます。

9. **正確** を選択して預金支払を処理します。
10. 領収書の検証:

    1. **仕訳帳の表示** を選択します。
    2. 取引を選択します。
    3. **レシート** を選択します。

    ![レシート例](media/apac-ind-gst-s1-receipt1.png)

11. 小売用バックオフィスで販売注文および税ドキュメントを検証します:

    1. **小売** \> **顧客** \> **すべての販売注文** の順に移動します。
    2. 販売注文を選択します。
    3. アクション ペインの、**販売** タブの、**税** グループで、**税務書類** を選択します。

    ![税務書類](media/apac-ind-gst-tax-document.png)

12. 顧客の注文を取り消して処理します:

    1. POS へのサインイン
    2. **注文の取り消し** を選択します。
    3. 注文を検索して選択します。
    4. **ピッキングと梱包** \> **集荷** の順に選択します。
    5. **すべて選択** および **集荷** を選択します。

    ![顧客注文の処理](media/apac-ind-gst-process-customer-order.png)

13. **正確** を選択して支払を処理します。
14. 領収書を検証します。

    ![レシート例](media/apac-ind-gst-s1-receipt2.png)

15. 伝票取引を検証します:

    1. **小売** \> **顧客** \> **すべての販売注文** の順に移動します。
    2. 販売注文を選択します。
    3. アクション ウィンドウの、**請求書** タブで、**請求仕訳帳** を選択します。
    4. **伝票** を選択します。

        | 勘定科目名  | 借方金額 (Rs.) | クレジット金額 (Rs.) |
        |----------------------|--------------------|---------------------|
        | 顧客口座     | 19,692.00          |                     |
        | CGST 未払勘定 |                    | 1,832.00            |
        | SGST 未払勘定 |                    | 1,660.00            |
        | 売上勘定        |                    | 16,200.00           |

    5. **税務書類** を選択します。
    6. 領収書番号が取引 ID として更新されたことを確認します。

    ![税務書類](media/apac-ind-gst-tax-document2.png)

### <a name="scenario-2-sell-taxable-goods-to-a-consumer"></a>シナリオ 2: 顧客に対する課税対象商品の販売

未登録の顧客に販売するとき、その販売は *企業と顧客間* (B2C) 販売と呼ばれます。 税は B2B および B2C 販売と同じ方法で計算されます。 

1. POS へのサインイン
2. 品目を入力してから、**入力** を選択します。
3. **顧客の追加** を選択して、顧客アカウントを選択します。

    ![取引の例](media/apac-ind-gst-example-trx.png)

    > [!NOTE]
    > この例では、店舗場所の州は Delhi ですが、顧客住所の州は Bengaluru (Bangalore) です。 州が異なるため、州間 GST が計算されます。

4. **正確** を選択して支払を処理します。
5. 領収書の検証:

   1. **仕訳帳の表示** を選択します。
   2. 取引を選択します。
   3. **レシート** を選択します。

      ![領収書の検証](media/apac-ind-gst-receipt-validation.png)

6. 小売用バックオフィスでの小売販売請求書を確認します:

    1. **小売** \> **小売 IT** \> **データ配送** の順に移動します。
    2. ジョブ **P-0001** を実行します (**チャネル取引**)。
    3. ページを閉じます。

7. 明細書の転記:

    1. **小売** \> **チャネル** \> **小売店舗** \> **未処理の明細書** の順に移動します。
    2. 明細書を作成します。
    3. **明細書の計算** および **明細書の転記** の順に選択します。

8. 伝票取引を検証します:

    1. **小売** \> **顧客** \> **すべての販売注文** の順に移動します。
    2. 売上請求書を選択します。
    3. **販売注文行** \> **税金情報** の順に選択します。
    4. 該当するタブで、場所 (店舗アドレス) および顧客の住所を確認します。

        ![税金情報](media/apac-ind-gst-tax-info.png)

    5. **OK** を選択します。
    6. アクション ウィンドウの、**請求書** タブで、**請求仕訳帳** を選択します。
    7. **伝票** を選択します。

        | 勘定科目名  | 借方金額 (Rs.) | クレジット金額 (Rs.) |
        |----------------------|--------------------|---------------------|
        | 顧客口座     | 12,500.00          |                     |
        | IGST 未払勘定 |                    | 2,500.00            |
        | 売上勘定        |                    | 10,000.00           |

    8. **税務書類** を選択します。
    9. 領収書番号が取引 ID として更新されたことを確認します。

### <a name="scenario-3-sell-taxable-goods-to-an-anonymous-customer-where-gst-is-price-inclusive"></a>シナリオ 3: GSTが価格に包まれる、匿名の顧客への課税対象商品の販売

1. 小売店舗での価格包含性を定義します。

    1. **小売** \> **チャンネル** \> **小売店舗** \> **すべての小売店舗**の順に移動します。
    2. 小売店舗を選択します。
    3. **税込価格** オプションに **はい** をセットします。

2. 配送スケジュールを実行します:

    1. **小売** \> **小売 IT** \> **データ配送** の順に移動します。
    2. POS データベース内の変更を更新するジョブを実行します。
    3. ページを閉じます。

3. トランザクションの入力:

   1. POS へのサインイン
   2. 品目を入力してから、**入力** を選択します。 この例では、次の値を持つ品目を使用します:

       - **課税対象値:** 10,000.00
       - **CGST:** 12 %
       - **SGST:** 11 %

      ![取引の例](media/apac-ind-gst-trx-example.png)

4. **正確** を選択して支払を処理します。
5. 領収書を検証します。

    ![レシート例](media/apac-ind-gst-s3-receipt4.png)

6. 小売用バックオフィスでの小売販売請求書を確認します:

    1. **小売** \> **小売 IT** \> **データ配送** の順に移動します。
    2. ジョブ **P-0001** を実行します (**チャネル取引**)。
    3. ページを閉じます。

7. 明細書の転記:

    1. **小売** \> **チャネル** \> **小売店舗** \> **未処理の明細書** の順に移動します。
    2. 明細書を作成します。
    3. **明細書の計算** および **明細書の転記** の順に選択します。

8. 伝票取引を検証します:

   1. **小売** \> **顧客** \> **すべての販売注文** の順に移動します。
   2. 売上請求書を選択します。
   3. アクション ウィンドウの、**請求書** タブで、**請求仕訳帳** を選択します。
   4. **伝票** を選択します。


      | 勘定科目名  | 借方金額 (Rs.) | クレジット金額 (Rs.) |
      |----------------------|--------------------|---------------------|
      |   顧客口座   |     10,000.00      |                     |
      | CGST 未払勘定 |                    |       975.61        |
      | SGST 未払勘定 |                    |       894.31        |
      |    売上勘定     |                    |      8,130.08       |


   5. **税務書類** を選択します。
   6. GST 参照番号順序グループで定義されている GST 番号順序に従って取引 ID が更新されることを確認します。

      ![税務書類のトランザクション ID](media/apac-ind-gst-tax-doc-trx-id.png)

### <a name="scenario-4-sell-an-exempted-good"></a>シナリオ 4: 免税品の販売

1. POS へのサインイン
2. 免税品目を入力します。

    ![免税品目トランザクション](media/apac-ind-gst-exempted-item-trx.png)

3. **正確** を選択して支払を処理します。
