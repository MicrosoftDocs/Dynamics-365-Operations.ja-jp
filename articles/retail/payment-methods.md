# <a name="payment-methods"></a><span data-ttu-id="7a458-104">支払方法</span><span class="sxs-lookup"><span data-stu-id="7a458-104">Payment methods</span></span>

[!include[banner](includes/banner.md)]


<span data-ttu-id="7a458-105">小売業者が受け入れる各支払タイプは、システムの設定時にコンフィギュレーションする必要があります。</span><span class="sxs-lookup"><span data-stu-id="7a458-105">Each payment type that a retailer accepts must be configured when the system is set up.</span></span> <span data-ttu-id="7a458-106">この記事では、設定可能な支払タイプおよびその設定方法について説明します。</span><span class="sxs-lookup"><span data-stu-id="7a458-106">This article describes the payment types that you can set up and describes the process for setting them up.</span></span>

<span data-ttu-id="7a458-107">小売業者は、販売する製品およびサービスと引き換えにさまざまなタイプの支払方法を受け入れることができます。</span><span class="sxs-lookup"><span data-stu-id="7a458-107">Retailers can accept various types of payment in exchange for the products and services that they sell.</span></span> <span data-ttu-id="7a458-108">最も一般的な支払形式は現金ですが、小売業者は小切手、カード、伝票などの形式で支払を受け取ることもできます。</span><span class="sxs-lookup"><span data-stu-id="7a458-108">Although cash is the most common form of payment, retailers can also receive payment in the form of checks, cards, vouchers, and so on.</span></span> <span data-ttu-id="7a458-109">小売業者が受け入れる各支払タイプは、システムの設定時に Microsoft Dynamics 365 for Retail でコンフィギュレーションする必要があります。</span><span class="sxs-lookup"><span data-stu-id="7a458-109">Each payment type that the retailer accepts must be configured in Dynamics 365 for Retail when the system is set up.</span></span> <span data-ttu-id="7a458-110">次の一覧は、Dynamics 365 for Retail で設定できる各支払タイプを示しています。</span><span class="sxs-lookup"><span data-stu-id="7a458-110">The following list describes each payment type that can be set up in Dynamics 365 for Retail:</span></span>

-   <span data-ttu-id="7a458-111">\[現金\]: 紙幣や硬貨など、通貨の物理的な形式。</span><span class="sxs-lookup"><span data-stu-id="7a458-111">**Cash** – Money in the physical form of currency, such as banknotes and coins.</span></span> <span data-ttu-id="7a458-112">この通貨は、法人基本通貨または店舗の国内通貨のいずれかにすることができます。</span><span class="sxs-lookup"><span data-stu-id="7a458-112">This currency can be either the company currency or the store's local currency.</span></span>
-   <span data-ttu-id="7a458-113">\[小切手\]: 特定の銀行に特定の通貨で特定の金額の支払を振り出すことを指示する流通証券。</span><span class="sxs-lookup"><span data-stu-id="7a458-113">**Check** – A negotiable instrument that instructs payment of a specific amount of a specific currency, and that is drawn on a specific bank.</span></span> <span data-ttu-id="7a458-114">小切手の有効期限は、特に指定のない限り、一般に、無期限にまたは発行日から 6 か月です。</span><span class="sxs-lookup"><span data-stu-id="7a458-114">A check is typically valid either indefinitely or for six months after the date of issue, unless another period of validity is specified.</span></span> <span data-ttu-id="7a458-115">この期間は、小切手が振り出された銀行によって異なります。</span><span class="sxs-lookup"><span data-stu-id="7a458-115">This period varies, depending on the bank that the check is drawn on.</span></span> <span data-ttu-id="7a458-116">指図式小切手、払戻小切手、持参人払式小切手、受取人指定小切手など、さまざまな種類の小切手があります。</span><span class="sxs-lookup"><span data-stu-id="7a458-116">There are various kinds of checks, such as order checks, counter checks, bearer checks, and account payee checks.</span></span> <span data-ttu-id="7a458-117">各店舗の支払方法として小切手を設定できます。</span><span class="sxs-lookup"><span data-stu-id="7a458-117">You can set up checks as a payment method for each store.</span></span> <span data-ttu-id="7a458-118">小切手は、法人レベルまたは店舗レベルで定義された通貨で受け入れることができます。</span><span class="sxs-lookup"><span data-stu-id="7a458-118">Checks can be accepted in the currency that is defined at either the company level or the store level.</span></span> <span data-ttu-id="7a458-119">店舗で支払として小切手を受け入れる前に、支払方法として小切手を設定する必要があります。</span><span class="sxs-lookup"><span data-stu-id="7a458-119">You must set up checks as a payment method before you can accept a check as payment in a store.</span></span>
-   <span data-ttu-id="7a458-120">\[通貨\]: 法人の既定の通貨以外の主要な支払形式。</span><span class="sxs-lookup"><span data-stu-id="7a458-120">**Currency** – The primary form of payment other than the company's default currency.</span></span> <span data-ttu-id="7a458-121">硬貨と紙幣はどちらも通貨の形式です。</span><span class="sxs-lookup"><span data-stu-id="7a458-121">Coins and paper money are both forms of currency.</span></span> <span data-ttu-id="7a458-122">通貨の支払方法は、使用されるすべての通貨を表します。</span><span class="sxs-lookup"><span data-stu-id="7a458-122">The currency payment method represents all currency that is used.</span></span> <span data-ttu-id="7a458-123">この支払方法を使用する前に、通貨を設定し、通貨の小売取引情報を指定する必要があります。</span><span class="sxs-lookup"><span data-stu-id="7a458-123">Before you can use this payment method, you must set up currencies and specify retail exchange information for the currencies.</span></span>
-   <span data-ttu-id="7a458-124">**カード** – デビット カード、クレジット カードなど、使用されるあらゆる種類のカード。</span><span class="sxs-lookup"><span data-stu-id="7a458-124">**Card** – All the kinds of cards that are used, such as debit cards and credit cards.</span></span> <span data-ttu-id="7a458-125">あらゆる種類のカードを表す 1 つのカード支払方法を組織レベルで設定することは良い考えです。</span><span class="sxs-lookup"><span data-stu-id="7a458-125">It's a good idea to set up one card payment method at the organization level, to represent every kind of card.</span></span> <span data-ttu-id="7a458-126">その後、同じ設定を使用して処理されるカードまたはカード セットごとに、支払方法を店舗レベルで設定します。</span><span class="sxs-lookup"><span data-stu-id="7a458-126">Then, at the store level, set up a payment method for each card or set of cards that is processed by using the same settings.</span></span> <span data-ttu-id="7a458-127">店舗での支払としてカードを受け入れる前に、デビット カード、クレジット カードなど、市場で一般に使用できるメーカーのカードを設定する必要があります。</span><span class="sxs-lookup"><span data-stu-id="7a458-127">You must set up the manufacturer cards that are available in the market, such as debit cards and credit cards, before you can accept the cards as payment in a store.</span></span>
-   <span data-ttu-id="7a458-128">[クレジット メモ] – POS で発行または引き換えが行われるクレジット メモ。</span><span class="sxs-lookup"><span data-stu-id="7a458-128">**Credit memo** – Credit memos that are issued or redeemed at the point of sale.</span></span> <span data-ttu-id="7a458-129">クレジット メモには、クレジット メモと、返品販売に対して発行される返品クレジット メモがあります。</span><span class="sxs-lookup"><span data-stu-id="7a458-129">The credit memo can be a credit or a return credit memo that is issued against a return sale.</span></span> <span data-ttu-id="7a458-130">クレジット メモの一部だけが引き換えられた場合、新しい残高に対する新しいクレジット メモが発行されます。</span><span class="sxs-lookup"><span data-stu-id="7a458-130">If credit memos are only partially redeemed, the program issues a new credit memo for the new balance.</span></span> <span data-ttu-id="7a458-131">新しいクレジット メモには、新しい番号が割り当てられます。</span><span class="sxs-lookup"><span data-stu-id="7a458-131">The new credit memo has a new number.</span></span> <span data-ttu-id="7a458-132">クレジット メモは 1 回だけ使用可能であり、使用された番号はすべてシステムで記録されます。</span><span class="sxs-lookup"><span data-stu-id="7a458-132">A credit memo can be used only one time, and the system keeps a record of all the numbers that are used.</span></span> <span data-ttu-id="7a458-133">レコードは [クレジット メモ テーブル] ページに表示できます。</span><span class="sxs-lookup"><span data-stu-id="7a458-133">The record can be viewed on the **Credit memo table** page.</span></span> <span data-ttu-id="7a458-134">顧客は、クレジット メモの金額を超えて引き換えることはできません。</span><span class="sxs-lookup"><span data-stu-id="7a458-134">A customer can't redeem more than the value of the credit memo.</span></span>
-   <span data-ttu-id="7a458-135">\[ギフト カード\]: POS で発行および引き換えが行われるギフト カード。</span><span class="sxs-lookup"><span data-stu-id="7a458-135">**Gift Card** – Gift cards that are issued and redeemed at the point of sale.</span></span> <span data-ttu-id="7a458-136">ギフト カードの金額を超えて支払うことはできません。</span><span class="sxs-lookup"><span data-stu-id="7a458-136">Overpayment isn't allowed on gift cards.</span></span>

-   <span data-ttu-id="7a458-140">[ロイヤルティ ポイント] – ロイヤルティ プログラムを通して顧客が収集するポイント。</span><span class="sxs-lookup"><span data-stu-id="7a458-140">**Loyalty points** – The points that customers accumulate through loyalty programs.</span></span> <span data-ttu-id="7a458-141">ロイヤルティ プログラムを作成すると、顧客はポイントを獲得してさまざまな方法で交換することができます。</span><span class="sxs-lookup"><span data-stu-id="7a458-141">If you create loyalty programs, customers can earn points and then redeem them in various ways.</span></span> <span data-ttu-id="7a458-142">たとえば、あるロイヤルティ プログラムで、顧客が割引の形式でロイヤルティ ポイントを買い戻し、さらに支払形式として使用できます。</span><span class="sxs-lookup"><span data-stu-id="7a458-142">For example, in some loyalty programs, customers can redeem loyalty points in the form of a discount or even use them as a form of payment.</span></span>
