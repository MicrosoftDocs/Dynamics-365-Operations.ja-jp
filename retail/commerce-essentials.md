# <a name="commerce-essentials-overview"></a>Commerceエッセンシャルの概要

この記事は、Dynamics、Microsoft Dynamics 365の(POS)とコマースEのfor Retail POSのコンフィギュレーション オプションであるコマースの概要について説明します。 Commerce エッセンシャルは、小売業者がすばやく Microsoft Dynamics ベースの小売用ソリューションを稼働させ、小売固有の機能に焦点を合わせるのを助ける、すぐに使える小売用ワークロードを提供します。 

<a name="overview"></a>概要
--------

ビジネス ソフトウェアの環境には、それぞれが極めて特殊な機能を実行する多くのシステムが含まれます。 多くの場合、小売業者は、Microsoft Dynamics 365 for Retailの管理を支援しますが、その存在するか、または人事管理の(HR)システムを取り替え場合に役立ちます。 これらの小売業者のために、小売固有のワークロードが必要です。 コマースの概要は、Dynamics 365 for Operationsのコンフィギュレーション オプションとしてこれらの小売固有の機能があります。 コンフィギュレーションが完了したら、小売固有でないソリューションをすべて非表示にし、小売組織の管理という重要なことにフォーカスできるようにします。 Commerce エッセンシャルは、Microsoft Dynamics AX 2012 R3 CU8 のコンフィギュレーション オプションとして最初に導入されました。 それ以降、従来型の組織だけでなく、無指向性チャンネルの小売業者に対応できる解決策となるよう、改良が多数加えられてきました。

## <a name="what-is-included"></a>含まれているもの。
コマースの概要は、工程の小売用ソリューションのコアDynamics 365に作成されるコール センターの機能を除くすべての小売でバック オフィスの機能が含まれます。 すべてのチャンネル、作業者、製品、店舗、レジスター、およびデバイス管理機能が、1 つのナビゲーション ノードに含まれています。 また、すべての完全な小売ワークスペースが含まれており、すぐに使えるロール固有のランディング ページを提供します。 コマースの概要は、Retail POSの現代 (MPOS) とコマースEチャンネルに寄与する携帯電話シナリオまたは基本的な在庫がある小売業者を対象とします。 [Commerce エッセンシャル] を使用して、小売業者は 1 つの場所でこれらのチャンネルを管理できるため、本当の無指向性チャンネルの機能を得られます。

## <a name="how-is-commerce-essentials-different"></a>[Commerce エッセンシャル] は、どこが異なっているのか。
統合ナビゲーションや小売専用ページの提供に加えて、[Commerce エッセンシャル] は、フィールドや管理などのすべての非小売ページ要素を非表示にします。 小売業者が手元にあるタスクに集中するのに役立つ、合理化されたユーザー インターフェイスという結果になります。 完全な製品のライセンスを持っている小売業者は、小売用のユーザーに対して開始点としてコマースの概要を使用できます。 そのほかの要素は、高度HR機能のような市販の概要に設定されている使用許諾契約書合意条件に従って、必要に応じて追加することができます。 [Commerce エッセンシャル] が提供する合理化および簡略化の一部として、会計エクスポート機能により、小売業者は　[Commerce エッセンシャル] で未収の合計を既存の会計システムの勘定番号へマップすることができます。 小売システムが財務データを生成するときに、このデータは [Commerce エッセンシャル] から小売業者の会計システムに簡単にエクスポートできます。 高度の統合機能が必要な場合は、コマースの概要は、Dynamics 365 for Operationsに含まれるすべてのインポートとエクスポート フレームワークが含まれます。 既定で含まれる何百ものデータ エンティティにより、[Commerce エッセンシャル] には、複数の特殊なシステムが使用されている複数階層の環境に容易に適合する柔軟性があります。
