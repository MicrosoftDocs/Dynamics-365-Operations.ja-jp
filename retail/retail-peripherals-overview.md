## <a name="troubleshooting"></a>トラブルシューティング
### <a name="modern-pos-can-detect-the-hardware-station-in-its-list-for-selection-but-it-cant-complete-the-pairing"></a>現代品目を選択するリストのハードウェア ステーションを検出できます組み合わせを実行できません

**ソリューション: **潜在的障害ポイントの次の一覧を確認してください:

-   選択した現代POSであるコンピュータは、ハードウェアのステーションを実行するコンピュータで使用される証明書を信頼されます。
    -   この設定は、Webブラウザで確認するには、次のURLは、実行されます: https://Computerの&lt;名前&gt;:&lt;ポート&gt;番号やHardwareStation/ping。
    -   このURLは証明書が信頼されているかどうかをコンピュータにアクセスできるブラウザを表すことを確認している場合でも、pingを使用します。  (たとえば、Internet Explorerで、ロックのアイコンは、アドレス バーに表示されます。 このアイコンをクリックすると、Internet Explorerは証明書が現在信頼されているかどうかを確認します。 ローカル コンピュータに表示される証明書の詳細を表示することにより、証明書をインストールできます)。
-   ハードウェアのステーションを実行するコンピュータで、ハードウェアのステーションで使用されるポートは、ファイアウォールで開きます。
-   ハードウェアのステーションは、ハードウェアの最後に実行を尋ねるを配置するインストール商社情報ツールを使用して正しく商社の先情報をインストールします。

### <a name="modern-pos-cant-detect-the-hardware-station-in-its-list-for-selection"></a>現代品目を選択するリストのハードウェア ステーションを検出できます

**ソリューション: **次の要素のいずれかによりこの問題が生じることができます:

-   ハードウェアのステーションは本部に設定されませんでした。 ハードウェアのステーションのプロファイルとハードウェアのステーションが正しく入力されていることを確認する、このトピックの手順を先に使用します。
-   ジョブがチャンネルの構成を更新する実行しませんでした。 この場合、チャンネルの構成に1070のジョブを実行します。

### <a name="modern-pos-doesnt-reflect-new-cash-drawer-settings"></a>現代POSによって、新しいキャッシュ ドロワーの設定を反映されません

**ソリューション: **近い現在のバッチ。 現代POSにキャッシュ ドロワーへの変更は、現在のバッチが閉じられるまで更新されません。

### <a name="modern-pos-is-reporting-an-issue-with-a-retail-peripheral"></a>現代POSでは、の周辺機器がある問題を報告します。

**ソリューション: **この問題のある標準的な理由を挙げます。:

-   他のデバイスドライバのConfiguration Utilityが終了したことを確認します。 これらのユーティリティは開かれている場合、現代POSまたはハードウェアのステーションはデバイスを要求することを防ぐかもしがあります。
-   Retailの周辺機器が複数のPOSのデバイスに関して共有する場合、次のカテゴリの一つに属することを確認してください:
    -   キャッシュ ドロワー
    -   レシート プリンター
    -   支払ターミナル

    周辺機器がこれらのカテゴリのいずれに属さなければ周辺機器が複数のPOSのデバイス間で共有されるようにするために、ハードウェアのステーションは作成されません。
-   場合によっては、デバイスドライバによってコモン コントロール オブジェクト(CCOs)が正しく動作しなくなりますできます。 デバイスが、新たにインストールされている場合、またはそのほかの問題に正しく勤務または気づけば、通常はCCOs再インストールして問題を解決できます。 CCOsをダウンロードするには、http://monroecs.com/oposccos_current.htmを<参照してください>。
-   テスト環境またはトラブルシューティングの間に、な周辺を変更する場合は、品目を更新するためにキャッシュ待ちの代わりにIISをリセットする必要があるかもしがあります。 IISをリセットするには、次の手順に従います。:
    1.  **開始**メニューから、" ** CMD **。
    2.  検索結果で、右クリック**コマンド プロンプト** **クリックし、[管理者として実行します。**。
    3.  **コマンド プロンプト**ウィンドウで、タイプiisreset ** /Restart **と、Enterキーを押します。
    4.  IISの再起動後、現代POS.を再起動します。
-   周辺機器に頻繁に変更を加えるときに、よくPOSのクライアントを起動し、出れば、前のPOSセッションのdllhostプロセスは、現在のセッションに干渉できます。 この場合、デバイスが前のセッションを管理するダイナミックリンク ライブラリ(DLL)のホストを閉じるまで使用可能ではないかもしがあります。 DLLのホストを終了するには、次の手順に従います。:
    1.  **開始**メニューから入力します。**タスク マネージャー**。
    2.  検索結果には、をクリックして**タスク マネージャー**。
    3.  タスク マネージャでは、**詳細**タブで、テーブルをアルファベット順に並べ替えるように**名前**名前で分類する列ヘッダーをクリックします。
    4.  dllhost.exeが見つかるまでスクロールします。
    5.  各DLLのホストを選択し、[]をクリックします。**決算タスク**。
    6.  DLLのホストがクローズされていると、POS現代.を再起動します。
