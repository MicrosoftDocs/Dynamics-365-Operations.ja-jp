# <a name="track-commissions-in-pos-using-sales-groups"></a>売上グループを使用してPOSの追跡コミッション

これは、顧客提供のサポート、販売、クロス セールス、トランザクションの処理を使用した関連して販売を追跡する一般的なRetailの提案です。

販売担当者による販売を追跡すると、レジ担当者による販売にスピードと実効率の基準になりますが能力を販売する関連する測定です。 販売担当者が追跡する販売は、よく使用されますコミッションまたはそのほかの還付を計算するために使用されます。

## <a name="configuring-a-worker-to-be-a-sales-representative-in-pos"></a>POS販売担当者にワーカーを設定します。
労働災害が売上グループに追加すると、コミッションに適用できるようになり、システムの販売担当者として指定できます。 売上グループに存在しないため、コミッションに適用がないし(POS)、アプリケーションの品目が販売担当者として表示されません。 POSで、仕入先のリストは、店舗に割り当てられている1人以上の作業者を含むすべての売上グループから取得されます。 リストには、売上グループのIDと名前 (IDの組み合わせとしてPOSに表示されます: 名前)。 既定の売上グループは、小売業者がPOSのライン販売担当者を自動的に設定することもできます。サポート シナリオにために割り当てることができます。 ユーザーは、売上グループのメンバがあるため、選択できます。

## <a name="displaying-the-sales-representative-information-on-the-pos-transactions-screen"></a>POSでトランザクションの画面の販売担当者情報の表示
POSでトランザクションの画面のレイアウトやコンテンツは、店舗、レジスター、またはために画面レイアウト デザイナー、割り当てられた画面レイアウトを使用して設定できます。 **販売担当者**フィールドは、入荷ウィンドウの[明細行]タブに追加できます。  これは、トランザクションの画面の明細行ごとに指定売上グループのIDを表示します。

## <a name="adding-sales-representative-operations-to-pos-button-grids"></a>販売担当者の工程がPOSに追加するグリッドのボタンをかけます
POSは、ユーザーがPOS操作にアクセスする画面レイアウトに含まれるボタン グリッドを設定することができます。 次のPOS操作はその販売担当者に関係するボタン グリッド ボタンに割り当てることができます。

|                                           |                                                                                                                                                                                                                                                                                              |
|-------------------------------------------|----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| [**操作**]                             | **Description**                                                                                                                                                                                                                                                                              |
| 明細行の販売担当者の設定          | このPOS操作は適用できる売上グループ (IDの一覧を表示します: 店舗の名前)。 このリストから販売グループを選択すると、現在のトランザクション明細行の値を設定します。                                                                                                            |
| 明細行の販売担当者のクリア        | このPOS操作は、現在のトランザクション明細行から現在の売上グループの値を特性。                                                                                                                                                                                                  |
| トランザクションの販売担当者の設定   | このPOS操作は適用できる売上グループ (IDの一覧を表示します: 店舗の名前)。 このリストから販売グループを選択すると、現在のトランザクションの既定値を設定します。 割り当てられた売上グループがない既存の行が、後で、追加された行設定されます。 |
| トランザクションに関する明確な販売担当者 | このPOS操作は、現在のトランザクションから現在の既定のグループの値を特性。 これは既にトランザクションにある行を影響しません。                                                                                                                             |

## <a name="calculating-commissions"></a>コミッションの計算
コミッションは、明細書の転記または販売注文の転記時に選択した売上グループのために計算されます。 コミッション金額が顧客の売上グループおよび関連する手数料の計算の設定/トランザクションの製品で定義されているためのコミッションの比率に基づいて決定します。
